# 交互模式库 (Interaction Pattern Library)

> **状态**: 草稿 (Draft) — 启动于 Technical Setup → Pre-Production 门控检查
> **版本**: v0.1
> **适用范围**: 全部 UI 屏幕
> **关联文档**: `design/art/art-bible.md` §7, `src/DndGame/UI/PixelTheme.cs`, `design/ux/hud.md`

---

## 1. 概述

本文档定义《酒馆与命运》中跨屏幕共享的通用交互模式。任何新增 UI 屏幕必须：

1. 优先使用本文档中已有的交互模式
2. 如需新模式，先在此文档登记（附使用场景和理由）
3. 不引入与现有模式冲突的交互行为

## 2. 模式 1: 确认/取消对话框

### 使用场景
- 删除存档、解散角色、覆盖文件等**不可逆操作**
- LLM API 调用失败时的**错误提示**

### 视觉规范
```
┌─────────────────────────────────┐
│  ⚠ 确认解散 [角色名]？          │ ← 标题（18px, 白色）
│                                  │
│  该角色将永久离开队伍，           │ ← 描述（14px, SecondaryText）
│  此操作不可撤销。                 │
│                                  │
│  ┌──────────┐ ┌──────────┐      │
│  │   确认   │ │   取消   │      │ ← 按钮（PixelTheme.ButtonStyle）
│  └──────────┘ └──────────┘      │
└─────────────────────────────────┘
```

### 交互规则
| 输入 | 行为 |
|------|------|
| `Enter` | 执行确认操作（默认焦点在"确认"按钮） |
| `Esc` | 执行取消操作，关闭对话框 |
| `← →` | 在按钮间切换焦点 |
| `Tab` | 焦点在按钮间循环 |

### 键盘快捷约定
- **确认（危险操作）**: `Enter` 不可直接触发 —— 需先 `→` 将焦点移到"确认"按钮再 `Enter`
- **确认（安全操作）**: `Enter` 可直接触发（默认焦点在确认按钮）
- **取消**: 始终 `Esc`

### 无障碍
- 焦点状态用 2px 边框高亮（`ButtonHoverBorder` 颜色）
- 确认/取消按钮**永远不**仅靠颜色区分（文字本身就是区分）

## 3. 模式 2: 列表选择

### 使用场景
- 角色招募列表（酒馆）
- 物品栏浏览（战斗/酒馆）
- 法术书选择
- 存档槽位选择

### 视觉规范
```
┌─────────────────────────────────┐
│  冒险者招募                      │ ← 标题
│ ┌───────────────────────────┐   │
│ │ ▶ 卡尔·石拳  Lv3 战士    │   │ ← 选中项（浅蓝背景）
│ │   艾琳·星歌  Lv2 法师    │   │ ← 未选中项
│ │   格雷戈·铁盾 Lv1 牧师   │   │
│ │   ...                    │   │
│ └───────────────────────────┘   │
│                                  │
│  [选中角色详细信息预览区域]       │ ← 右侧/下方预览
└─────────────────────────────────┘
```

### 交互规则
| 输入 | 行为 |
|------|------|
| `↑ ↓` | 在列表项间导航 |
| `Enter` | 确认选择当前项 |
| `Tab` | 切换到详情预览面板（若存在） |
| `Esc` | 退出列表，返回上级菜单 |
| `PageUp/Down` | 翻页（列表 > 10 项时） |

### 列表项显示约定
- 每项 ≥ 36px 高度（1280×720, 24px 字体 + 12px 间距）
- 选中项：浅蓝半透明背景 + 左侧 `▶` 三角标记
- 不可选项：灰色文字 (`HintText` 色) + 不可聚焦
- 列表超过可视区域时，右侧显示滚动条（8px 宽）

## 4. 模式 3: 数值增减

### 使用场景
- 属性点分配（创建角色）
- 物品数量调整（购买/出售/使用）
- 设置滑块（音量、亮度等）

### 视觉规范
```
┌──────────────────────────────┐
│  力量:  [◀]  15  [▶]       │ ← 左右箭头 ± 文字显示
│  敏捷:  [◀]  12  [▶]       │
│  剩余点数: 3                │
└──────────────────────────────┘
```

### 交互规则
| 输入 | 行为 |
|------|------|
| `← →` | 选择属性行 |
| `↑ ↓` | 在当前属性上增减（若可行） |
| `+ / -` | 增减当前选中属性值 |
| `Enter` | 确认分配 |
| `Esc` | 取消，恢复原值 |

### 视觉反馈
- 达到上限时 `[▶]` 箭头变灰
- 达到下限时 `[◀]` 箭头变灰
- 数值变化时短暂闪烁（100ms, 白色→恢复）

## 5. 模式 4: 标签页/分区导航

### 使用场景
- 角色属性面板（属性/装备/法术/关系 4 标签）
- 设置界面（画面/声音/控制/游戏 4 标签）
- 酒馆主界面（招募/任务/商店/日志 4 标签）

### 视觉规范
```
┌─────────────────────────────────┐
│  ┌──────┬──────┬──────┬──────┐  │
│  │ ▶属性│ 装备 │ 法术 │ 关系 │  │ ← 标签栏
│  └──────┴──────┴──────┴──────┘  │
│  ┌──────────────────────────┐   │
│  │                          │   │
│  │  当前标签页内容           │   │
│  │                          │   │
│  └──────────────────────────┘   │
└─────────────────────────────────┘
```

### 交互规则
| 输入 | 行为 |
|------|------|
| `← →` | 切换标签页 |
| `Tab` | 焦点进入标签页内容区域 |
| `Shift+Tab` | 焦点回到标签栏 |
| `Esc` | 关闭面板 |

### 标签设计约定
- 选中标签：亮色背景 + 下划线（2px）
- 未选中标签：暗色背景
- 标签宽度自适应文字长度（最少 80px）
- 最多 6 个标签（超出用 `...` + 下拉）

## 6. 模式 5: 通知/Toast

### 使用场景
- 战斗结果通知（"对地精造成 8 点伤害"）
- 系统通知（"存档成功"）
- LLM 状态通知（"AI 叙述生成中..."）
- 物品获取（"获得 铁剑 ×1"）

### 视觉规范
```
┌────────────────────────────┐
│ 🗡 对地精造成 8 点伤害     │ ← 半透明暗底 + 白色文字
└────────────────────────────┘
         ↑ 屏幕上方居中，向下偏移 80px
```

### 交互规则
| 属性 | 值 |
|------|-----|
| 显示时长 | 2.5 秒（自动消失） |
| 最大同时显示 | 3 条（FIFO 队列） |
| 堆叠方向 | 从上往下，间距 8px |
| 动画 | 淡入 200ms, 淡出 300ms |
| 可中断 | 否 —— 通知为纯信息展示，不接受输入 |

### 通知类型与颜色

| 类型 | 图标 | 边框色 | 示例 |
|------|:--:|--------|------|
| 伤害 | 🗡 | `DamageRed` | "对地精造成 8 点伤害" |
| 治疗 | ✚ | `HealingGreen` | "回复 5 点生命值" |
| 获得物品 | 📦 | `Gold` | "获得 治疗药水" |
| 系统信息 | ℹ | `SecondaryText` | "存档成功" |
| 错误 | ⚠ | `DamageRed` | "连接失败，使用离线模式" |
| LLM 状态 | 🤖 | `HintText` | "AI 正在生成叙述..." |

## 7. 全局交互约定

### 焦点管理
- 每个屏幕有**明确的初始焦点**（通常是第一个可交互元素）
- `Tab` 和 `Shift+Tab` 在所有可交互元素间循环
- 焦点切换遵循**视觉阅读顺序**（从上到下，从左到右）
- 不可交互元素永不获取焦点

### 键盘快捷
| 全局快捷键 | 行为 | 适用范围 |
|-----------|------|---------|
| `Esc` | 返回上一级 / 关闭当前面板 | 全局 |
| `Enter` | 确认当前选择 | 全局 |
| `Tab` | 焦点移至下一个元素 | 全局 |
| `Shift+Tab` | 焦点移至上一个元素 | 全局 |
| `F1` | 显示当前上下文帮助 | 全局（可选） |
| `F5` | 快速存档 | 非战斗场景 |
| `F9` | 快速读档 | 非战斗场景 |

### 像素主题
所有 UI 元素共享 `PixelTheme.cs` 定义的样式。新增控件不得创建临时颜色值——必须从 `PixelTheme` 静态常量引用。

## 8. 模式 6: 工具提示 (Tooltip)

### 使用场景
- 战斗条件图标悬停 — 显示条件名称/来源/剩余轮数/效果/结束条件
- 敌人信息悬停 — AC 大致/已知抗性/威胁等级
- 物品栏悬停 — 物品名称/稀有度/属性摘要
- 装备槽悬停 — 已装备物品详情 + 背包同槽位物品对比
- 法术列表悬停 — 射程/范围/施法时间/消耗
- 技能列表悬停 — 技能描述/调整值/熟练状态
- 关系连线悬停 — 关系值 + 最近事件摘要
- 交互物体悬停 — 物体名称/交互提示（"调查"/"推开"/"阅读"）
- 属性值悬停 — 属性加值的来源分解（基础值 + 种族 + 专长 + 装备）

### 视觉规范

```
┌─────────────────────────────────────┐
│ 中毒 (Poisoned)                     │ ← 标题（14px, 白色, 1px 深色描边）
│ ─────────────────────────────────── │ ← 分隔线 (1px, SecondaryText)
│ 来源: 哥布林毒镖                    │ ← 属性行（12px, SecondaryText）
│ 剩余: 2 轮                          │
│ 效果: 攻击和技能检定具有劣势         │
│ 结束: 每轮结束 CON 豁免 DC 13       │
└─────────────────────────────────────┘
     ↑ 8px 间距
  [目标元素]
```

### 尺寸约束

| 属性 | 值 |
|------|-----|
| 最大宽度 | 280 px（设计空间 1280×720） |
| 最小宽度 | 120 px |
| 最大行数 | 8 行（超出截断 + "..."） |
| 字体 | 标题 14px / 正文 12px |
| 内边距 | 8px 水平 / 6px 垂直 |
| 背景 | 深色半透明 (rgba(10,10,30,0.92)) + 1px 浅色边框 (`SecondaryText`) |
| 阴影 | 右下 2px 偏移, rgba(0,0,0,0.6) |

### 交互规则

| 输入 | 行为 |
|------|------|
| 鼠标悬停 ≥ 300ms | 工具提示弹出（淡入 150ms） |
| 鼠标移出元素 | 工具提示关闭（淡出 100ms） |
| 键盘/手柄聚焦 ≥ 500ms | 工具提示弹出（同动画） |
| 焦点移出 / 按 `Esc` | 工具提示关闭 |
| 工具提示本身 | 不获取焦点，不可交互 |

### 定位规则（智能翻转）

```
优先级 1: 元素右侧, 垂直居中对齐（8px 间距）
优先级 2: 右侧空间不足 → 翻转到左侧
优先级 3: 上下空间不足 → 翻转到上方
优先级 4: 所有方向不足 → 居中覆盖（降低透明度至 0.85）

翻转判断基于工具提示最大宽度（280px）+ 间距（8px）vs 屏幕剩余空间
```

### 内容结构约定

- **标题**：必填。14px 白色 + 1px 深色文字描边
- **分隔线**：标题后有 1 条（可选）
- **属性行**：`标签: 值` 格式，12px `SecondaryText` 色
  - 稀有度/品质用文字标签（"稀有"）而非仅颜色
  - 数值变化用 +/- 前缀 + 颜色（`+3` 绿色, `-2` 红色），同时显示数字本身
- **最多 8 行**：超过则截断最后一行 + "…"
- **无交互元素**：工具提示内部不可放置按钮或链接

### 无障碍

- 所有以颜色传达的信息必须有文字标签双重编码（符合 Basic 等级要求）
- 工具提示文字与背景对比度 ≥ 4.5:1（标题）/ ≥ 3:1（正文）
- 触发的 300ms/500ms 延迟防止快速划过时频繁弹出
- 工具提示不遮挡正在交互的 UI 元素（智能翻转保证）
- 键盘用户可通过聚焦触发（无需鼠标悬停能力）

## 9. 模式 7: 覆盖层弹出/关闭 (Overlay Open/Close)

### 使用场景
- 酒馆内全部功能面板：招募板、任务板、角色面板、商店、对话、事件、训练、英雄之壁、酒馆升级
- 冒险中覆盖层：法术选择、物品使用、反应提示、感知检定结果
- 设置界面
- 存档/读档界面

> **区分**：本模式适用于**可自由开关的非阻塞覆盖层**。确认/取消对话框（模式 1）是模态阻塞对话框，适用不同规则。

### 视觉规范

```
┌──────────────────────────────────────────────────────────────────┐
│ 1280×720 全屏                                                      │
│                                                                    │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │ ← 遮罩层
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │   黑色 50% alpha
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓┌──────────────────────┐▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│                      │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│   覆盖层内容区域       │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │ ← 内容窗口
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│   (FF6 暗色窗框架)     │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │   不透明
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│                      │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│           [关闭 X]   │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │ ← X 按钮
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓└──────────────────────┘▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  ↑ 遮罩可点击关闭      ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
└──────────────────────────────────────────────────────────────────┘
```

### 入场/退场动画

| 覆盖层类型 | 入场动画 | 时长 | 退场动画 | 时长 |
|-----------|---------|:---:|---------|:---:|
| 内容型（招募板/商店/角色面板/对话） | `overlay_pop` — 从中心缩放弹出 | 150ms | 反向缩入 + 淡出 | 120ms |
| 底部面板（物品详情/技能说明） | `overlay_slide_up` — 从底部滑入 | 200ms | 下滑出 + 淡出 | 150ms |
| 反应提示（时间敏感） | `overlay_pop` — 快速缩放弹出 | 100ms | 反向缩入 + 淡出 | 80ms |

### 关闭机制（三重关闭）

| 关闭方式 | 行为 | 说明 |
|----------|------|------|
| `Esc` | 关闭最顶层覆盖层 | 全局约定（§7） |
| 点击关闭按钮 `[X]` | 关闭当前覆盖层 | 位于窗口右上角，24×24 px 触控区 |
| 点击遮罩区域 | 关闭最顶层覆盖层 | 点击内容窗口区域**不**触发关闭 |
| 手柄 `B` / `Circle` | 关闭最顶层覆盖层 | 等价于 `Esc` |

### 覆盖层栈规则

| 规则 | 说明 |
|------|------|
| 最大深度 | 4 层（含最低层主屏幕不算） |
| 第 5 层请求 | 自动关闭最底层覆盖层（FIFO），弹出 Toast 通知 "已达到最大窗口数量" |
| `Esc` 行为 | 逐层关闭（LIFO）— 每次关闭最顶层 |
| 场景切换冲突 | 先关闭所有覆盖层（播放退场动画 200ms），再执行场景切换动画（见 GDD §1E.3 E14） |
| 初始焦点 | 覆盖层打开后，焦点自动移到内容区域的第一个可交互元素 |

### 遮罩规范

| 属性 | 值 |
|------|-----|
| 颜色 | 黑色, alpha = 0.50 |
| 行为 | 可点击（关闭顶层覆盖层） |
| 动画 | 与覆盖层同步淡入/淡出 |
| 内容区域 | 不透明，拒绝遮罩点击事件穿透 |

### 无障碍

- 覆盖层打开时，遮罩下方的所有元素从 Tab 序中移除（焦点不逃逸到背景）
- 关闭按钮 `[X]` 有 2px 焦点高亮边框（`ButtonHoverBorder` 颜色）
- `Esc` 和手柄 `B` 均可关闭（双重键盘/手柄可达）
- 遮罩不依赖仅颜色——同时提供焦点锁定作为"无法与背景交互"的辅助信号
- 覆盖层打开/关闭动画时长在 `reducedMotion=true` 时缩短为 0ms（即时切换）

## 10. 模式 8: 长列表虚拟滚动 (Virtual Scrolling)

### 使用场景
- 物品栏（可能 > 200 件）
- 商店商品列表
- 角色招募列表（酒馆）
- 任务列表
- 法术书
- 战斗日志历史（最近 50 条滚动查看）
- 存档槽位列表
- 技能列表

> **与模式 2 的关系**：模式 2（列表选择）定义了列表项的交互规则（↑↓ 导航/Enter 选择/PageUp-Down 翻页）。本模式定义**大列表的渲染优化**。两者配合使用：模式 10 负责"渲染哪些行"，模式 2 负责"如何与这些行交互"。

### 启用阈值

| 条件 | 行为 |
|------|------|
| ≤ 50 项 | 全量渲染（标准列表，模式 2） |
| > 50 项 | 启用虚拟化滚动（本模式接管渲染） |

### 渲染窗口

```
            ┌─────────────────────────┐
  buffer    │  第 17 项 (不可见)       │ ← 上方缓冲 2 行
  above     │  第 18 项 (不可见)       │
            ├─────────────────────────┤
            │  第 19 项 (可见)         │ ← 视口顶部
            │  第 20 项 (可见)         │
            │  第 21 项 (可见) ← 选中 │
            │  第 22 项 (可见)         │
            │  ...                    │
            │  第 38 项 (可见)         │ ← 视口底部
            ├─────────────────────────┤
  buffer    │  第 39 项 (不可见)       │ ← 下方缓冲 2 行
  below     │  第 40 项 (不可见)       │
            └─────────────────────────┘

  视口高度: 根据容器动态计算
  缓冲行数: 可见行 ± 2 行（上下各 2 行）
  项高度: ≥ 36px（与模式 2 一致）
```

### 滚动条规范

| 属性 | 值 |
|------|-----|
| 宽度 | 8px |
| 位置 | 列表右侧，与容器边缘间距 2px |
| 滑块位置 | `scrollOffset / totalItemCount`（基于条目数比例） |
| 滑块高度 | `max(12px, visibleCount / totalItemCount × 轨迹高度)` — 最小 12px 保证可点击 |
| 轨迹颜色 | 深灰半透明 (`rgba(30,30,50,0.6)`) |
| 滑块颜色 | `SecondaryText` 色（半透明） |
| 显示时机 | 列表聚焦或鼠标悬停时显示；空闲 2s 后淡出（可选） |

### 交互规则（补充模式 2）

| 输入 | 行为 |
|------|------|
| `↑ ↓` | 逐项导航。到达视口边缘时自动滚动（平滑滚动，非跳转） |
| `PageUp / PageDown` | 翻一页（可见行数 - 1）。边界页不足一页时显示剩余项 |
| `Home` | 跳至第一项 |
| `End` | 跳至最后一项 |
| 鼠标滚轮 | 逐项滚动（`ScrollDelta` > 阈值时加速） |
| 拖拽滚动条滑块 | 即时跳转到对应位置（基于条目数比例计算目标 `scrollOffset`） |
| 点击滚动条轨迹（滑块上方/下方） | 翻一页 |

### 性能约束

| 指标 | 目标 |
|------|------|
| 渲染行数上限 | ≤ 30 行（含缓冲，1280×720 下约 20 可见 + 4 缓冲 = 24 行） |
| 初始加载 | 虚拟化窗口内项目的纹理/数据懒加载，非一次性加载全部 |
| 滚动帧率 | 保持 60fps（非虚拟化列表 > 200 项时 Myra 可能掉帧） |
| 回收复用 | 行 Widget 对象池复用 —— 离开视口的行 Widget 回收后重新绑定数据给进入视口的行 |

### 无障碍

- 虚拟化对键盘导航透明：`↑ ↓` 和 `PageUp/Down` 行为与全量列表一致
- 屏幕阅读器（如未来支持）看到的是"列表中有 N 项"而非"列表中有 24 项"
- 滚动条有足够对比度（`SecondaryText` vs 深灰轨迹）—— ≥ 3:1 最小对比度

## 11. 模式 9: 进度条/状态条 (Progress/Status Bar)

### 使用场景
- HP 条 — 角色当前生命值占最大值的比例
- 法术位条 — 分段显示已用/剩余法术位
- 耐久条 — 装备当前耐久占最大耐久的比例
- 专注持续条 — 当前持续轮数 / 总轮数
- 加载进度条 — 资源加载 / 场景切换进度（0-100%）
- 酒馆声望条 — 当前声望值 / 升级所需声望
- XP 条 — 当前经验值 / 升级所需经验值
- 充能/冷却条 — 能力冷却的剩余比例

### 通用规范

#### 尺寸与布局

```
┌──────────────────────────────────────────┐
│ HP  ████████████████░░░░░░░░  28 / 28   │ ← 标签(左) + 填充条 + 数值(右)
└──────────────────────────────────────────┘
     ↑ 8px 标签      ↑ 填充区域     ↑ 数值文本
```

| 属性 | 值 |
|------|-----|
| 高度 | 12–16 px（设计空间 1280×720） |
| 最小宽度 | 80 px |
| 标签 | 左侧，12px 字体，距填充条 4px |
| 数值 | 右侧，12px 等宽字体，距填充条 4px |
| 填充方向 | 从左到右（水平） |
| 背景（空） | 深灰 `rgba(20,20,40,0.8)` |
| 边框 | 1px 深色 (`#1a1a2e`)，无圆角（像素风格） |
| 内边距 | 填充条边缘距边框 1px |

#### 颜色映射

> 通用颜色由各屏幕 UX 规格定义。以下为推荐默认值（战斗 HP 条遵循 hud.md §5）：

| 填充比例 | 推荐颜色 | 使用场景示例 |
|:--------:|---------|-------------|
| > 66% | 绿色 `#44FF44` | HP 健康 / 耐久良好 |
| 33–66% | 黄色 `#FFFF44` | HP 受伤 / 耐久警告 |
| < 33% | 红色 `#FF4444` | HP 濒危 / 耐久危险 |
| = 0 | 灰色 `#666666`（无填充） | HP 倒地 / 耐久耗尽 |
| 加载中 | 金色 `#FFD700` | 进度条不确定模式 |
| 法术/魔法 | 蓝色 `#4488FF` | 法术位 / 魔力 |

#### 动画

| 动画 | 规范 |
|------|------|
| 数值变化 | 平滑过渡（`lerp` over 300ms）—— 填充宽度从旧值平滑变化到新值。治疗/伤害时先跳变再平滑会更好（但跳变→平滑已够用） |
| 低值警告 | HP < 33% 时填充条闪烁（500ms 亮/暗周期），持续至 HP 恢复至 ≥ 33% |
| 填充完成 | 到达 100% 时短暂亮度脉冲（100ms），然后稳定 |
| 不确定模式 | 加载进度不确定时，使用往复滑动光带（200px 宽光带在轨道上来回滑动，周期 1.5s） |

#### 交互

进度条/状态条为**纯信息展示元素**——不获取焦点，不接受输入。悬停时显示工具提示（模式 8）展示精确数值和来源分解。

### 特殊变体

#### 分段条（法术位）

```
1环 [██][██][██][░░]  3/4
      ↑ 已用        ↑ 剩余
```

- 每段宽度固定（≥ 16px），段间距 2px
- 已用段：对应颜色填充；剩余段：深灰背景
- 段的含义由上方/左侧标签说明

#### 先攻顺序条

> 见 `design/ux/hud.md` §5 —— 横排角色头像 + 先攻值 + 颜色编码（友方蓝框 / 敌方红框）。不重复在此定义。

### 无障碍

- 数值始终显示在条右侧（Basic 等级要求：不依赖颜色传达信息）
- 低值闪烁的视觉警告同时触发屏幕边缘微红 overlay（额外信号通道）
- 颜色映射始终配合数值显示——色盲玩家可通过数字判断状态
- filling 动画在 `reducedMotion=true` 时禁用平滑过渡（直接跳变）

## 12. 模式 10: 倒计时指示器 (Countdown Indicator)

### 使用场景
- 反应提示计时 — "借机攻击？倒计时 15 秒"（09-ui-ux §4.4）
- 条件剩余轮数 — 中毒/恐慌/加速等条件的持续倒计时
- 专注法术剩余轮数 — "Haste 持续: 8/10 轮"
- 限时冒险节点 — "剩余 3 回合完成目标"
- 环境效果倒计时 — "火墙持续: 2 轮"

### 与模式 11 的关系

倒计时指示器是**进度条的时间变体**。底层继承模式 11 的通用填充条规范（尺寸/边框/布局标签），叠加以下倒计时特有行为。

### 视觉规范

```
┌──────────────────────────────────────────┐
│ ⚡ 反应  ████████░░░░░░░░  12s          │ ← 标签 + 填充条 + 剩余秒数
└──────────────────────────────────────────┘
  正常时: 白色填充      紧迫时: 红色+闪烁
```

| 属性 | 值 |
|------|-----|
| 高度 | 14 px（略厚于普通进度条，加强紧迫感） |
| 标签 | 左侧图标 + 简短文字（如 "⚡ 反应"） |
| 数值 | 右侧 `剩余秒数 + s` 格式（如 `12s`） |
| 填充颜色（正常） | 白色 `#EEEEEE` |
| 填充颜色（紧迫） | 红色 `#FF4444`，触发条件：剩余 ≤ 30% 总时长 |
| 填充颜色（到期） | 红色闪烁 3 次（300ms/次），填充降为 0，然后条消失 |

### 行为

| 阶段 | 剩余比例 | 视觉行为 | 音频 |
|------|:-------:|---------|------|
| 正常 | > 30% | 平滑递减填充（每 100ms 更新） | 无 |
| 紧迫 | ≤ 30% | 填充条变红 + 闪烁（500ms 周期） | ≤ 5s 时每秒滴答声（短促） |
| 到期 | = 0 | 红色闪烁 3 次（300ms/次）→ 触发到期事件 → 条消失 | 警示音（低音锣声） |
| 被取消 | — | 条 fade out 200ms | 无 |

### 交互

| 输入 | 行为 |
|------|------|
| 倒计时条本身 | 不获取焦点，不响应输入 |
| 玩家做出决策（如反应提示中选择"借机攻击"） | 取消倒计时，条 fade out |
| 超时 | 触发默认行为（如反应提示超时 = 放弃反应） |

### 倒计时数值精度

| 总时长 | 显示精度 | 更新频率 |
|:------:|:-------:|:-------:|
| ≤ 10s | 1s 精度（显示 `5s`） | 每秒更新 |
| ≤ 60s | 1s 精度（显示 `45s`） | 每秒更新 |
| > 60s | 轮数精度（显示 `8/10 轮`） | 每轮更新 |

### 无障碍

- 倒计时数值始终显示（不依赖颜色和填充比例传达紧迫性）
- 紧迫阶段的闪烁频率 ≤ 2Hz（500ms 周期），符合光敏性安全阈值
- 到期音效有对应的屏幕闪烁视觉信号
- `reducedMotion=true` 时闪烁禁用（填充条保持红色常亮，不闪烁）
- 键盘用户可通过 `Esc` 取消部分倒计时（如反应提示）——等同于"放弃"

## 13. 模式 11: 拖放 — 装备 (Drag & Drop for Equipment)

### 使用场景
- 物品栏 → 装备槽：装备物品
- 装备槽 → 物品栏：卸下物品
- 装备槽 ↔ 装备槽：交换同槽位物品（如主手/副手互换）
- 物品栏 → 丢弃区域：丢弃物品（触发确认对话框，模式 1）
- 商店：物品栏物品拖放到商店出售区域

> **与模式 1 的关系**：涉及不可逆操作（丢弃、出售绑定物品）时，拖放释放后弹出模式 1（确认对话框）。拖放本身不执行不可逆操作。

### 设计原则

**双轨操作**：鼠标用户通过拖放操作（直觉快），键盘/手柄用户通过菜单操作（模式 2 列表选择 + Enter 弹出操作菜单）。两条路径等价，无功能差异。

### 拖放视觉流程

```
┌─ 阶段 1: 抓起 ──────────────────────────────────────────┐
│                                                          │
│  鼠标按下 (在背包物品图标上, 按住 ≥ 100ms)                │
│    → 物品图标略微放大 (scale 1.0 → 1.1, 100ms 缓出)     │
│    → 光标切换为抓取手型 (grab)                           │
│    → 播放轻微拾取音效 (短促咔嗒声)                       │
│                                                          │
├─ 阶段 2: 拖动 ──────────────────────────────────────────┤
│                                                          │
│  鼠标移动 (按住不放)                                     │
│    → 半透明物品图标跟随光标 (alpha 0.8, 偏移 +8px 右下方)│
│    → 有效目标槽位高亮                                   │
│        · 正常可放置: 绿色虚线边框 2px, 脉冲 800ms       │
│        · 同槽位已有物品: 黄色虚线边框 (即将交换)        │
│        · 非法目标: 槽位短暂红色闪烁 100ms               │
│    → 非法区域 (装备面板外): 物品图标 alpha 降至 0.5     │
│    → 光标切换为拖动中手型 (grabbing)                     │
│                                                          │
├─ 阶段 3: 释放 ──────────────────────────────────────────┤
│                                                          │
│  释放鼠标 (有效目标上)                                   │
│    → 物品图标缩放入槽位 (200ms 缓入动画)                │
│    → 如为交换: 被替换物品图标从槽位弹出 → 飞回背包      │
│        原位置 (300ms 弧线动画)                           │
│    → 播放装备/交换音效 (金属咔嗒声)                     │
│    → 属性面板即时刷新                                    │
│                                                          │
│  释放鼠标 (无效位置)                                     │
│    → 物品图标弹回原位 (300ms 缓出, overshoot 5%)        │
│    → 播放轻微否定音效 (低沉短促)                        │
│                                                          │
│  释放鼠标 (背包区域外 / 空白处)                          │
│    → 同"无效位置"——物品弹回原位                         │
│    → 不触发丢弃 (丢弃需拖到专用丢弃区域)               │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 槽位约束

| 槽位 | 接受物品类型 | 容量 | 非法拖入反馈 |
|------|------------|:---:|-------------|
| 头盔 | 头盔类护甲 | 1 | 槽位红色闪烁 100ms + 物品弹回 |
| 护甲 | 护甲（轻甲/中甲/重甲） | 1 | 同上 |
| 主手 | 武器 / 盾牌 / 法器 | 1 | 同上 |
| 副手 | 盾牌 / 轻武器 / 法器 | 1 | 同上 |
| 戒指 1 | 戒指 | 1 | 同上 |
| 戒指 2 | 戒指 | 1 | 同上 |
| 护符 | 护符 / 圣徽 | 1 | 同上 |
| 靴子 | 靴子类 | 1 | 同上 |

> **熟练项约束**：如角色不熟练该武器/护甲类型，槽位显示黄色警告（可装备但有惩罚）。程序通过 Tooltip（模式 8）显示惩罚详情。设计决策：允许玩家强行装备不熟练物品（DND 5e 允许但施加惩罚），UI 用黄色而非红色警告。

### 键盘/手柄替代方案

| 操作 | 键盘/手柄等效 |
|------|-------------|
| 装备物品 | 在背包中选中物品 → `Enter` → 弹出操作菜单选择"装备" |
| 卸下物品 | 在装备槽上按 `Enter` → 弹出操作菜单选择"卸下" |
| 交换物品 | 在装备槽上按 `Enter` → 选择"交换" → 从背包列表中选替代品 |
| 丢弃物品 | 在背包中选中物品 → `Enter` → "丢弃" → 触发模式 1 确认对话框 |

操作菜单复用模式 2（列表选择）交互规则：`↑↓` 导航选项，`Enter` 确认，`Esc` 取消。

### 无障碍

- 拖放的所有功能均有键盘等效操作（Basic 等级要求）
- 有效/无效目标通过形状（虚线边框）和颜色（绿/黄/红）双重编码
- 拖放操作在 `reducedMotion=true` 时动画时长缩短 50%，不跳过——动画提供空间关系线索
- 手柄用户：装备界面默认焦点在第一个装备槽，`← →` 切换槽位，`↑ ↓` 在槽位和背包列表间切换

## 14. 模式 12: 高亮/选中态 (Highlight / Selection State)

### 使用场景

本模式为**跨所有模式的元模式**——定义 UI 元素在悬停/聚焦/选中三种状态下的通用视觉反馈。所有其他模式引用本模式的状态定义，而非各自重复定义。

### 三层高亮体系

```
L1 悬停                     L2 聚焦                      L3 选中/激活
(Hover)                    (Focused)                    (Selected/Active)
    ↓                          ↓                              ↓
亮度 +10%              2px 金色边框 + 亮度 +5%        浅蓝背景 + ▶ 三角标记
1px 浅色边框            (ButtonHoverBorder)           (rgba(64,128,255,0.25))
(鼠标悬停)              (键盘Tab/手柄导航)            (列表项/标签页/切换态)
```

| 层级 | 状态 | 触发 | 视觉规范 | 适用元素 |
|:---:|------|------|---------|---------|
| **L1** | 悬停 (Hover) | 鼠标指针进入元素边界 | 亮度 `+10%`（RGB 各通道 × 1.10）；1px 浅色边框（`SecondaryText` 色, alpha 0.6） | 所有可交互元素 |
| **L2** | 聚焦 (Focused) | 键盘 `Tab` / 手柄方向键导航 | 2px 金色边框（`ButtonHoverBorder`, `#FFD700`）；亮度 `+5%` | 所有可聚焦元素 |
| **L3** | 选中/激活 (Selected/Active) | 点击/`Enter` 确认 / 状态切换 | 浅蓝半透明背景 `rgba(64,128,255,0.25)`；列表项左侧 `▶` 三角标记（8px, `SecondaryText` 色）；标签页底部 2px 下划线（同金色） | 列表项/标签页/开关按钮 |

### 层级关系

- **互不冲突**：同一元素可同时处于多个层级（如 L1+L2 = 鼠标悬停在已聚焦元素上）
- **L3 优先于视觉效果**：元素处于 L3 时，L1 的浅色边框被 L3 的背景覆盖（不显示双重边框）。仅亮度叠加仍然生效
- **L2 始终可见**：聚焦边框在 L3 状态下仍然显示（确保键盘用户始终看到焦点位置）

### 叠加示例

```
元素状态         视觉效果
─────────────────────────────────────────────
无状态           默认外观
L1 (悬停)        亮度 +10% + 1px 浅灰边框
L2 (聚焦)        2px 金色边框 + 亮度 +5%
L3 (选中)        浅蓝背景 + ▶ 标记
L1+L2            2px 金色边框 + 亮度 +15% (10%+5%)
L2+L3            浅蓝背景 + ▶ + 2px 金色边框
L1+L3 (罕见)     浅蓝背景 + ▶ + 亮度 +10% (L1 边框被 L3 隐藏)
L1+L2+L3         浅蓝背景 + ▶ + 2px 金色边框 + 亮度 +15%
```

### 过渡动画

| 过渡 | 时长 | 缓动 |
|------|:---:|------|
| 悬停进入 (none → L1) | 100ms | 线性 |
| 悬停退出 (L1 → none) | 80ms | 线性 |
| 聚焦进入 (none → L2) | 0ms（即时） | — |
| 聚焦切换 (元素A L2 → 元素B L2) | 0ms（即时） | — |
| 选中切换 (none → L3) | 120ms | 缓出 |

> **聚焦即时切换的原因**：键盘导航时，延迟的焦点动画会造成"跟不上"的感觉，降低输入置信度。

### 特殊元素的高亮规则

| 元素 | 高亮方式 | 说明 |
|------|---------|------|
| 按钮（模式 1） | L1: 亮度 +10%；L2: 2px 金色边框 | 无 L3 —— 按钮无"选中"态，只有"按下"态 |
| 标签页（模式 4） | L3: 亮色背景 + 底部 2px 下划线（替代 ▶ 三角） | 标签页不使用 ▶ 标记 |
| 工具提示（模式 8） | 不接受任何高亮 | 提示不可交互，无焦点 |
| 覆盖层遮罩（模式 9） | L1: 遮罩整体微亮（alpha 降低 3%）作为"可点击关闭"提示 | 遮罩整体可点击 |
| 进度条/倒计时（模式 11/12） | 不接受高亮 | 纯信息元素 |

### 无障碍

- 聚焦边框（L2）颜色金色 `#FFD700` vs 深色背景对比度 ≥ 3:1
- 悬停（L1）和聚焦（L2）不依赖颜色区分——边框宽度不同（1px vs 2px）
- 选中态（L3）不依赖颜色区分——`▶` 三角标记提供形状信号
- 聚焦切换 0ms 延迟确保键盘导航响应即时
