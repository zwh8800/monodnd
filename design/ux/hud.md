# 战斗 HUD 信息层级规格

> **状态**: 草稿 (Draft) — 启动于 Technical Setup → Pre-Production 门控检查
> **版本**: v0.1
> **关联文档**: `design/gdd/09-ui-ux-design.md`, `design/art/art-bible.md` §3.0（正交投影）, §7（像素主题）, `src/DndGame/UI/PixelTheme.cs`
> **支柱对应**: P2（战术 DND 深度 — "清晰即是沉浸"）

---

## 1. 设计目标

战斗 HUD 的核心使命：让玩家在 0.3 秒内理解战术态势，在 2 秒内做出有意义决策。

- **信息可读性优先于装饰性** —— 艺术圣经原则 "清晰即是沉浸"
- **回合制结构允许信息分层展示** —— 不急于在一帧内展示全部
- **键盘可达** —— 无障碍 Basic 级要求

## 3. 信息层级

### 第一眼（0.3 秒内 —— 态势认知）

| 视觉信号 | 传达信息 | 实现方式 |
|----------|---------|---------|
| 当前行动角色高亮 | "轮到谁了" | 精灵周围闪烁边框（2px, 金色, 周期 500ms） |
| 敌方威胁标记 | "谁最危险" | 敌方格上方的威胁等级图标（▲/▲▲/▲▲▲） |
| 当前回合阶段提示 | "该做什么" | 顶部横幅 "选择行动" / "确认目标" / "敌人回合" |
| 角色生命状态 | "队友还活着吗" | HP 条颜色渐变（绿→黄→红）+ 倒地图标 |

### 第二眼（0.5–2 秒 —— 决策信息）

| 信息 | 位置 | 格式 |
|------|------|------|
| 行动菜单（6 选项） | 右侧面板 | 垂直列表，当前选中项高亮（浅蓝背景） |
| 可选目标（若已选行动） | 战斗区域 | 可攻击敌人闪烁红框，可治疗队友闪烁绿框 |
| 命中率预览（若已选目标） | 行动菜单下方 | "命中: 65% (需 ≥14)" |
| 可用资源 | 行动菜单各选项旁 | "🎯 攻击 (1/1)" "✨ 法术 (2/3)" |

### 第三眼（需要时 —— 深度信息）

| 信息 | 触发方式 | 格式 |
|------|---------|------|
| 角色详细面板 | 选中角色 → 按 Tab | 浮动窗口：全属性 + 装备 + 条件 |
| 敌人信息 | 悬停/选中敌人 → 按 Tab | 浮动窗口：AC + HP 大致 + 已知抗性 |
| 战斗日志历史 | 日志区域 ↑↓ 滚动 | 滚动列表（最近 50 条） |
| 法术详情 | 法术菜单中 → 按 F1 | 浮动提示：射程/范围/效果/消耗 |

## 4. 行动选择交互流（P2 核心流）

```
                         ┌──────────────┐
                         │  轮到玩家行动  │
                         └──────┬───────┘
                                │
                                ▼
                    ┌──────────────────────┐
                    │  Step 1: 选择行动类型  │
                    │  ↑↓ 导航菜单          │
                    │  Enter 确认           │
                    └──────┬───────────────┘
                            │
                ┌───────────┼───────────┐
                ▼           ▼           ▼
          ┌─────────┐ ┌─────────┐ ┌─────────┐
          │ 攻击    │ │ 法术    │ │ 其他    │
          │ 显示目标│ │ 显示法术│ │ 对应UI  │
          └────┬────┘ └────┬────┘ └────┬────┘
               │           │           │
               ▼           ▼           ▼
          ┌──────────────────────────────────┐
          │  Step 2: 选择目标/确认参数        │
          │  ← → 切换目标                    │
          │  Tab  查看目标详细信息           │
          │  Enter 确认执行                  │
          │  Esc   返回 Step 1              │
          └──────────────┬───────────────────┘
                         │
                         ▼
          ┌──────────────────────────────────┐
          │  Step 3: 确认执行                 │
          │  显示命中率 + 预期伤害/效果       │
          │  "确认对 [地精] 使用 [近战攻击]?" │
          │  Enter 执行  Esc 取消            │
          └──────────────────────────────────┘
```

### 关键交互规则

1. **每个 Step 都可撤销** —— Esc 随时回到上一步
2. **信息逐步揭示** —— Step 1 只显示行动类型，Step 2 才显示目标范围和命中率
3. **确认前显示预期结果** —— Step 3 必须显示命中率和伤害范围
4. **键盘全程可达** —— ↑↓←→ 导航，Enter 确认，Esc 撤销，Tab 详情

## 5. 状态显示约定

### HP 条颜色阶梯

| HP 比例 | 颜色 | 含义 |
|:-------:|------|------|
| > 66% | 绿色 `#44FF44` | 健康 |
| 33–66% | 黄色 `#FFFF44` | 受伤 |
| < 33% | 红色 `#FF4444` | 濒危 |
| = 0 | 灰色 + ☠ 图标 | 倒地 |

### 条件/状态图标

| 条件 | 图标 | 位置 | 格式 |
|------|------|------|------|
| 中毒 | ☠ (绿) | HP 条左侧 | 2px 像素图标 |
| 眩晕 | ◎ (黄) | HP 条左侧 | 闪烁 800ms |
| 魅惑 | ♥ (紫) | HP 条左侧 | — |
| 其他 | 按条件类型 | — | 8×8px 图标 |

### 先攻顺序条

- 顶部横向排列：当前行动角色在最左 + 高亮
- 角色头像 + 名字缩写（8px 字体）
- 敌人标记为红色边框，友方为蓝色边框

## 8. 动画 Timing 规格

> **像素美术约束**：不支持复杂渐变/缩放/平滑缓动。动画使用逐帧精灵切换、即时状态切换、或阶梯式位移（固定像素步长每帧推进）。目标帧率 60fps，单帧 ≈ 16.67ms。

| # | 元素 | 动画类型 | 持续时间 | 缓动 | 初始状态 → 最终状态 | 备注 |
|---|------|---------|:-------:|------|---------------------|------|
| 1 | 行动菜单滑入 | 阶梯平移 | 300ms | 阶梯 16px/帧 | x=1280(屏右外) → x≈1000 | 回合开始时触发；菜单宽约 280px，18 帧完成 |
| 2 | 行动菜单滑出 | 阶梯平移 | 200ms | 阶梯 16px/帧 | x≈1000 → x=1280(屏右外) | 行动确认后触发；末尾超出屏幕部分即时隐藏 |
| 3 | 浮动伤害数字 | 平移 + 淡出 | 800ms 合计 | 线性上升 + 阶梯透明度 | 目标顶部 y₀ → y₀−32px，同时 α: 100% → 0% | 上升 500ms → 保持 100ms → 淡出 200ms；红色 `#FF4444`；前缀 `−`；含小幅度水平随机偏移 ±4px |
| 4 | 浮动治疗数字 | 平移 + 淡出 | 800ms 合计 | 线性上升 + 阶梯透明度 | 目标顶部 y₀ → y₀−32px，同时 α: 100% → 0% | 与伤害数字相同方向；绿色 `#44FF44`；前缀 `+` |
| 5 | HP 条减少（排水） | 填充变化 | 300ms | 线性 1px/帧 | 旧 HP 填充宽度 → 新 HP 填充宽度 | 从当前值逐帧缩减至目标值；若变化量 < 18px 则最小步长 1px/帧 |
| 6 | HP 条增加（充水） | 填充变化 | 500ms | 线性 1px/帧 | 旧 HP 填充宽度 → 新 HP 填充宽度 | 略慢于排水，增强恢复满足感；最小步长 1px/帧 |
| 7 | MP 条变化 | 填充变化 | 同 HP：减 300ms / 增 500ms | 线性 1px/帧 | 旧 MP 填充宽度 → 新 MP 填充宽度 | 与 HP 条完全一致的 timing |
| 8 | 状态图标闪烁 | 即时开关 | 800ms 周期 | 即时 | 可见 ↔ 不可见 | 占空比 50%（400ms 开 / 400ms 关）；中毒 ☠ / 眩晕 ◎ / 魅惑 ♥ 及其他所有条件统一使用此周期 |
| 9 | 回合切换横幅 | 淡入 + 淡出 | 300ms(入) + 200ms(出) | 阶梯透明度 3 级 | 透明(α=0%) → 不透明(α=100%) → 透明(α=0%) | 透明度级数：0% → 50% → 100%，每级 ~100ms；显示期间保持 100% |
| 10 | 确认对话框弹出 | 即时出现 + 逐帧边框 | 200ms(出) / 150ms(消) | 阶梯 | 不可见 → 完整显示 | 出现：背景遮罩即时显示 + 对话框边框逐帧（4 帧）从四角向中心绘制；消失：整体即时隐藏 |
| 11 | 当前行动角色高亮 | 即时闪烁 | 500ms 周期 | 即时 | 金色边框(2px, `#FFD700`) 可见 ↔ 不可见 | 占空比 50%（250ms 开 / 250ms 关）；§3 已定义，此处确认 |
| 12 | 目标选择高亮 | 即时闪烁 | 500ms 周期 | 即时 | 红框(敌方)/绿框(友方)(2px) 可见 ↔ 不可见 | 占空比 50%；与角色行动高亮（#11）保持一致周期 |
| 13 | 骰子动画（d20 判定） | 逐帧精灵 | 1650ms 合计 | 逐帧 4 帧循环 | 不可见 → 滚动 → 定格结果 → 消失 | 详见下方骰子动画分镜 |

### 骰子动画分镜

```
阶段 1: 骰子出现    150ms    骰子精灵在战斗区域中央淡入（阶梯透明度 0%→100%，3 级）
                            ┌──────────┐
                            │   ⚀?     │  ← d20 像素精灵约 40×40px
                            └──────────┘

阶段 2: 骰子滚动    800ms    4 帧精灵循环，每帧 200ms
         帧 1 (200ms)       d20 旋转姿态 A（像素精灵 1/4）
         帧 2 (200ms)       d20 旋转姿态 B（像素精灵 2/4）
         帧 3 (200ms)       d20 旋转姿态 C（像素精灵 3/4）
         帧 4 (200ms)       d20 旋转姿态 D（像素精灵 4/4）
         ±2px 随机抖动增强动感

阶段 3: 结果定格    500ms    骰子停止 → 判定结果数字显示在骰子上方
                            ┌──────────┐
                            │   "17"    │  ← 结果数字（20px 金色 #FFD700）
                            │   ⚄      │  ← 骰子精灵最终帧
                            └──────────┘

阶段 4: 骰子消失    200ms    阶梯淡出（α: 100%→50%→0%，每级 ~67ms）
```

| 参数 | 值 |
|------|:--:|
| 骰子精灵尺寸 | 40×40px（可调至 48×48px） |
| 逐帧数量 | 4 帧（4 种 d20 旋转姿态） |
| 单帧持续时间 | 200ms |
| 总持续时间 | 1650ms（≈1.7 秒） |
| 结果数字字体 | 20px 像素字体，金色 `#FFD700` |
| 抖动幅度 | ±2px 随机偏移 |
| 触发时机 | Step 3 确认执行后，攻击/法术命中判定时 |

## 9. 音效触发点

> **优先级规则**：高优先级可压低低优先级音量（ducking），同优先级叠加。HP 警告循环持续期间，其他 UI 音效默认 -6dB。

| # | 触发条件 | 音效类别 | 优先级 | 音效描述 | 备注 |
|---|---------|:------:|:------:|---------|------|
| 1 | 行动菜单选项移动 (↑↓) | UI | 低 | 短促滴答声，~50ms，方波/锯齿波 | 每个选项不同音高（相邻选项 ±1 半音），增强菜单定位感 |
| 2 | 行动确认 (Enter) | UI | 中 | 确认音，~150ms，上升双音 | 先低后高，二度上行 |
| 3 | 行动取消 (Esc) | UI | 低 | 取消音，~100ms，下降单音 | 低沉短促，区别于确认音 |
| 4 | 目标切换 (←→ / LB/RB) | UI | 低 | 同 #1 滴答声，~50ms | 音高固定，区别于菜单选项音高变化 |
| 5 | 攻击命中 | 战斗 | 高 | 命中音，~200ms | 根据武器类型变化：斩击(金属撞击)、钝击(沉重闷响)、穿刺(尖锐短促)、魔法(共鸣嗡鸣) |
| 6 | 攻击未命中 | 战斗 | 中 | 挥空音，~150ms，风声/呼啸 | 轻量，避免与命中音混淆 |
| 7 | 暴击 | 战斗 | 高 | 强击音，~300ms，多重叠加 | 命中音基础 + 额外低频冲击 + 短暂混响；明显区别于普通命中 |
| 8 | 伤害数字弹出 | UI | 中 | 短促冲击音，~50ms | 与伤害数字同时触发；低音短脉冲 |
| 9 | 治疗数字弹出 | UI | 中 | 治愈音，~100ms，柔和上升 | 明亮音色，与伤害冲击音形成对比 |
| 10 | HP < 33% 警告 | UI | 高 | 警告提示音 + 心跳循环 | 首次触发：短警报音 ~200ms；随后心跳循环（~70BPM，约 857ms 周期）；仅在玩家回合播放 |
| 11 | 角色倒地 | 战斗 | 高 | 倒地音，~400ms，沉重下坠 | 含低音冲击 + 短混响；区别于命中/暴击 |
| 12 | 回合切换（我方→敌方） | UI | 中 | 过渡音，~200ms，下行音阶 | 暗示"交出控制权"；小调下行 |
| 13 | 回合切换（敌方→我方） | UI | 中 | 过渡音，~200ms，上行音阶 | 暗示"获得控制权"；大调上行；与 #12 形成对称 |
| 14 | 战斗胜利 | UI | 高 | 胜利号角，~2000ms | 大调上行音阶 + 终止式；可含简短旋律片段 |
| 15 | 战斗失败 | UI | 高 | 失败音，~2000ms | 小调下行 + 不协和终止；低沉持续音 |

## 10. 无障碍规格

> **当前目标等级**：Basic（已承诺）。Standard 层接口预留，待资源允许时实施。

### 10.1 Basic 层（当前承诺——必须实施）

#### 色彩审计表

对 HUD 中每个颜色信号，列出非颜色替代方案：

| # | 颜色信号位置 | 颜色 | 传达信息 | 非颜色替代方案 |
|---|-------------|------|---------|---------------|
| 1 | HP 条 > 66% | 绿色 `#44FF44` | 健康 | 条内/旁同时显示数值（例: `HP 45/50`） |
| 2 | HP 条 33–66% | 黄色 `#FFFF44` | 受伤 | 数值 + 条内文字标签 `受伤`（8px 叠加） |
| 3 | HP 条 < 33% | 红色 `#FF4444` | 濒危 | 数值 + 条闪烁（2Hz）+ 心跳音效（§9 #10） |
| 4 | HP = 0 | 灰色 `#888888` | 倒地 | ☠ 图标（已有 §5）+ 文字 `倒地` 显示在角色名旁 |
| 5 | 敌方威胁 ▲ | 红色 `#FF4444` | 高威胁 | 三角形数量（▲/▲▲/▲▲▲）——已满足非颜色区分 |
| 6 | 可选攻击目标边框 | 红色闪烁 `#FF4444` | 可攻击 | 边框样式差异化：攻击目标 = 虚线边框，治疗目标 = 点线边框 + `+` 图标 |
| 7 | 可选治疗目标边框 | 绿色闪烁 `#44FF44` | 可治疗 | 同 #6：点线边框 + `+` 图标，区别于攻击目标虚线边框 |
| 8 | 中毒图标 ☠ | 绿色 `#44FF44` | 中毒状态 | ☠ 图标本身已满足（图标 > 颜色） |

#### 最小字体检查

| 元素 | 当前设计 | 要求 | 状态 |
|------|:-------:|:----:|:----:|
| 行动菜单文字 | 20px（估算） | ≥ 20px | ✓ |
| 队伍状态栏 HP/MP 数值 | 20px | ≥ 20px | ✓ |
| 战斗日志文字 | 16px（估算） | ≥ 16px（三级信息可略小） | ✓ |
| 条件图标标签（眩晕/魅惑） | 16px | ≥ 16px | ✓ |

#### 光敏性风险标注

| 元素 | 频率 | 面积占比 | 风险评级 | 说明 |
|------|:----:|:-------:|:-------:|------|
| HP < 33% 闪烁 | 2Hz (500ms 周期) | < 5% 屏幕 | **低** | 仅 HP 条区域闪烁 |
| 状态图标闪烁 | 1.25Hz (800ms 周期) | < 1% 屏幕 | **极低** | 单个 8×8px 图标 |
| 目标选择边框闪烁 | 2Hz (500ms 周期) | < 3% 屏幕 | **低** | 单目标 2px 边框 |
| 骰子动画 | 5Hz（逐帧切换） | < 2% 屏幕 | **低** | 逐帧精灵切换，非明暗交替闪烁；40×40px 区域 |
| 回合切换横幅淡入淡出 | 阶梯透明度（~3Hz 变化） | < 10% 屏幕 | **低** | 非闪烁，为透明度渐变 |

> **总体评估**：战斗中无高频（>3Hz）闪烁、无大面积（>25% 屏幕）闪烁。所有动画均通过光敏性安全检查。✓

## 11. Gamepad 导航顺序

### 导航状态机

```
                          ┌──────────────┐
                          │   战斗空闲    │
                          └──────┬───────┘
                                 │ [A]
                                 ▼
                     ┌──────────────────────┐
                     │     行动菜单          │
                     │  (焦点: 第一可用行动)  │
                     └────┬───────┬─────────┘
               [↑↓]       │       │ [B]
              切换选项     │ [A]   └──→ 返回战斗空闲
                           ▼
                     ┌──────────────────────┐
                     │     目标选择          │
                     │  (焦点: 最近可选目标)  │
                     └────┬───────┬─────────┘
        [←→/LB/RB]       │       │ [B]
        切换目标          │ [A]   └──→ 返回行动菜单
                           ▼
                     ┌──────────────────────┐
                     │     确认弹窗          │
                     │  (焦点: "确认"按钮)    │
                     └────┬───────┬─────────┘
         [←→]            │       │ [B]
      切换确认/取消       │ [A]   └──→ 返回目标选择
                           ▼
                     ┌──────────────────────┐
                     │     执行行动          │
                     └──────────────────────┘

        任意界面 ── [Start] ──→ 暂停菜单
        任意界面 ── [Select/View] ──→ 角色详情（当前行动角色）
        [Y] ──→ 聚焦战斗日志（↑↓ 滚动，[B] 取消聚焦）
        按住 LB + [←→] ──→ 切换查看角色状态栏
```

### 焦点头视觉指示器

```
┌─────────────────────────┐
│                         │← 2px 金色虚线边框 (#FFD700)
│     菜单选项 / 按钮      │  每段线长 8px，间隔 4px
│                         │  四角缺口 4px
└─────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 边框颜色 | 金色 `#FFD700` |
| 边框宽度 | 2px |
| 边框样式 | 虚线矩形（线段 8px + 间隔 4px） |
| 四角缺口 | 4px（像素风格——四角不闭合） |
| 闪烁周期 | 500ms（与 §8 #11 角色高亮一致） |
| 闪烁占空比 | 50%（250ms 可见 / 250ms 不可见） |

### 各区域导航顺序

| 区域 | 进入方式 | 焦点起始位置 | ↑↓←→ 行为 | A 按钮 | B 按钮 |
|------|---------|-------------|-----------|:------:|:------:|
| 行动菜单 | 回合开始自动聚焦 | 第一个可用行动（默认「攻击」） | ↑↓ 移动选项，首尾循环 | 确认选项 → 进入目标选择 | 关闭菜单 → 战斗空闲 |
| 目标选择 | 行动确认后 | 最近的可选目标 | ←→ 切换目标；LB/RB 切换目标组（敌方/友方） | 确认目标 → 进入确认弹窗 | 返回行动菜单 |
| 确认弹窗 | 目标确认后 | 「确认」按钮 | ←→ 切换「确认」/「取消」按钮 | 执行行动 | 返回目标选择 |
| 战斗日志 | 按 Y 聚焦 | 最新条目 | ↑↓ 滚动历史（最近 50 条） | 无操作 | 取消聚焦 → 返回原界面 |
| 队伍状态栏 | 按住 LB + ←→ | 当前行动角色 | ←→ 切换查看角色 | 查看角色详情（浮动窗口） | 释放 LB 返回 |
| 暂停菜单 | 按 Start | 第一个选项 | ↑↓ 移动选项 | 确认 | 关闭菜单返回游戏 |

### Step 流程 Gamepad 映射

| Step | ↑↓ | ←→ | A | B | LB/RB | Start |
|------|:--:|:--:|:--:|:--:|:-----:|:-----:|
| Step 1: 选择行动 | 切换行动选项 | — | 确认行动 | 关闭菜单 | — | 暂停 |
| Step 2: 选择目标 | — | 切换目标 | 确认目标 | 返回 Step 1 | 切换目标组（敌/友） | 暂停 |
| Step 3: 确认执行 | — | 切换确认/取消 | 执行 | 返回 Step 2 | — | 暂停 |
