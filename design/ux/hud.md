> **状态**: 草稿 (Draft) — 启动于 Technical Setup → Pre-Production 门控检查

> **关联文档**: `design/gdd/09-ui-ux-design.md`, `design/art/art-bible.md` §7, `src/DndGame/UI/PixelTheme.cs`
> **支柱对应**: P2（战术 DND 深度 — "清晰即是沉浸"）

---

## 1. 设计目标

战斗 HUD 的核心使命：让玩家在 0.3 秒内理解战术态势，在 2 秒内做出有意义决策。

- **信息可读性优先于装饰性** —— 艺术圣经原则 "清晰即是沉浸"
- **回合制结构允许信息分层展示** —— 不急于在一帧内展示全部
- **键盘可达** —— 无障碍 Basic 级要求

## 3. 信息层级

### 第一眼（0.3 秒内 —— 态势认知）

| 视觉信号 | 传达信息 | 实现方式 |
|----------|---------|---------|
| 当前行动角色高亮 | "轮到谁了" | 精灵周围闪烁边框（2px, 金色, 周期 500ms） |
| 敌方威胁标记 | "谁最危险" | 敌方格上方的威胁等级图标（▲/▲▲/▲▲▲） |
| 当前回合阶段提示 | "该做什么" | 顶部横幅 "选择行动" / "确认目标" / "敌人回合" |
| 角色生命状态 | "队友还活着吗" | HP 条颜色渐变（绿→黄→红）+ 倒地图标 |

### 第二眼（0.5–2 秒 —— 决策信息）

| 信息 | 位置 | 格式 |
|------|------|------|
| 行动菜单（6 选项） | 右侧面板 | 垂直列表，当前选中项高亮（浅蓝背景） |
| 可选目标（若已选行动） | 战斗区域 | 可攻击敌人闪烁红框，可治疗队友闪烁绿框 |
| 命中率预览（若已选目标） | 行动菜单下方 | "命中: 65% (需 ≥14)" |
| 可用资源 | 行动菜单各选项旁 | "🎯 攻击 (1/1)" "✨ 法术 (2/3)" |

### 第三眼（需要时 —— 深度信息）

| 信息 | 触发方式 | 格式 |
|------|---------|------|
| 角色详细面板 | 选中角色 → 按 Tab | 浮动窗口：全属性 + 装备 + 条件 |
| 敌人信息 | 悬停/选中敌人 → 按 Tab | 浮动窗口：AC + HP 大致 + 已知抗性 |
| 战斗日志历史 | 日志区域 ↑↓ 滚动 | 滚动列表（最近 50 条） |
| 法术详情 | 法术菜单中 → 按 F1 | 浮动提示：射程/范围/效果/消耗 |

## 4. 行动选择交互流（P2 核心流）

```
                         ┌──────────────┐
                         │  轮到玩家行动  │
                         └──────┬───────┘
                                │
                                ▼
                    ┌──────────────────────┐
                    │  Step 1: 选择行动类型  │
                    │  ↑↓ 导航菜单          │
                    │  Enter 确认           │
                    └──────┬───────────────┘
                            │
                ┌───────────┼───────────┐
                ▼           ▼           ▼
          ┌─────────┐ ┌─────────┐ ┌─────────┐
          │ 攻击    │ │ 法术    │ │ 其他    │
          │ 显示目标│ │ 显示法术│ │ 对应UI  │
          └────┬────┘ └────┬────┘ └────┬────┘
               │           │           │
               ▼           ▼           ▼
          ┌──────────────────────────────────┐
          │  Step 2: 选择目标/确认参数        │
          │  ← → 切换目标                    │
          │  Tab  查看目标详细信息           │
          │  Enter 确认执行                  │
          │  Esc   返回 Step 1              │
          └──────────────┬───────────────────┘
                         │
                         ▼
          ┌──────────────────────────────────┐
          │  Step 3: 确认执行                 │
          │  显示命中率 + 预期伤害/效果       │
          │  "确认对 [地精] 使用 [近战攻击]?" │
          │  Enter 执行  Esc 取消            │
          └──────────────────────────────────┘
```

### 关键交互规则

1. **每个 Step 都可撤销** —— Esc 随时回到上一步
2. **信息逐步揭示** —— Step 1 只显示行动类型，Step 2 才显示目标范围和命中率
3. **确认前显示预期结果** —— Step 3 必须显示命中率和伤害范围
4. **键盘全程可达** —— ↑↓←→ 导航，Enter 确认，Esc 撤销，Tab 详情

## 5. 状态显示约定

### HP 条颜色阶梯

| HP 比例 | 颜色 | 含义 |
|:-------:|------|------|
| > 66% | 绿色 `#44FF44` | 健康 |
| 33–66% | 黄色 `#FFFF44` | 受伤 |
| < 33% | 红色 `#FF4444` | 濒危 |
| = 0 | 灰色 + ☠ 图标 | 倒地 |

### 条件/状态图标

| 条件 | 图标 | 位置 | 格式 |
|------|------|------|------|
| 中毒 | ☠ (绿) | HP 条左侧 | 2px 像素图标 |
| 眩晕 | ◎ (黄) | HP 条左侧 | 闪烁 800ms |
| 魅惑 | ♥ (紫) | HP 条左侧 | — |
| 其他 | 按条件类型 | — | 8×8px 图标 |

### 先攻顺序条

- 顶部横向排列：当前行动角色在最左 + 高亮
- 角色头像 + 名字缩写（8px 字体）
- 敌人标记为红色边框，友方为蓝色边框

## 6. P2 支柱验证检查

| 检查项 | 实现 |
|--------|------|
| 玩家能否 0.3 秒内辨认威胁类型？ | ✅ 威胁等级 ▲ 图标 + 红色边框 |
| 伤害数值是否清晰可见？ | ✅ 伤害红色、治疗绿色弹出数字 |
| 行动选择是否让玩家感到"我的选择有意义"？ | ✅ 命中率预览 + 预期伤害范围 |
| 信息是否分层展示而非一次倾泻？ | ✅ 3 Step 认知流 |
| 是否避免了"氛围 vs 可读性"的妥协？ | ✅ 清晰优先，装饰为信息服务 |
