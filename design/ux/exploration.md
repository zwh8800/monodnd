## Purpose & Player Need

探索界面是《酒馆与命运》冒险循环的**核心游戏空间**——玩家在这里体验"踏入未知"的全部张力。它不是一个"从A点到B点"的过渡画面，而是一个充满发现、危险和战术决策的**大型连续地图**上的**活环境**。玩家始终停留在同一张连续地图上——无场景切换、无独立房间——所有遭遇、互动和战斗都在这张地图上实时发生。

**一句话定义**：
> "玩家来到探索界面，想要的是在大型连续地图上揭开迷雾、发现秘密、与环境互动，并在每一步中感受到——前方可能藏着宝藏，也可能藏着陷阱。"

**四种核心需求**：

| 需求 | 探索界面如何满足 | 设计承诺 |
|------|-----------------|---------|
| 🔍 **逐步发现** | 三层迷雾系统（未探索/已探索/可见）+ FOV 视野锥实时计算，覆盖整张连续地图 | 玩家不是"看到一切"——每走一步都有新东西被揭开 |
| ⚔️ **战术环境利用** | 8 种交互标签（可燃/可推/可攀/可破坏/可阅读/可燃液体/导电/可躲藏）+ 战斗在同地图上以区域限制模式触发 | 环境不是背景——推柱子堵路、点燃油桶、攀爬高处都是战术选择；战斗无需加载，直接在地图上展开 |
| 🎲 **技能驱动的探索** | 被动察觉提示 + 主动搜索/调查/解除陷阱检定 | 高感知角色先发现线索，但所有角色都能通过主动搜索找到核心内容 |
| 🌫️ **氛围与沉浸** | 冷色主导 + 暖色火把光点 + 视野边缘像素抖动过渡 | 遵循 Art Bible "蓄势待发、若隐若现、孤独感"的氛围目标 |

**如果探索界面不存在或难用**：
- 迷雾不透明或视野计算不对 → 玩家感到"瞎走"，失去发现张力
- 交互物体无高亮 → 玩家错过 80% 的环境互动，环境变成纯装饰
- 感知检定无反馈 → 堆了感知的角色感受不到优势；没堆感知的角色永远错过隐藏内容
- 区域过渡不流畅 → 大型连续地图上的区域间感受失去方向感和叙事连贯性

## Player Context on Arrival

探索界面是"冒险进行中"状态下的默认主视图——玩家醒着的大部分冒险时间都在这里。根据玩家从哪个入口进入大型地图，到达时的上下文有所不同：

| # | 到达路径 | 触发条件 | 玩家刚才在做什么 | 情绪状态 | 到达时的初始状态 |
|---|---------|---------|-----------------|---------|----------------|
| ① | **从 ADVENTURE_START 进入冒险** | Party Select 确认出发 → fade_black → 冒险起始区域 | 在队伍选择界面确认了队伍和任务——"准备好了，出发吧！" | 🧐 期待与好奇 — "冒险会是什么样的？前方有什么？" | 起始区域入口周围自动可见。其余范围被黑色迷雾覆盖。相机居中在队伍位置。小地图更新显示当前区域高亮。 |
| ② | **从相邻区域进入（区域边界触发）** | 角色跨越连续地图上的区域边界 | 刚探索完上一个区域——可能搜刮了物品、触发过战斗、或只是经过 | 😌→🤔 状态延续 — "刚才那片区域不错，这边会有什么？" | 同①。但保留上一区域的记忆（已探索区域在小地图上显示灰色轮廓）。无场景切换——相机平滑跟随过渡。 |
| ③ | **从战斗返回** | 战斗胜利 / 逃跑成功 → 同地图战斗区域限制解除 | 刚结束一场战斗——HP 可能不满、法术位可能消耗 | 😤 战斗余热 — 紧张尚未完全消退，需要评估"还能继续吗？" | 区域恢复探索状态。迷雾保留战斗前的可见区域（已探索的不会重新变黑）。敌人尸体/掉落物在对应 tile 上可见。战斗区域限制边界淡化消失。 |
| ④ | **从对话/NPC交互返回** | 对话结束、商人交易关闭 | 刚与 NPC 对话完毕——可能获得了信息、物品、或触发了新任务 | 😐 信息处理 — "那个精灵说的话……等等，暗门在哪？" | 对话窗口关闭，探索界面恢复。如果是"open_gate"效果触发的新区域解锁，新区域边界高亮闪烁提示（2 次脉冲）。 |
| ⑤ | **从谜题返回** | 谜题解决或失败后 | 刚解开（或没解开）一个谜题 | 🎉 成就感 或 😤 挫败感 | 谜题面板关闭。如果是成功解锁了新通道，新区域边界高亮闪烁提示（2 次脉冲）。 |

**首次遭遇探索（新玩家）**：
- 探索阶段开始时，底部操作栏上的「搜索」按钮显示脉冲引导（周期 1s，脉冲 3 次后自动消失），提示"点击搜索可以发现隐藏物体"。
- 第一个交互物体（距入口最近的 Readable/Climbable 等）以淡金色光晕脉冲提示（比正常高亮更显眼，脉冲 3 次后降级为普通高亮）。
- 任何用户交互（点击/按键/移动角色）立即停止所有引导脉冲。

**回访（老玩家）**：
- 无引导。所有交互物体使用标准高亮规则（§6.4 的 5 层视觉提示）。相机、迷雾、小地图按默认状态初始化。

## Navigation Position

探索界面在游戏导航树中的定位：

```
TAVERN_MAIN（酒馆主屏幕）
    │
    └──→ PARTY_SELECT → ADVENTURE_START
            │
            ▼
        EXPLORATION（探索阶段）← ★ 本界面
            │
            ├──→ 战斗（同地图区域限制模式 — 非场景切换）
            ├──→ DIALOGUE（对话覆盖层）
            ├──→ MERCHANT_SHOP（商店覆盖层）
            ├──→ REST_POINT（休整覆盖层）
            ├──→ PUZZLE（谜题覆盖层）
            └──→ SETTLEMENT（冒险结算）
```

探索界面**不是顶层目的地**——它仅在"冒险进行中"状态下存在。到达它有两种方式：从队伍选择进入冒险，或从事件/战斗返回。从这里可以触发的所有子状态（战斗/对话/商人/休整/谜题）都在同一张连续地图上执行——战斗在同地图上以区域限制模式触发（不切换场景），其余以覆盖层形式出现。所有子状态结束后**返回探索阶段**——它是冒险中的"地面状态"。

## Entry & Exit Points

### 进入路径

| # | 来源 | 触发方式 | 携带上下文 | 界面初始状态 |
|---|------|---------|-----------|-------------|
| ① | **ADVENTURE_START** | Party Select 确认出发 → fade_black 过渡 | 任务名称、冒险难度、队伍构成的完整快照 | 过渡动画（`fade_to_black` 0.5s）→ 大型连续地图加载 → 起始区域入口周围可见 + 全图迷雾 + 小地图更新 |
| ② | **区域边界触发** | 角色跨越相邻区域之间的边界 | 上一区域的探索状态、队伍位置延续、消耗资源不恢复 | 方向感知的平滑相机过渡，无场景切换。新区域迷雾叠加 |
| ③ | **战斗结束返回** | 战斗胜利 / 逃跑成功 → 战斗区域限制解除 | 队伍 HP/法术位/状态、敌人掉落物位置 | 战斗区域限制边界淡化消失。迷雾保留战斗前的可见区域。掉落物在对应 tile 可见 |
| ④ | **对话结束返回** | 对话覆盖层关闭 | 对话结果（获得信息/解锁通道/触发战斗） | 对话覆盖层 `overlay_pop` 逆向缩回。如触发 open_gate 效果 → 新区域边界高亮闪烁 |
| ⑤ | **谜题/商人/休整返回** | 对应覆盖层关闭 | 谜题结果/交易结果/休整恢复 | 同④，覆盖层缩回。探索界面恢复该覆盖层打开前的状态 |

### 离开路径

| # | 目的地 | 触发方式 | 动画 | 备注 |
|---|--------|---------|:---:|------|
| 1 | **地图其他区域** | 角色跨越区域边界 | 相机平滑跟随过渡 0.4s（方向感知） | 连续地图上的自然移动——非离散房间切换。方向由行进方向决定 |
| 2 | **战斗（同地图区域限制模式）** | 遭遇触发（proximity/interaction/skill_check_failure/timed） | 战斗区域边界脉冲视觉提示 0.3s（红色边框脉冲） | ⚠️ 非场景切换——战斗在同地图上触发、以区域限制模式运行。不可逆——一旦触发必须解决（逃跑或胜利） |
| 3 | **对话覆盖层** | 点击/走近 NPC → 触发对话 | `overlay_pop` 0.15s（覆盖在探索上） | 探索界面保持渲染（暗化），对话覆盖层在上层 |
| 4 | **商人覆盖层** | 点击商人 NPC → 触发商店 | `overlay_pop` 0.15s | 同上 |
| 5 | **休整覆盖层** | 使用休整节点 | `overlay_pop` 0.15s | 同上 |
| 6 | **谜题覆盖层** | 与谜题物体交互 | `overlay_pop` 0.15s | 同上 |
| 7 | **冒险结算** | Boss 击败 + 走到出口 / 冒险目标完成 | `fade_to_black` 0.3s → SETTLEMENT | ⚠️ 不可逆——冒险结束 |
| 8 | **死亡/团灭** | 所有角色 HP=0 → 死亡豁免失败 | `fade_to_black` 1.0s（慢速）→ FAILURE_SCREEN | ⚠️ 不可逆——触发永久后果 |

## Layout Specification

### Information Hierarchy

探索界面的信息按"玩家决策的自然顺序"分为五个优先级层级：

| 优先级 | 信息 | 理由 |
|:------:|------|------|
| **P0 — 第一眼 (0-0.3s)** | A1 地形、A2 队伍角色、A5 迷雾、A6 FOV | "我在哪？我能看到什么？"——空间定位是其他一切的基础 |
| **P1 — 即时战术 (0.3-2s)** | A7 交互高亮、A8 "!"提示、A9 光源、A11 区域边界、B3 队伍HP、B4 区域名 | "这片区域有什么？队友状态如何？"——这是玩家开始探索前的态势感知 |
| **P2 — 持续参考 (始终)** | B1 小地图、B2 底部操作栏、B5 光源状态 | 导航辅助 + 操作入口，玩家需要时随时可看 |
| **P3 — 按需关注** | A3 NPC、A4 敌人、A10 特效、A12 掉落物、B7 工具提示 | 这些信息只在特定情境下才重要（走到附近/悬停时） |
| **P4 — 事件触发** | B6 通知、C1-C5 各弹出层 | 由事件驱动，出现时抢占注意力 |

| C5 | 遭遇预警提示（接近触发范围） | 角色距遭遇触发点 ≤ trigger_radius_tiles |

### Layout Zones

**选定方案：方案 A — 全屏沉浸 (Full-Bleed Immersive)**

> **设计哲学**：把屏幕交还给游戏世界。HUD 只是半透明叠加层，不占有独立空间。与 FF6 暗色半透明窗的既定视觉语言一致，最大化"蓄势待发、若隐若现"的探索氛围（Art Bible §2.2）。
>
> **视觉投影约束**：探索界面是游戏世界层的直接呈现，必须严格遵循 [Art Bible §3.0](../art/art-bible.md#30-摄像机与透视正交投影铁律) 正交投影铁律——固定俯视角 3/4 overhead、无消失点、无近大远小、Tile 和 Sprite 保持固定像素尺寸、深度仅通过 Z-order 层叠遮挡表达。迷雾边缘使用像素抖动过渡（非景深模糊），FOV 视野锥是 2D 叠加层而非 3D 光照计算。参考：Stardew Valley / Chrono Trigger 的野外探索视角。

```
┌──────────────────────────────────────────────────────────┐
│ 1280×720 全屏                                              │
│                                                            │
│  ┌─ [被遗忘的图书室] ──────────────────────── [小地图] ─┐ │ ← Zone T: 顶栏 36px, alpha 0.7
│  │                                                      │ │
│  ├──────────────────────────────────────────────────────┤ │
│  │                                                      │ │
│  │            Zone G: TILE GRID（全屏铺满）               │ │
│  │            32×32 tiles @ 2.0× = 64px/tile            │ │
│  │            相机跟随队伍，平滑滚动（连续大地图）          │ │
│  │                                                      │ │
│  │    [角色]  [NPC]    [🔥交互物]                        │ │
│  │                                                      │ │
│  │                                                      │ │
│  ├──────────────────────────────────────────────────────┤ │
│  │ Zone P: [索林 ████] [莉莉丝 ███] [艾尔登 ██] [维拉 ███] │ ← 队伍状态 48px
│  ├──────────────────────────────────────────────────────┤ │
│  │ Zone A: [🔍搜索] [🤝交互] [🎒物品] [👤躲藏] [🔧解除] [🔎检查] │ ← 操作栏 80px
│  └──────────────────────────────────────────────────────┘ │
│    ↑ 底部 HUD 半透明背景 (rgba 10,10,30, 0.85)              │

```

**区域尺寸与属性**：

| 区域 | 代号 | 位置 | 尺寸 | 透明度 | 说明 |
|------|:---:|------|:---:|:---:|------|
| 顶栏 | **Zone T** | 顶部全宽 | 1280×36px | 0.7 | 左：区域名(12px)。右：小地图 200×150（从顶栏向下延伸覆盖右上角） |
| 网格主视图 | **Zone G** | 全屏 | 1280×720px | 1.0 | 全屏铺满，扣除 HUD 后纯网格可视区约 20×8.5 tiles（2.0× 下）。相机平滑跟随队伍在大型连续地图上移动 |
| 队伍状态 | **Zone P** | 底部操作栏上方 | 1280×48px | 0.85 | 4 角色横排。头像 32×32 + 名字 10px + HP 条 100×6px + 条件图标 |
| 操作栏 | **Zone A** | 底部全宽 | 1280×80px | 0.85 | 6 按钮横排，间距 20px，居中分布。按钮 80×56px（图标 32×32 在上 + 标签 10px 在下） |

**备选方案（已考虑，未采用）**：
- **方案 B — 固定 UI 框架**：不透明独立区域，网格区缩小约 25%。与 FF6 半透明窗风格冲突，降低沉浸感。
- **方案 C — 自适应 HUD**：自动隐藏操作栏，闲置 3s 后收起。最大化视野但增加新玩家困惑风险和实现复杂度。

### Component Inventory

| **S2** | **空状态（空区域）** | 进入的区域不含任何交互物体/NPC/敌人/隐藏物（如纯走廊或已清空的区域） | 地形+墙壁正常渲染。无交互高亮、无"!"提示。底部操作栏按钮全部灰显（无可交互对象）。小地图正常显示 | 仅移动可用。搜索按钮灰显（无可搜索内容 — tooltip:"周围没有值得搜索的东西"）。其余操作按钮同样灰显 |
| **S3** | **首次进入（新玩家引导）** | 玩家的探索阶段首次进入（全局标记 `has_completed_first_exploration = false`） | **搜索按钮**：Zone A 的「搜索」按钮边框脉冲高亮（暖金色，周期 1s，亮度 100%→60%→100%），脉冲 3 次后自动停止。**最近交互物**：距入口最近的可交互物体以加强脉冲（淡金色光晕，比标准高亮更显眼），脉冲 3 次后降级为普通高亮。任何用户输入（点击/按键/移动）立即停止全部引导脉冲 | 全部操作可用。引导脉冲不阻塞操作——玩家可以直接操作而无需等待脉冲结束 |
| **S4** | **加载态（进入探索阶段过渡）** | 玩家从 ADVENTURE_START 进入大型连续地图 | 根据 GDD §4.1 转换类型渲染过渡动画：`fade_to_black`（0.5s 渐黑）→ 大地图初始区域加载 → 渐亮。过渡期间：旧画面渐隐，探索地图渐显。底部可能出现"加载中…"文字（若加载 > 0.5s） | **全部输入屏蔽**。过渡期间不接受任何玩家输入。过渡完成 + 0.2s 防误触缓冲后恢复输入 |
| **S5** | **黑暗环境（无光源）** | 队伍没有任何光源（火把耗尽/Light 法术到期）且所有角色均无 Darkvision | Grid 区：仅队伍周围 **1 tile** 区域以微光渲染（alpha 0.3，几乎不可辨识）。其余区域全黑（G5 迷雾 = 100% 不透明黑）。Zone T 的 T3 光源状态闪烁红色警告。Toast："四周一片漆黑——你需要一个光源。"（持续 3s）。条件图标：无 Darkvision 角色头部叠加「目盲」图标 | **移动受限**：仅可移动到相邻 1 tile（无法看到更远的目标 tile）。A* 寻路降级为仅相邻格。搜索、交互按钮灰显（无法在黑暗中操作）。使用物品按钮可用——玩家可以打开背包使用火把/Light 法术 |
| **S6** | **感知检定进行中（骰子动画）** | 玩家点击「搜索」按钮 → 主动搜索触发 | 屏幕中央出现骰子动画覆盖层（参照 HUD §8 骰子动画分镜，1650ms 合计）。背景探索界面略微暗化（alpha 降至 60%）。骰子上方显示"察觉检定"标签。结果定格 500ms：成功 → 绿色"17" + "发现！"弹出；失败 → 灰色"8" + "没有发现异常"弹出 | **仅骰子覆盖层可交互**：玩家可 `Esc` 跳过动画（直接显示结果）。探索界面本身在检定期间不可操作。检定完成后覆盖层自动关闭 |
| **S7** | **分支选择弹出（Branch 区域）** | 玩家在 branch 区域触发选择（进入 branch 区域 → 自动弹出 或 与信息柱交互后弹出） | 覆盖层弹出（`overlay_pop` 0.15s）：2-4 个分支选项卡片，每个卡片含：分支名 + 难度提示（如"安全"） + 奖励提示（如"古老魔法物品"） + 风味文字。背景探索界面暗化（alpha 降至 40%）。选中的分支高亮（模式 12 — L3 选中态）。已封锁路径显示🔒标记（小地图同步更新） | **两步确认机制**（GDD §2.2.8）：① 首次点击→选项高亮 + 警告"此选择不可更改，确定前往 [分支名] 吗？" ② 再次点击确认→执行。`Esc` 在①后取消选择。锁定动画：确认后未选分支路径播放"关闭"动画（0.3s） |
| **S8** | **全屏地图覆盖层** | 按 `Tab` / `M` 键 | `instant` 切换：探索界面上叠加全屏地图覆盖层（非场景切换）。显示已探索区域 + 地形轮廓 + 当前区域高亮 + 队伍位置标记。切换时探索状态不丢失——队伍位置/迷雾/交互状态保持不变 | 全屏地图覆盖层交互：可浏览所有已探索区域，点击已探索位置可导航角色/相机至该位置（同地图上移动）。再按 `Tab` / `M` 关闭覆盖层回到探索阶段。`Esc` 同样关闭覆盖层 |
| **S9** | **战斗预警态（遭遇即将触发）** | 角色进入遭遇触发范围（距离触发点 ≤ 2 tiles 且 `trigger_type = proximity`），且遭遇 `can_avoid = false` | 屏幕四边 16px 区域以红色脉冲闪烁（`#FF4444`，alpha 30%→60%，周期 500ms，持续 0.5s）。Toast："前方有危险！"（1.5s）。相机略微 zoom out（如果战斗自动缩放已启用）。当前行动角色精灵边框闪烁红色 | **0.5s 预警期间**：移动自动停止（角色停在触发范围边界）。操作栏全部灰显。0.5s 后触发战斗——同地图区域限制模式激活（红色边框标记战斗区域）。不可取消——这是强制遭遇 |
| **S10** | **死亡/团灭过渡** | 所有角色 HP=0 → 死亡豁免全部失败（3 轮无治疗） | `fade_to_black` 1.0s（慢速，比正常过渡慢一倍）。fade 期间：角色精灵逐渐褪色（saturation 降至 0 → grayscale）。屏幕边缘暗角从四角向中心收缩。最后画面定格在队伍倒下的场景 0.5s → 完全黑屏。Audio：BGM 渐弱至静音，低沉持续音。0.5s 黑屏后进入 FAILURE_SCREEN | **全部输入屏蔽**。1.5s 过渡期间玩家无法进行任何操作。不可逆——一旦触发必定进入结算 |

**补充说明**：

- **S5 黑暗环境 与 Darkvision**：如果队伍中至少 1 名角色拥有 Darkvision（精灵/矮人等，12 tiles），则不触发 S5。Darkvision 角色周围 12 tiles 以微光渲染（灰度滤镜），非 Darkvision 角色共享此视野（"队友告知"的叙事解释）。
- **S6 感知检定 与 S7 分支选择**：两者不会同时触发。如果玩家在 S7（分支选择弹窗中）按 `Esc` 取消后执行搜索 → 先关闭 S7 覆盖层，再触发 S6 检定动画。
- **S9 战斗预警 的可躲避遭遇**：如果 `can_avoid = true`（如可通过隐匿绕过），不触发 S9 的红屏脉冲——改为 Toast "前方有动静…" + 隐匿/绕路选项提示。

## Accessibility

> **目标等级**：Standard（`design/accessibility-requirements.md`）。覆盖：键盘导航、手柄导航、色彩双重编码、字体缩放、文本对比度、Reduced-Motion。

---

### G.1 纯键盘导航路径

> **断言**：探索界面的所有核心操作必须可通过键盘完成，无需鼠标。

**键盘导航流程**：

```
移动与导航:
  WASD / 方向键 → 逐格移动角色（替代鼠标点击 tile）
  Tab / M      → 全屏地图覆盖层开关（S8）
  Tab          → 焦点循环: 游戏世界 → Zone P 角色槽 → Zone A 操作栏 → 循环
  Shift+Tab    → 反向循环

角色选择:
  1-4 数字键   → 快速选择第 1-4 位角色（G2）

物体交互:
  E            → 交互（A2）——前提：角色距物体 ≤ 1 tile
  F            → 搜索（A1）
  I            → 使用物品（A3）
  H            → 躲藏（A4）
  D            → 解除陷阱（A5）
  X            → 详细检查（A6）

覆盖层:
  Enter (在角色槽聚焦时) → 打开角色详情卡片（P1）
  Esc (卡片打开时)       → 关闭卡片

系统:
  Start               → 暂停菜单弹出
```

**键盘 Tab 序**（焦点循环顺序）：游戏世界（WASD 默认态）→ Zone P 角色槽 → Zone A 按钮 → 循环。小地图不设键盘焦点——键盘用户使用 `Tab/M` 切换到全屏地图覆盖层进行区域浏览。

**键盘局限**：长距离 tile 级精确移动仅可通过 `WASD` 逐格完成。建议键盘用户使用全屏地图覆盖层（`Tab`）进行跨区域导航。

### G.2 手柄导航顺序

> 手柄支持级别：**Partial**（`technical-preferences.md`）。D-Pad 用于 UI 导航和角色移动。

| 手柄按键 | 探索界面行为 |
|---------|------------|
| `D-Pad / L-Stick` | 逐格移动角色 / UI 焦点导航 |
| `A` | 确认操作（交互/选择/打开） |

**手柄限制**（Partial 支持级别）：小地图节点点击不可用——使用 `Select` 全屏地图覆盖层替代。相机缩放不可用——使用默认 2.0×。所有核心探索操作均可通过手柄完成。

### G.5 色盲模式适配

启用色盲模式时：交互物体高亮的形状编码保持不变，颜色映射替换为色盲安全色板。HP 条增加图案填充备选（竖条纹=健康，斜条纹=受伤，交叉条纹=濒危）。小地图节点图标不变（符号已是主要识别手段）。

### G.6 屏幕阅读器

Standard 等级**不要求**屏幕阅读器支持。Myra 框架原生不支持无障碍树。缓解：文字缩放最大档（1.6×）+ 多感官通道（视觉 + 文字 + 音效）。

## Acceptance Criteria

> QA 测试人员必须能仅凭以下条件逐一验证通过/失败，**无需阅读任何其他设计文档**。

---

### 性能

- [ ] **AC-PERF**：进入探索阶段到完全可交互（迷雾渲染完成 + 操作栏可点击），加载时间 ≤ 2.0 秒（含 `fade_to_black` 过渡动画时长）。

---

### 导航

- [ ] **AC-NAV1**：从队伍选择进入探索阶段 → 经过 `fade_to_black`（0.5s）过渡动画 → 进入大型连续地图。验证：顶栏显示正确区域名；小地图显示当前区域高亮边框；起始区域周围可见、其余被黑色迷雾覆盖。

- [ ] **AC-NAV2**：角色跨越区域边界 → 相机平滑跟随过渡 0.4s（无场景切换——同一连续地图上的自然移动）→ 进入相邻区域。验证：新区域名更新 + 相邻区域迷雾初步展示 + 无场景切换（游戏世界始终保持渲染）。

---

### 迷雾与视野

- [ ] **AC-FOV1**：进入全新区域（该区域此前从未被访问过）→ 仅入口周围 tiles 区域完全可见（无色覆盖）；其余区域被**完全不透明的黑色**覆盖。验证：无法透过黑色区域看到任何地形或物体。

- [ ] **AC-FOV2**：角色移动（WASD 或点击 tile）→ FOV 视野锥实时更新。验证：新位置周围的 tile 变为可见（黑色迷雾被揭开）；离开后原位置 tile 降级为灰色半透明（50% alpha）——地形可见但角色/物体不可见。

---

### 交互高亮

- [ ] **AC-HIGHLIGHT**：角色移动至距离交互物体 ≤ 2 tiles → 物体边缘出现彩色闪烁边框（颜色 + 形状双重编码：Pushable=棕色▨, Flammable=橙色▲, Climbable=绿色⬆, Breakable=红色✕, Readable=蓝色◈）。角色离开 > 2 tiles → 高亮消失。验证：高亮仅近距离激活，远距离不显示。

---

### 小地图

- [ ] **AC-MINIMAP**：小地图（右上角 200×150px）始终可见。验证：显示当前区域轮廓 + 已访问过区域的灰色填充 + 未访问区域不显示或显示 `?` 标记；队伍位置以青色三角 ▼ 闪烁指示；当前区域边框高亮（淡金色闪烁）。

---

### 搜索操作

- [ ] **AC-SEARCH1**：在有隐藏物体的探索区域中，点击「搜索」按钮（或按 `F`）→ 屏幕中央出现骰子动画（d20 滚动约 1.6s）。检定成功时 → 绿色结果数字弹出 + 隐藏物体从不可见变为可见（暗门显现、隐藏物品浮现、陷阱高亮）。检定失败时 → 灰色结果数字 + Toast："没有发现异常"。

- [ ] **AC-SEARCH2**：在空区域（无隐藏物体/NPC/敌人/陷阱）中，搜索按钮灰显。悬停灰显按钮 → Tooltip："周围没有值得搜索的东西"。

---

### 空状态

- [ ] **AC-EMPTY**：进入一个不含任何交互物体、NPC、敌人、隐藏物、陷阱的纯走廊区域 → 底部操作栏所有 6 个按钮（搜索/交互/物品/躲藏/解除/检查）均灰显且不响应点击。仅移动操作（WASD/点击 tile）可用。

---

### 黑暗环境

- [ ] **AC-DARK**：队伍没有任何光源（火把耗尽、Light 法术到期）且所有角色均无 Darkvision → 画面仅队伍周围 1 tile 区域以微弱光线渲染（几乎全黑）。Toast 弹出："四周一片漆黑——你需要一个光源。"（持续 3s）。光源状态指示器（Zone T）闪烁红色警告。搜索和交互按钮灰显。使用「物品」按钮打开背包 → 可使用火把——使用后视野恢复。

---

### 连续地图过渡

- [ ] **AC-TRANSITION**：角色从西侧接近区域边界 → 相机平滑向西平移（0.4s，无场景切换）→ 新区域内容逐渐显示。验证：相机移动方向与行进方向一致。角色从北侧接近区域边界 → 相机向北平移。全程无场景切换——游戏世界保持连续渲染。

---

### 全屏地图覆盖层

- [ ] **AC-OVERLAY**：按 `Tab` 键 → 全屏地图覆盖层以 `instant` 方式叠加在探索界面上（非场景切换）。验证：覆盖层显示已探索区域轮廓 + 队伍位置标记 + 当前区域高亮；下层探索界面状态完整保留。再按 `Tab` → 覆盖层移除，返回探索阶段。`Esc` 同样关闭覆盖层。

---

### 战斗触发（同地图区域限制）
