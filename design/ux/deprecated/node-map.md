## Purpose & Player Need

> ⚠️ **已废弃**: 以下描述的"节点地图"概念已被统一的"探索阶段"取代。战斗现在在同地图上以区域限制模式触发。请参考 `design/ux/exploration.md`。

**节点地图是冒险的"战略导航层"——玩家在这里俯瞰已揭示的路线拓扑、评估每条分支的风险与回报、做出（可能不可逆的）路线抉择，然后将决策交付给微观的房间探索。**

### 一句话定义

> "玩家来到节点地图，想要的是**看清前方的路——已经可以看到哪些节点、每条路通向哪里、我的选择会锁死其他可能吗——然后带着策略意图踏入下一个房间。**"

### 三种核心需求

| 需求 | 节点地图如何满足 | 设计承诺 |
|------|-----------------|---------|
| 🗺️ **路线认知** | 相邻可见节点提前揭示（1-2 跳），更远节点逐步展开。玩家能看清"目前的选项"但看不到"全局的完整结局"。 | 揭示节奏从属于 GDD 05 迷雾系统——逐步展开，保留探索的"未知感" |
| ⚖️ **风险权衡** | 每个可见节点展示类型图标（⚔/💬/🔍/🧩/🪙/🔥/💀/✦）+ 悬停难度提示。分支节点让玩家在对比路径后做出有信息量的选择。 | 类似 Slay the Spire 的路线规划，但仅限已揭示区域——保持"前方迷雾"的紧张 |
| 🔒 **选择有重量** | 分支节点的选择**不可撤回**——未选路径进入锁定态（🔒图标覆盖 + 路径断裂动画），强化 Pillar 3"每个决策有后果"。 | 锁定态是第四种节点视觉状态，补充 GDD 09 §5.1 原有的三种状态体系 |

### 如果节点地图不存在或难用

- 玩家看不到路线结构 → **盲目游走，"不是我在选择，是游戏在推我走"**
- 节点类型不可辨识 → **踩到 Boss 房毫无准备，频繁团灭**
- 分支选择不清晰 → **不知道是不可逆选择，后悔想回退发现不行 → 信任破裂**
- 锁定态没有视觉反馈 → **不知道自己失去了什么，好奇心变挫败感**

### 设计铁律

> **"揭示，但不剧透；选择，但让玩家知道代价。"**

节点地图遵循底层节点图的可见性规则——只揭示**拓扑上可达的下一步**。这保证：
- **探索感**：玩家永远不知道三跳之外的节点是什么
- **策略感**：玩家能看到当前面临的所有路线选项，做出有信息量的决策
- **后果感**：锁定态让玩家明确感知"那些路已经永远关闭了"

### 与房间探索的对偶关系

| | 节点地图（本屏幕） | 房间探索（下一层） |
|---|---|---|
| **空间尺度** | 整个冒险 | 单个房间 |
| **时间节奏** | "接下来走哪条路？" | "这扇门后面有什么？" |
| **决策后果** | 可能不可逆（分支锁定） | 可重复（多个房间内行动） |
| **信息揭示** | 逐步展开 → 锁定关闭 | 逐格揭开（迷雾） |
| **情感基调** | 战略 — 权衡/犹豫/承诺 | 战术 — 发现/警觉/执行 |

## Player Context on Arrival

节点地图是冒险循环的中枢——玩家会在两种截然不同的情境下反复到达。

### 路径 ①：首次进入冒险（Party Select → Adventure Start → Map View）

| 维度 | 描述 |
|------|------|
| **触发条件** | Party Select 中确认队伍 → 按下「出发」→ fade_black 场景切换 |
| **玩家刚才在做什么** | 仔细阅读任务预览、搭配队伍、检查装备——完成了一套"战前准备仪式" |
| **情绪状态** | 🎯 **混合型**——短冒险偏好奇探索（"这次会遇到什么？"），高难度/深冒险偏策略专注（"第一步必须走对"）。共性：adrenaline 正在上升但尚未到达峰值 |
| **携带的上下文** | 任务名称、难度等级、队伍构成和状态、出发前的最后记忆 |
| **设计响应** | 首次到达时展示**轻量引导**——1-2 个"脉冲高亮"提示指向当前位置标记和图例行，传递"这是地图，你可以点击节点前进"的信息。提示在 3 秒后或首次交互后自动消失——不打断节奏。节点地图展示冒险标题 + 起始节点（Opening）+ 已揭示的相邻 1-2 跳节点 |

**轻量引导的两条提示**：
1. 当前位置标记（▲）脉冲高亮 3 次（500ms 周期）——"你在这里"
2. 图例行淡入高亮 2 秒——"点击任意可见节点即可前往"

### 路径 ②：从房间探索返回（最常见，占据屏幕访问的 80%+）

| 维度 | 描述 |
|------|------|
| **触发条件** | 房间探索完成一个节点（战斗胜利/对话结束/探索完成/休息恢复）→ **自动返回节点地图**（非玩家主动选择） |
| **玩家刚才在做什么** | 战斗（可能险胜或大胜）、对话（获得了情报）、探索（发现了宝藏或陷阱）、休息（正在恢复）——每个节点的经历完全不同 |
| **情绪状态** | ⚖️ **策略评估模式**——不是"闲逛"，而是带着明确意图："我的状态还能走多远？"、"那条路最优？"、"该不该先休息？"。消耗越严重，策略评估越急切 |
| **携带的上下文** | 更新后的队伍状态（HP/法术位/消耗品变动）、新揭示的相邻节点、可能的锁定状态变化、队伍成员可能获得的新条件/伤疤 |
| **设计响应** | 返回时**不弹出任何确认对话框或中断提示**——播放平滑过渡动画（fade_to_black → 节点地图渐亮），让玩家立刻回到俯瞰视角。地图必须在返回后清晰展示三件事：① 刚离开的节点现在标记为已访问（灰色），② 当前位置标记（▲）移动到新节点，③ 新揭示的相邻节点以"迷雾消散"动画出现 |

### 回访时的"变化感"

节点地图在整个冒险中会被反复访问——每次返回都可能揭示新路径、锁定旧路径、展示不同的节点状态。回访时的设计重点：

| 变化 | 视觉信号 |
|------|---------|
| **新节点揭示** | 以"迷雾消散"动画出现（0.3s，从暗到亮），从当前位置向外辐射扩散 |
| **节点变为已访问** | 刚刚完成的节点从"可见态（白色）"平滑过渡到"已访问态（灰色）"，过渡时长 0.3s |
| **分支路径锁定** | 未选路径以"关门"动画显示（0.3s），路径连接线断裂消失，节点上覆盖 🔒 图标 |
| **当前位置移动** | ▲ 标记沿路径连接线"滑动"到新节点位置（0.3s，跟随路径走向），强化"我前进了"的空间感知 |

### 到达方式的本质区别

| | 路径 ① 首次进入 | 路径 ② 返回 |
|---|---|---|
| **玩家主动权** | 主动选择（按了出发） | 被动推送（系统自动返回） |
| **到达前行为** | 战前准备（阅读/搭配） | 战斗/对话/探索/休息 |
| **到达时意图** | "探索世界" | "评估下一步" |
| **认知负荷** | 低——好奇心态 | 高——携带最新战斗消耗记忆 |
| **地图展示** | 首次揭示 + 轻量引导 | 增量变化 + 当前状态 |
| **入场动画** | fade_black 0.5s 渐亮 | fade_to_black 0.3s 快速渐亮 |
| **是否需要确认** | 否——直接进入 | 否——不弹出任何对话框 |

## Navigation Position

节点地图位于游戏导航树中**冒险子系统的根节点**——它只在冒险进行期间存在，不是全局可达的覆盖层或主菜单项。

```
TITLE_SCREEN
    │
    ▼
TAVERN_MAIN（酒馆主屏幕）
    │
    ├──→ PARTY_SELECT（队伍选择）
    │       │
    │       └──→ ADVENTURE_START ──→ MAP_VIEW ◄── 冒险子系统入口
    │                                      │
    │                                      │  ← 冒险核心循环 →
    │                                      │
    │                               ┌──────┴──────┐
    │                               │             │
    │                          ROOM_EXPLORE   ROOM_EXPLORE
    │                          (当前节点)      (下一节点)
    │                               │             │
    │                               └──────┬──────┘
    │                                      │
    │                               [自动返回 MAP_VIEW]
    │                                      │
    │                                      ▼
    │                               SETTLEMENT（结算）
    │                                      │
    │                                      ▼
    │                               TAVERN_MAIN（返回酒馆）
```

**关键性质**：
- **不是覆盖层**：节点地图是完整的全屏场景——不是可通过 Esc 随时打开关闭的弹窗。玩家不能"取消"地图回到酒馆
- **仅在冒险期间存在**：冒险开始（Adventure Start）后进入，冒险结算（Settlement）后离开。在酒馆中无法直接打开
- **是冒险循环的中枢**：所有房间探索都从节点地图进入，完成后返回节点地图。它是玩家在冒险中的"空间锚点"

## Entry & Exit Points

### 进入路径

| # | 来源 | 触发方式 | 玩家携带的上下文 |
|---|------|---------|----------------|
| ① | **Party Select** | 确认队伍 → 按下「出发冒险」→ fade_black 场景切换 | 任务名称、冒险难度、队伍构成的完整快照、出发时的期望和心情 |
| ② | **房间探索完成** | 节点目标达成（战斗胜利/对话结束/探索完成/休息结束等）→ 自动 fade_to_black 返回 | 更新后的队伍状态（HP/法术位/新条件/消耗品变动）、刚完成节点的经历记忆、新揭示的相邻连接 |
| ③ | **存档恢复** | 从标题画面「继续」→ 选择冒险中的存档槽 → 恢复节点地图状态 | 存档时的队伍状态、已访问节点列表、已揭示的路径拓扑、世界状态标记 |

> **存档恢复注意事项**：恢复存档时，节点地图应以"已揭示状态"立即展示整个已知拓扑（不需要重新播放迷雾消散动画），仅在存档时尚未被发现的节点保持隐藏。当前位置标记（▲）直接放置在存档时的当前节点上——不做滑动动画。

### 离开路径

| # | 目标 | 触发方式 | 说明 |
|---|------|---------|------|
| ① | **房间探索** | 点击/选择可见节点（非锁定态）→ 确认出发 → 转入对应玩法场景 | **主流程出口**。离开节点地图后进入该节点对应的玩法场景（战斗/对话/探索/休息/商店/Boss/谜题等） |
| ② | **冒险结算** | 完成特定节点（通常是 Boss 或终点节点，或无剩余可达节点）→ 系统自动触发结算 | **不可逆**——一旦结算开始，无法返回节点地图。玩家进入 Settlement 界面，看到成功/失败结果，然后返回酒馆 |
| ③ | **暂停覆盖层** | 按 `Esc` → 暂停覆盖层弹出 | **不离开节点地图**——暂停覆盖层是覆盖层（模式 7），关闭后返回节点地图。此出口不在本屏幕的"离开"范围内 |

> **分支锁定**不被视为"离开路径"——它是节点地图内的状态变化。选择分支后，地图保持在当前屏幕，仅锁定未选路径并标记当前位置。

### 进入/离开均不弹出确认对话框

节点地图是信息展示+选择导航的屏幕，不存在"你是否确定要查看地图？"的场景。唯一的确认发生在玩家**点击可见节点准备出发时**（见 Layout & Interaction 节）。

## Layout Specification

### Information Hierarchy

节点地图的信息按"第一眼必须看到 → 可探索时发现"分为三个层级：

| 层级 | 信息 | 理由 |
|:----:|------|------|
| **T1**（立即可见） | 当前位置标记（▲）、节点拓扑连接、节点类型图标 | 核心决策信息——"去哪"由这三者共同决定 |
| **T2**（扫一眼） | 节点视觉状态（已访问/可见/未探索/锁定）、冒险标题、图例 | 状态辅助判断、上下文确认。新人需要图例，老手可忽略 |
| **T3**（主动获取） | 节点预览详情（悬停浮层）、队伍状态摘要、冒险进度、全图切换 | 需要时触发，不抢占地图空间 |

> **关键洞察**：节点地图不是 HUD——玩家不需要像战斗中那样实时追踪队伍 HP。节点地图是"规划站"：先看地图 → 决定方向 → 出发前检查状态。T3 信息通过悬停/按钮主动获取。

### Layout Zones

采用**方案 C：地图优先 + 按需浮层**。1280×720 基准分辨率下，像素艺术地图占据约 90% 屏幕。

```
┌──────────────────────────────────────────┐
│ Zone H: 顶栏 — 标题 + [全图] 按钮 (32px)  │
├──────────────────────────────────────────┤
│                                          │
│ Zone M: 节点拓扑图 (~90%屏幕)             │
│   └─ M4: 悬停浮层 — 按需弹出              │
│   └─ M5: 分支确认对话框 — 按需弹出         │
│                                          │
├──────────────────────────────────────────┤
│ Zone S: 底部状态条 — 队伍HP + 进度 (20px) │
├──────────────────────────────────────────┤
│ Zone L: 图例栏 — 图标+状态说明 (24px)     │
└──────────────────────────────────────────┘
```

### Component Inventory

#### Zone H — 顶栏 (Header Bar, 32px)

| # | 组件 | 类型 | 显示内容 | 可交互 | 模式引用 |
|---|------|------|---------|:---:|---------|
| H1 | 冒险标题 | Label | `冒险: {冒险名称}`（14px，白色，1px 深色描边） | 否 | — |
| H2 | 全图切换按钮 | Button | `[全图]` 或 `[返回]`（切换态），带 `Tab` 快捷键提示。点击后在全图模式（所有已揭示节点缩小显示 + 可选路线高亮）和默认模式之间切换 | **是** | Pattern 12 (Highlight) |

> **全图模式**：按下 `Tab` 或点击按钮，地图缩小至所有已揭示节点可见。在此模式下节点不可点击出发，仅用于全景预览。再次 `Tab` 回到默认视图。

#### Zone M — 节点拓扑图 (Node Topology Map, 主区域)

| # | 组件 | 类型 | 显示内容 | 可交互 | 模式引用 |
|---|------|------|---------|:---:|---------|
| M1 | **节点图标** | Icon (像素精灵) | 像素图标 + 下方 1 行名称（10px） | **是** | **🆕 新模式** |
| M2 | **连接线** | Line render | 节点间连线：实线=可达、虚线=隐藏、断裂=锁定 | 否 | **🆕 新模式** |
| M3 | **当前位置标记** | Animated marker | ▲ 三角标记 + 脉冲光晕（500ms 周期），置于当前节点上方 | 否 | **🆕 新模式** |
| M4 | **悬停浮层** | Tooltip panel | 节点名称、类型图标、难度提示、简短风味文本、[前往] 按钮 | **是** | Pattern 6 (Tooltip) |
| M5 | **分支确认对话框** | Modal overlay | 选中分支后弹出："此选择不可更改，确定要前往 {分支名} 吗？"+ 再次点击确认 | **是** | Pattern 1 (Confirm) |
| M6 | **地图平移** | Pan gesture | 鼠标拖拽 / 方向键平移地图视口（仅当地图超出屏幕时启用） | **是** | — |
| M7 | **缩放指示器** | Icon + label | 右下角显示当前缩放比例，如 `1.0×`（全图模式时变化） | 否 | — |

##### M1: 节点图标 — 视觉规格（🆕 新模式）

| 节点类型 | 像素图标 | 默认尺寸 | 说明 |
|---------|:------:|:------:|------|
| 战斗 (combat) | ⚔ 交叉剑 | 32×32 px | 常规遭遇 |
| 对话 (dialogue) | 💬 气泡 | 32×32 px | NPC 交互 |
| 探索 (exploration) | 🔍 放大镜 | 28×28 px | 发现/搜寻 |
| 谜题 (puzzle) | 🧩 拼图块 | 32×32 px | 谜题/机关 |
| 商人 (merchant) | 🪙 硬币 | 28×28 px | 商店/交易 |
| 休息 (rest) | 🔥 营火 | 28×28 px | 恢复点 |
| Boss | 💀 头骨 | **48×48 px** | Boss 战——最大尺寸，一眼识别 |
| 分支 (branch) | ✦ 星芒 | **40×40 px** | 路线抉择点——第二大尺寸 |

> **尺寸逻辑**：Boss 和分支是节点地图上最重要的两种节点——一个决定"终极挑战在哪"，一个决定"我在哪做不可逆选择"。它们通过放大尺寸建立视觉层级，不依赖颜色传达。

##### M2: 连接线 — 视觉编码（🆕 新模式）

| 路径状态 | 线型 | 颜色/样式 | 含义 |
|---------|------|----------|------|
| 可达 (open) | 实线 | 白色/浅灰，2px | 已揭示、可通行的路径 |
| 隐藏 (hidden) | 虚线 | 深灰，1px | 未揭示的连接（仅在相邻节点可见时显示虚线暗示"前方有路"） |
| 锁定 (locked) | 断裂线 | 深红/暗色，1px，带 🔒 图标在断裂点 | 分支选择后永久封锁的路径 |
| 已走过 (traversed) | 实线 | 灰色，2px | 已被玩家走过的路径 |

## States & Variants

节点地图在不同游戏条件下呈现不同的视觉和行为模式：

| # | 状态 / 变体 | 触发条件 | 视觉变化 | 交互变化 |
|---|------------|---------|---------|---------|
| **D1** | **首次进入** | 冒险开始后首次显示节点地图 | 仅起始节点（Opening）可见为白色 + ▲ 标记；相邻 1-2 跳节点可见；更远区域全黑（未探索）。两条轻量引导提示：▲ 脉冲高亮 3 次 + 图例行淡入高亮 2 秒 | 引导提示在首次交互或 3 秒后自动消失。全功能可用——无强制教程锁定 |
| **D2** | **正常探索中** | 完成 ≥ 1 个节点后返回地图（最常见，占 80%+） | 已访问节点灰色 + 已走到过路径为粗灰线；当前节点 ▲ 标记 + 脉冲光晕；可见节点白色 + 实线连接；未探索区域暗色；锁定节点 🔒 + 断裂线。回访变化以动画呈现（迷雾消散/节点变色/▲ 滑动/路径断裂） | 所有可见节点可点击/聚焦；悬停浮层可用；全图切换可用 |
| **D3** | **全图模式** | 按 `Tab` 或点击 `[全图]` 按钮 | 相机缩小至所有已揭示节点可见（缩放因子自动计算）。节点图标等比缩小。未探索区域仍为暗色。顶栏按钮变为 `[返回]`，提示 `Tab 返回`。右下角缩放指示器显示当前缩放比 | 节点**不可点击出发**——仅用于全景预览。`Tab` / `Esc` / 点击 `[返回]` 回到默认视图。地图平移（M6）在全图模式下也可用 |
| **D4** | **分支确认中** | 玩家点击分支节点（✦）的某个分支方向 | 被选分支高亮（L3 选中态——浅蓝背景 + 边框脉冲）。其他可见分支保持普通可见态。遮罩层 20% alpha 覆盖非分支区域。弹出确认对话框（模式 1）："此选择不可更改，确定要前往 {分支名} 吗？" + [确认]/[取消] 按钮 | 可选分支方向接受点击——点击已选分支再次确认。`Esc` 取消确认恢复到 D2。其他节点不可点击 |
| **D5** | **锁定后** | 分支确认后，未选路径被锁定 | 未选路径播放"关门"动画（0.3s）：连接线断裂 + 🔒 图标覆盖 + 路径方向指示消失。已选路径变为"已走过"粗线。此状态为瞬时动画——播完后自动回到 D2 | 同 D2。锁定节点不可点击，悬停时浮层显示"此路径已封锁" |
| **D6** | **加载/过渡** | 场景切换中（fade_to_black / fade_from_black） | 画面渐黑（进入动画）或渐亮（退出动画）。加载期间不显示任何交互元素。过渡时长：首次进入 0.5s，返回 0.3s | 全部输入屏蔽（GDD §1E.4 E16） |
| **D7** | **错误/降级** | 节点图数据损坏、蓝图验证失败、或降级到离线模板 | 顶栏标题后追加 `[离线模式]` 标签（11px，橙色）。地图仅显示基础拓扑——无 LLM 生成的节点名称/风味文本。悬停浮层的风味文本行替换为 `（叙述生成中...）`。错误严重时显示 Toast 通知（模式 5）："冒险数据加载异常，已使用备用地图" | 交互功能不变——节点仍可点击出发。锁定功能正常（程序层不受影响） |
| **D8** | **平台变体：手柄** | 检测到手柄为当前输入设备 | 按键提示切换为手柄图标：`A=选择 B=返回 Y=全图`。焦点高亮（L2 金色边框 2px）在节点间以方向键导航。全图切换按钮显示 `[全图 Y]`。底部状态条取消鼠标悬停依赖——聚焦节点时自动显示浮层 | 方向键在节点间跳转（基于拓扑连接而非屏幕位置——↑=从当前节点沿向上连接移动）。`A` 确认节点出发或分支选择。`B` 取消/返回。`Y` 全图切换 |
| **D9** | **冒险失败 / 队伍团灭** | 全队 HP 归零或触发不可恢复的死亡条件 | **"熄灭"动画**：所有节点逐个变暗——从当前位置向外扩散，每节点 0.5s，最后整个地图黑屏。顶栏标题替换为 `冒险终结: {冒险名称}`（红色文字）。底部状态条全红闪烁 3 次（500ms 周期） | **全部输入屏蔽**——无任何交互。3 秒后自动过渡至 Settlement（结算）界面 |

### 手柄节点导航规则

手柄方向键在节点间跳转**基于拓扑连接而非屏幕位置**：

| 手柄方向键 | 导航目标 |
|-----------|---------|
| `↑` | 沿当前节点的**上行连接**移动到目标节点。如有多个上行连接，默认选择最左侧 |
| `↓` | 沿当前节点的**下行连接**移动到目标节点。如有多个，默认最左侧 |
| `←` | 在当前节点的**同级/左侧连接**中移动。如在分支节点，←/→ 在分支选项间切换 |
| `→` | 在当前节点的**同级/右侧连接**中移动 |

> **设计原则**：手柄导航基于拓扑关系而非屏幕坐标——因为节点地图的视觉布局（自上而下）与节点的有向图结构（连接方向）天然对齐。基于拓扑的导航确保"按上键 = 向前走"的空间直觉一致。

### 键盘焦点导航规则

| 键盘 | 行为 |
|------|------|
| `Tab` | 焦点在可交互元素间循环：节点图标 → [全图]按钮 → 节点图标 → ... |
| `Shift+Tab` | 反向循环 |
| `← ↑ → ↓` | 当有节点聚焦时：在相连节点间导航。当无节点聚焦时：平移地图 |
| `Enter` | 确认当前聚焦节点的出发（或分支确认） |
| `Esc` | 取消当前操作（关闭浮层/取消分支确认/退出全图模式）。如在 D2 且无任何子状态 → 弹出暂停覆盖层 |

### 不可交互时的输入行为

| 状态 | 输入行为 |
|------|---------|
| D6（加载/过渡） | 全部输入屏蔽——0.2s 延迟后恢复（GDD §1E.4 E16） |

## Accessibility

### 无障碍等级：Standard

基于 `design/accessibility-requirements.md` 的项目级承诺，本屏幕需满足 Standard 等级要求。

### 键盘完整导航路径

所有交互元素均可通过键盘访问（已在 Interaction Map 详细定义）。导航路径总结：

```
Tab ──→ [全图]按钮 ──→ 节点1 ──→ 节点2 ──→ ... ──→ 循环回[全图]
 ↑                        │
 └────── Shift+Tab ────────┘

节点聚焦时: ←↑→↓ 在相连节点间导航
Enter: 确认出发
Esc: 取消/返回
```

✅ 所有交互元素在 Tab 序中，无键盘死角。

### 颜色非依赖性——四状态双重编码

| 节点状态 | 颜色 | 非颜色信号（形状/图标/线型） |
|---------|------|--------------------------|
| **已访问** | 灰色 | 节点图标**仍可见**（不隐藏）+ 已走过路径用粗线 |
| **可见** | 白色/亮色 | 实线连接线 + 节点图标**存在** + 悬停可交互 |
| **未探索** | 暗色/黑色 | 节点图标**完全隐藏**（图标缺失 = 明确信号）+ 虚线连接提示 |
| **锁定** | 暗红/暗色 | 🔒 图标**覆盖** + 断裂连接线 + 悬停浮层文字"此路径已封锁" |

> **审计结论**：四种状态均以**图标存在/不存在**和**线型变化**作为颜色之外的独立信号通道。色盲模式下（红色盲/绿色盲/蓝色盲），四种状态仍可通过图标 + 线型组合区分。满足 WCAG 1.4.1（色彩非唯一区分方式）要求。

### 节点尺度层级——低视力辅助

节点地图通过**三层尺寸**区分节点重要性，不依赖颜色：

| 节点 | 尺寸 | 视觉占比 | 低视力下的可辨性 |
|------|:---:|---------|----------------|
| Boss (💀) | 48×48 px | 最大 | UI 缩放 1.5× 下为 72×72 px——显著大于其他节点 |
| 分支 (✦) | 40×40 px | 中等偏大 | UI 缩放 1.5× 下为 60×60 px——易于区分 |
| 普通节点 | 28-32 px | 基准 | UI 缩放 1.5× 下为 42-48 px |

✅ 即使完全无法区分颜色，节点尺寸层级本身即可传达"大 = Boss/分支 = 重要"的视觉层级。

### 文本可读性

| 文本 | 字号（设计空间） | 1.0× 1080p 实际 | 对比度 | 评估 |
|------|:---:|:---:|------|------|
| 顶栏标题 H1 | 14px | ~21px | 白色 #FFF vs 深色 #0A0A1E = 18.4:1 | ✅ AA/AAA 均通过 |
| 节点名称 M1 | 10px | ~15px | 浅色 #CCC vs 暗背景 = >7:1 | ⚠️ 字号偏小——文字缩放 1.3× 档下为 19.5px 可接受 |
| 悬停浮层标题 | 14px | ~21px | 白色 vs 深色半透明 = >10:1 | ✅ 通过 |
| 悬停浮层正文 | 12px | ~18px | 浅灰 vs 深色 = >5:1 | ✅ AA 通过 |
| 图例栏 L1/L2 | 10px | ~15px | 浅灰 vs 深色 = >7:1 | ⚠️ 非关键信息，小字号可接受 |
| 状态条 S1/S2 | 10px | ~15px | 浅色 vs 暗色 = >4.5:1 | ⚠️ 推荐文字缩放 1.3× 档提升可读性 |

> **建议**：项目级文字缩放默认推荐 1.3× 档（"大"），将 10px 提升至 19.5px、12px 提升至 23.4px，显著改善可读性而不破坏布局。

### 减少动态选项

所有动画在 `reducedMotion=true` 下的行为已在 Transitions 章节标记。汇总：

- 入场/出场动画：缩短至 0-0.1s
- 状态过渡（节点揭示/变色/锁定/位置滑动）：跳过动画——直接切换
- 悬停浮层：即时显示/关闭
- ▲ 脉冲光晕：静止
- 分支确认对话框：即时显示/关闭
- 团灭逐个熄灭动画：跳过 → 直接全黑

✅ 所有动画均有静态替代方案，无闪烁风险（闪烁频率均 ≤ 2Hz）。

### 手柄完全支持

手柄导航已定义（D8 平台变体 + E1-E9 交互映射），满足 Partial 手柄支持要求。方向键节点间导航基于拓扑连接关系而非屏幕坐标。

### 屏幕阅读器

屏幕阅读器支持明确**不在 Standard 等级范围内**（见 accessibility-requirements.md）。Myra UI 框架原生不支持可访问性树。

### 色盲模式兼容性

| 色盲类型 | 设计如何兼容 |
|---------|------------|
| 红色盲 (Protanopia) | 节点状态不依赖红/绿区分；锁定态有 🔒 图标 + 断裂线双重编码 |
| 绿色盲 (Deuteranopia) | 同上——尺寸层级 + 图标 + 线型足以区分所有状态 |
| 蓝色盲 (Tritanopia) | 节点视觉状态的颜色映射在色盲模式下自动替换为高对比度方案（GDD 09 §14.1） |
| 高对比度模式 | 节点边框加粗至 3px；连接线加粗至 4px；所有文本追加 2px 深色描边 |

### 40% 文本膨胀高影响项

| 组件 | 中文 | 德语（膨胀后） | 布局破坏风险 | 缓解措施 |

### 本地化建议

1. **图例栏国际化策略**：建议将图例栏 L1/L2 的文本替换为纯图标 + 色块方案（无文字依赖），通过首次进入时的引导提示传达图标含义。这样完全消除图例栏的翻译约束。
2. **节点名称长度限制**：程序侧对节点名称施加 8 字符上限（中文）或 20 字符上限（拉丁），超出自动截断。
3. **顶栏标题超出处理**：`冒险: ` 前缀固定。标题可用宽度 = 1280 - 100（按钮）- 40（边距）= 1140px，14px 中文约容纳 80 字符，足够绝大多数情况。

## Acceptance Criteria

- [ ] **AC-1 (性能)**: 从 Party Select 确认出发后，节点地图在 0.5s 内完成 fade_from_black 入场动画并展示初始拓扑（起始节点 + 相邻 1-2 跳节点）
- [ ] **AC-2 (性能)**: 从房间探索返回节点地图时，fade_from_black 入场动画在 0.3s 内完成，回访变化动画（节点变色/▲滑动/新节点揭示）在额外 0.5s 内全部完成
- [ ] **AC-3 (导航-进入)**: 从 Party Select 点击「出发冒险」后，成功进入节点地图并显示正确的冒险标题、起始节点和当前位置 ▲ 标记
- [ ] **AC-4 (导航-离开)**: 点击可见节点（非分支）后，节点被选中（缩放反馈）→ 转入 fade_to_black 过渡 → 成功进入对应房间探索场景
- [ ] **AC-5 (核心目的)**: 节点地图正确展示四种视觉状态的节点——已访问（灰色+图标可见）、可见（白色+实线连接）、未探索（暗色+图标隐藏）、锁定（🔒+断裂线），且分支选择后未选路径正确进入锁定态
- [ ] **AC-6 (核心目的)**: 悬停在可见节点上 ≥ 300ms 后，浮层弹出并显示节点名称、类型图标、难度提示和 [前往] 按钮
- [ ] **AC-7 (分支)**: 点击分支节点后弹出确认对话框，再次确认后锁定未选路径（播放断裂动画 + 🔒 覆盖），取消后恢复到正常探索状态
- [ ] **AC-8 (全图)**: 按 `Tab` 后相机缩小至所有已揭示节点可见——节点不可点击出发。再次 `Tab` 返回默认视图
- [ ] **AC-9 (空/首次状态)**: 首次进入冒险时，仅起始节点 + 相邻 1-2 跳节点可见，其余区域为暗色（未探索）。轻量引导提示（▲ 脉冲 + 图例行高亮）在 3s 后或首次交互后自动消失
- [ ] **AC-10 (团灭状态)**: 触发 D9 团灭后，所有节点逐个熄灭 → 全黑 → 标题变为红色"冒险终结" → 底部状态条全红闪烁 → 3s 后自动转入 Settlement 界面
- [ ] **AC-11 (无障碍-键盘)**: 所有交互元素可通过 `Tab` / `Shift+Tab` 循环聚焦，节点间可通过 `← ↑ → ↓` 导航，`Enter` 确认出发，`Esc` 取消/返回
- [ ] **AC-12 (无障碍-手柄)**: 手柄方向键可在节点间基于拓扑连接关系导航，`A` 确认出发，`B` 返回，`Y` 全图切换
- [ ] **AC-13 (无障碍-颜色)**: 四种节点视觉状态在色盲模拟（红色盲/绿色盲/蓝色盲）下均可通过图标存在/不存在、线型（实线/虚线/断裂）、🔒 图标等非颜色信号独立区分
- [ ] **AC-14 (错误/降级)**: LLM 生成失败或蓝图数据异常时，地图仍正常展示基础拓扑——顶栏追加 `[离线模式]` 标签，悬停浮层风味文本替换为 `（叙述生成中...）`，节点出发功能不受影响
