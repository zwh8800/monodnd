## Purpose & Player Need

战斗界面是《酒馆与命运》**赌注最高的游戏空间**——每一个决策都可能改变角色的命运。它不是一个"打怪界面"，而是一个**信息透明且充满紧张感的战术棋盘**。

### 一句话定义

> "玩家来到战斗界面，想要的是**在充分信息下做出有重量的战术决策——知道每一刀为什么命中、为什么失误，感受到死亡的真实逼近，并在骰子落地的瞬间体验到 DND 的灵魂。**"

### 五种核心需求

| 玩家需求 | 战斗界面如何满足 | 设计承诺（锚定来源） |
|----------|-----------------|---------------------|
| ⚔️ **看清局面** | 网格战场 + 先攻顺序条 + 角色状态栏 + 条件图标叠加。所有战术信息按 3 层信息层级渐进揭示（hud.md §3） | P2 "清晰即是沉浸"——0.3 秒态势认知，2 秒内做出有意义决策 |
| 🎲 **理解因果** | d20 骰子动画展示完整计算链（原始值→加值→总值 vs AC/DC），战斗日志记录每一步判定（GDD 04 §1A.2 信息测试） | 玩家从不猜测"为什么打不中"——每个结果可从战斗日志完整追溯 |
| 💀 **感受重量** | 双轨死亡系统（3 轮救援窗口 + 死亡失败累积）、HP 颜色阶梯（绿→黄→红→灰）、濒死角色视觉衰减（精灵降饱/光晕缩小） | P1+P3——死亡不是"重开一局"，角色的伤疤和倒下必须有可感知的视觉重量 |
| 🧠 **不被淹没** | 抗性/免疫/易伤自动计算（显示最终伤害+图标），优劣势自动汇总（⬆/⬇ 骰子标识），专注检定自动执行仅失败时通知（GDD 04 §1.X） | 保留 DND 全部规则深度，但活跃认知层从 >10 降至 6-7 |
| ⚡ **保持警醒** | 先攻每轮重掷、反应中断机制、回合切换横幅、HP 低值闪烁+心跳音效（hud.md §8-9） | 战斗不是"按最优 Build 自动运转"，而是每一轮都在做有风险的选择 |

### 如果战斗界面不存在或设计失败

- **信息不可见或不可追溯** → 玩家感到"在瞎按"，策略退化，**"DND 的灵魂"丧失**
- **死亡无视觉重量** → 角色沦为可替换资源，**Pillar 1（角色驱动的故事）情感绑定断裂**
- **认知过载** → 10+ 项同时活跃的决策变量使回合制变成脑力消耗而非策略享受
- **节奏拖沓** → 单场遭遇超过 7 轮未结束，战术深度变成认知负担，玩家寻求跳过

### 设计参考对齐

| 来源 | 已确立的约束 |
|------|------------|
| HUD 规范 §1 | 信息可读性优先于装饰性；回合制结构允许分层展示；键盘可达 |
| Art Bible §2.3 | 战斗 = 专注+紧绷；冷色网格 + 暖色战术高亮；P2 在此处最高浓度表达 |
| GDD 04 §1.X | 认知负荷控制——所有 DND 规则完整执行，计算过程透明但被动化 |
| GDD 04 §1A.2 | 五重设计测试（信息/紧张感/死亡临近/构筑验证/节奏）——所有 UX 决策通过测试方可通过 |
| Accessibility §1 | Standard 等级——色彩双编码、字体缩放、输入重映射、色盲模式 4 种 |

## Player Context on Arrival

### 核心设计：同图战斗

战斗**不切换场景**——它与探索共享同一张大地图。进入战斗时，地图不重新加载；只是在当前区域周围施加边界限制，战斗 HUD 叠加到地图视图之上。战斗结束，边界解除，HUD 收起，玩家在同一位置无缝回到探索。

### 触发方式

| 路径 | 触发条件 | 到达时玩家状态 |
|------|---------|--------------|
| **可视遭遇** | 大地图上发现敌人，玩家主动靠近触发 | 有心理准备——敌人已在地图上可见，玩家可控制接敌角度和距离 |
| **伏击触发** | 敌人先于玩家察觉并发起攻击（潜行/隐藏敌人、陷阱触发） | 措手不及——可能面临突袭轮、劣势开局 |

### 到达时的情绪与认知状态

| 维度 | 设计预期 |
|------|---------|
| **心境** | 警觉 → 专注 → 紧张——三级递进，从"发现敌人"到"进入战斗范围"短暂酝酿 |
| **认知负荷** | 态势评估在战斗边界锁定时完成：敌人数、位置、地形——对应 HUD §3 的 0.3 秒态势认知层 |
| **时间压力** | 回合制由玩家掌握节奏，伏击路径紧迫感天然更高 |
| **自愿性** | 可视遭遇时半主动（玩家选择靠近）；伏击时非自愿 |
| **空间连续性** | 玩家知道自己在哪里——战斗区域就是刚才探索的那块地形，方向感不中断 |

### 进入战斗前玩家刚刚在做什么

- **可视遭遇**：在大地图上探索移动，发现视野内的敌人 → 主动进入战斗范围 → 边界锁定 + HUD 展开
- **伏击**：在大地图上探索中，敌人从视野外/隐藏状态发起攻击 → 边界锁定 + HUD 展开（突袭判定）
- **携带的包袱**：队伍 HP/资源/现有状态条件在进入战斗时保持不变——无需"过渡加载"，状态直接延续

### 多次进入的情景

- **同一次探险中多场战斗**：战斗结束 → 边界解除 → 在同一地图上继续探索 → 再次遭遇 → 状态递减（资源消耗累积），决定继续推进还是撤退
- **重试**：团灭后从酒馆重新出发，再次到达同一区域——Roguelike 重置，地形可能变化

## Navigation Position

战斗不是独立屏幕——它是探索态的**约束叠加**。在游戏整体的屏幕层级中：

```
游戏整体
  ├── 酒馆 (Tavern Main) — 独立场景，唯一的安全锚点
  │   ├── 名册覆盖层
  │   ├── 英雄覆盖层
  │   └── ...
  │
  └── 冒险 / 大地图 (Exploration) — 独立场景
      ├── 探索态 (默认) — 自由移动、互动
      ├── 战斗态 ★ — 边界锁定 + HUD 叠层
      └── 死亡/撤退 → 回到酒馆
```

| 属性 | 值 |
|------|-----|
| **所在场景** | 冒险 / 大地图（与探索共享） |
| **进入方式** | 遭遇触发——自动，非玩家主动导航 |
| **退出方式** | 胜利 / 撤退 / 团灭 → 回到同场景探索态 或 → 回到酒馆 |
| **是否可直达** | 否——战斗只能从探索态被动触发，不可从酒馆或菜单直接"跳转"到战斗 |
| **暂停菜单可达** | 是——Start 键可随时暂停（暂停覆盖层位于战斗 HUD 之上） |
| **与其他屏幕的关系** | 战斗中不可访问酒馆覆盖层（名册/英雄等）：酒馆专属功能仅在酒馆场景可用 |

### 设计含义

- **无"返回"按钮**——战斗没有返回上一屏幕的概念。退出战斗只有三种结局：赢、跑、死
- **暂停优先**——Start 暂停是战斗态唯一的外部"出口"，但它不离开战斗，只是临时覆盖
- **胜利/失败后的流向**——不由 UI 导航控制，由游戏逻辑决定（胜利→边界解除继续探索；撤退→回到探索态但可能承受代价；团灭→回到酒馆）

## Entry & Exit Points

### 入口

| 入口来源 | 触发器 | 玩家携带的上下文 |
|----------|--------|-----------------|
| 大地图探索态 — 可视遭遇 | 玩家主动进入敌方触及范围；或敌方 AI 检测到玩家后进入警戒 → 触发战斗 | 当前队伍状态（HP/资源/条件）、当前地形位置、队伍阵型 |
| 大地图探索态 — 伏击 | 隐藏敌人/陷阱触发 → 立即锁定边界 + 突袭判定 | 当前队伍状态、伏击方可能获得突袭轮 |

### 出口

| 出口目的地 | 触发器 | 备注 |
|-----------|--------|------|
| 大地图探索态 — 胜利 | 所有敌方单位被击败（HP=0 或逃跑） | 边界解除、HUD 收起、在同一位置无缝回到探索。可能触发：战利品弹窗、XP 获得通知、战斗统计摘要 |
| 大地图探索态 — 撤退 | 玩家主动执行撤退动作并成功（撤退判定） | 边界解除、HUD 收起。可能承受撤退代价（追击伤害/资源惩罚）。撤退动作需在行动菜单中可选择 |
| 酒馆 — 团灭 | 所有队员 HP=0 且未能救援（死亡双轨制裁决） | 场景过渡到酒馆（`fade_black`）。触发失败/成长系统的结算。此出口不可逆——本场战斗数据丢失 |

### 非出口（叠层而非离开）

| 叠层 | 触发器 | 行为 |
|------|--------|------|
| 暂停菜单 | Start 键 | 暂停覆盖层叠加在战斗 HUD 之上。战斗状态冻结。关闭暂停后回到战斗 |
| 角色详情 | Select/View 键 或 Tab | 浮动窗口显示角色全属性/装备/条件。不离开战斗。关闭后回到原焦点

## Layout Specification

> **视觉投影约束**：战斗场景与探索共享同一张地图，必须遵循 [Art Bible §3.0](../art/art-bible.md#30-摄像机与透视正交投影铁律) 正交投影铁律——战斗网格为平面正方形（非透视梯形）、所有角色精灵和图块保持固定像素尺寸、深度仅通过 Z-order 层叠遮挡表达、无消失点无近大远小。战斗背景的"暗角"效果通过 §4.3 的色彩饱和度/亮度压制实现，不通过 3D 景深模糊。**严格禁止**：3D 透视地板网格、角色按距离缩放、摄像机绕场景旋转。
