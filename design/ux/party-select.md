> **Journey Phase(s)**: 酒馆休整 → 组队确认 → 冒险出发（场景切换中转站）
> **Template**: UX Spec

## 1. Purpose & Player Need

**Party Select 界面是冒险出发前的"最后确认站"——玩家在这里完成三件事：确认要去的冒险、选择出战的队伍、在出发前做最后的战术检查。**

### 一句话定义

> "玩家来到队伍选择界面，想要的是**清晰地看到即将面对的冒险、自信地选出 4 人队伍、并在出发前确认一切就绪。**"

### 四种核心需求

| 需求 | Party Select 如何满足 | 设计承诺 |
|------|----------------------|---------|
| 🎯 **冒险理解** | 任务名 + 9 字段完整预览（层级/推荐等级/金币经验/节点数/危险构成/预计时长/环境类型） | 玩家不需要凭记忆——一眼看清"我要去哪、有多难" |
| ⚔️ **战术组队** | 已选队伍区（4 槽）+ 可用角色列表（带状态/HP/职业）+ ⚠️/⚡/✅ 队伍构成评估 | 玩家能快速对比角色状态，搭配均衡队伍 |
| 🔍 **最后检查** | 只读角色卡片（点击角色弹出）+ 装备概览 + 技能一览 | 玩家可以"检查但不修改"——确认角色状态，如需调整则返回酒馆 |
| 🚪 **自由退出** | 双路径逃生舱：任务板进入→可放弃任务返回；酒馆进入→可返回酒馆 | 玩家不会被锁死——任何时刻都能安全退出 |

### 如果 Party Select 界面不存在或难用

- 玩家选择队伍时看不到冒险信息 → **盲目组队——带错了职业/没带治疗，出发后才发现**
- 无法检查角色装备/技能 → **发现法师没装备法杖时，已经进入冒险无法回头**
- 逃生路径不清晰 → **玩家感到"被锁住了"，焦虑 → 流失**
- 队伍构成没有任何提示 → **新手频繁因"没带前排/没带治疗"团灭，愤怒退出**

### 设计哲学核心

Party Select 遵循两条交互铁律：

> **"信息充分但不溢出"** — 9 字段预览提供所有战术决策所需信息，但绝不在此界面堆砌 LLM 风味文本或冗长描述。此刻玩家需要的战术情报，不是沉浸感。

> **"只读检查 + 安全退出"** — 这是一个确认站，不是配置站。玩家只能"看"不能"改"——需要调整则返回酒馆。这制造了出发行为的分量感："我确认一切就绪，按下出发。"

## 2. Player Context on Arrival

Party Select 根据玩家"从哪来"做出差异化响应——两条进入路径携带不同的情绪和意图。

| # | 到达路径 | 触发条件 | 玩家刚才在做什么 | 情绪状态 | 设计响应 |
|---|---------|----------|-----------------|----------|---------|
| ① | **从任务板进入** | 任务详情 →「选择队伍」 | 刚阅读完任务详情，已决定接这个任务 | 🎯 目标明确 + "我要带谁去？" | 任务信息预填充（锁定任务名/预览），焦点落在**可用角色列表**——玩家下一步是选人。任务栏显示任务名 +「放弃此任务，返回任务板」逃生链接 |
| ② | **从酒馆「出发冒险」进入** | 酒馆主屏 →「出发冒险」 | 在酒馆闲逛，想出发但还没决定去哪 | 🤔 在探索 — "先看看能去哪，再想带谁" | 任务区显示「选择任务」引导入口，焦点同时落在任务选择区和可用角色列表。底部「返回酒馆」按钮可见 |
| ③ | **从 Party Select 内部返回** | 角色详情卡片关闭后 | 刚检查过某个角色的装备/技能 | 😐 中性 — 刚获得信息，继续组队决策 | Party Select 恢复卡片打开前的状态：选中角色保留、滚动位置保留、评估徽章刷新（若检查的角色信息触发了评估变化则更新） |

### 首次进入 vs 回访

- **首次进入（新玩家）**：若名册中恰好有 ≥ 4 名角色且为首次使用 Party Select，在已选队伍区首槽显示脉冲高亮引导（`anim_speed` 控制），提示"点击空槽添加角色"。脉冲 3 次后自动消失。
- **回访（老玩家）**：无引导。上次选择的队伍构成记忆保留（若角色仍在名册中且状态为"就绪"，自动填充上次选择）。

## 3. Navigation Position

Party Select 在游戏导航树中的位置：

```
TAVERN_MAIN（酒馆主屏幕）
    │
    ├──→ QUEST_BOARD（任务板覆盖层）
    │       └──→ QUEST_DETAIL（任务详情覆盖层）
│               └──→ PARTY_SELECT ──→ ADVENTURE_START
│                                        │
│                                        ▼
│                                   探索场景（探索阶段）
│
└──→ PARTY_SELECT（直达）
        │
        └──→ ADVENTURE_START → 探索阶段
```

**关键定位**：Party Select 是覆盖层栈的**最深层节点**——它上面可能叠了角色详情卡片（只读），但它本身不能触发更深层的操作覆盖层（不可在此打开背包、商店等）。它是"确认行为"的最后一站：从这里出去只有两个方向——**前进（出发）或后退（返回来源）**。

## 4. Entry & Exit Points

### 4.1 进入路径

| # | 来源屏幕 | 触发方式 | 携带上下文 | 界面初始状态 |
|---|---------|---------|-----------|-------------|
| ① | **任务板 → 任务详情** | 点击「选择队伍」按钮（任务详情底部） | 任务已锁定（quest_id）。冒险名、等级、预览数据全部可读 | 任务区预填充该任务信息。「放弃此任务」链接可见。焦点落在"可用角色列表"第一个可聚焦角色 |
| ② | **酒馆主屏幕** | 点击「出发冒险」按钮（底部操作栏）或按 `E` | 名册中所有状态为"就绪/轻伤"的角色。无锁定任务 | 任务区显示"选择任务"引导入口 + 任务板快捷链接。角色列表中所有可用角色列出 |

### 4.2 离开路径

| # | 目的地 | 触发方式 | 动画 | 携带结果 | 备注 |
|---|--------|---------|:---:|---------|------|
| 1 | **冒险场景** | 「出发冒险！」按钮（按住 0.2s 防误触）| `fade_black` 0.3s | `{ party: [4×char_id], quest_id, adventure_instance_id }` → 发布 `adventure_departed` 事件 | ⚠️ **不可逆**。GDD 定义此动画之后不可 Esc 返回酒馆。若从任务板进入（路径①），任务详情覆盖层先关闭再执行场景切换 |
| 2 | **酒馆主屏幕** | 「返回酒馆」按钮 / `Esc` / 手柄 `B` / 点击遮罩 | `overlay_pop` 逆向缩回 (0.15s) | 无状态变更。队伍选择不保存（下次进入重置）。若从路径①进入（上面有任务详情覆盖层），一并关闭所有上层覆盖层 | 🔒 安全操作——无确认对话框。玩家可随时返回酒馆重新准备 |

> **单一出口原则**：两条进入路径共享相同的两个出口——出发或返回。路径①来时任务已锁，「放弃此任务」链接作为辅助逃生舱（见 §4.3）。不在覆盖层内增加"更换任务"按钮——保持界面简洁。

### 4.3 「放弃此任务」逃生舱（仅路径①）

从任务板进入（路径①）时，任务栏上方显示「← 放弃此任务，返回任务板」文本链接：

| 触发 | 行为 |
|------|------|
| 点击「放弃此任务」 | 关闭 Party Select + 任务详情覆盖层 → 回到任务板覆盖层。任务取消，玩家可重新浏览任务 |
| 确认 | 无确认对话框——此操作为"再看看其他任务"，非危险操作 |

## 5. Layout Specification

### 5.1 信息层级（Information Hierarchy）

优先级自上而下——对应玩家决策的自然顺序：

| 优先层 | 信息 | 理由 | 视觉权重 |
|:------:|------|------|:------:|
| **P0** | 冒险预览（任务名 + 9 字段） | 玩家需要先知道"去哪"才能决定"带谁" | 顶部面板，半透明深色背景板 + 任务名使用 `BASE_FONT_TITLE` 字号 |
| **P1** | 已选队伍（4 槽） | 这是界面的核心交互——选择谁 | 居中，4 槽等宽排列，空槽显示脉冲引导 |
| **P2** | 可用角色列表 | 从中挑选成员填入 P1 的槽位 | 列表区域，带状态标签（就绪/轻伤/重伤/力竭） |
| **P3** | 队伍构成评估（⚠️/⚡/✅） | 战术建议——帮助但不阻止 | 已选队伍下方，徽章 + 一行提示文字，不阻止操作 |
| **P4** | 行动按钮 | 确认/取消 | 底部栏，「出发冒险!」使用强调色（暖金）；返回按钮为次要色 |

### 5.2 布局分区（Layout Zones）

采用**三列仪表盘布局**，在 1280×720 (16:9) 基准分辨率下，覆盖层窗口约 1080×576 px（占视口 ~85%），分为五个区域——左侧角色浏览、中央队伍构建、右侧冒险参考。

```
┌──────────────────────────────────────────────────────────────────┐
│  Zone T: 标题栏（固定，h=36px）                                    │
├───────────┬────────────────────────┬─────────────────────────────┤
│ Zone L    │  Zone C                │  Zone R                     │
│ 可用角色   │  已选队伍               │  冒险预览                    │
│ 280px     │  440px                 │  360px                      │
│ (可滚动)  │  (2×2 网格)            │  (双列紧凑网格)              │
├───────────┴────────────────────────┴─────────────────────────────┤
│  Zone A: 操作栏（固定，h=48px）                                    │
└──────────────────────────────────────────────────────────────────┘
```

| 区域 | 宽度 | 高度 | 内容 | 间距 |
|------|:---:|:---:|------|:---:|
| Zone T | 1080px | 36px | 窗口标题 + X 关闭按钮 +（路径①）放弃任务链接 | — |
| Zone L | 280px | 492px | 可用角色列表（虚拟化滚动）。每行：头像 32×32 + 名字 + HP 条 + 状态标签 | 与 C 间距 12px |
| Zone C | 440px | 492px | 已选队伍 4 槽位（2×2 网格，间距 12px）+ 队伍评估 | 与 R 间距 12px |
| Zone R | 360px | 492px | 冒险预览（9 字段双列网格） | — |
| Zone A | 1080px | 48px | 返回按钮（左） + 出发按钮（右） | — |

**分区焦点顺序**（Tab / LB-RB）：Zone L → Zone C → Zone R → Zone A → 循环回 Zone L。

**双路径差异**：路径①（任务板进入）时 Zone R 预填充已锁定任务数据；路径②（酒馆进入）时 Zone R 显示「选择任务」引导入口——点击后弹出任务选择子覆盖层（保持 Party Select 上下文不丢失）。

## 6. States & Variants

### 6.1 双路径状态差异

| 界面元素 | 路径①（任务板进入） | 路径②（酒馆进入） |
|---------|-------------------|-------------------|
| Zone R 冒险预览 | 任务名 + 9 字段已填充；Zone T 顶部显示「← 放弃此任务」文字链接 | 显示「🏷️ 选择任务」引导按钮；9 字段均显示"—" |

### 6.2 边界状态

| # | 状态 | 触发条件 | 界面行为 |
|---|------|---------|---------|
| **S1** | 名册为空 | 角色名册中无任何角色 | 已选队伍区显示空槽（不可点击）。可用角色列表显示「暂无可用角色。回酒馆招募冒险者吧。」提示文字。`btn_depart` 灰显。路径②的任务选择引导按钮仍可用 |
| **S2** | 不足 4 人 | 名册中可用角色（就绪+轻伤）< 4 人 | 已选队伍区空槽可点击但角色列表不足。列表底部显示「⚠️ 可用角色不足 4 人，无法出发」提示。`btn_depart` 灰显 |
| **S3** | 全部重伤/力竭 | 所有角色 HP < 50% | 角色列表全部标记重伤/力竭状态。队伍评估显示「⚡ 所有角色状态不佳，建议休息后再出发」。`btn_depart` 可点击但不建议——允许玩家强行出发（尊重玩家选择） |
| **S4** | 触发出发条件 | 队伍=4人 + 任务已选 | `btn_depart` 从灰显→暖金色活跃态，带微呼吸动画（亮度 pulse ±5%，循环 1.5s）。手柄自动聚焦 `btn_depart` |
| **S5** | 「出发冒险!」被按下 | 玩家按下 `btn_depart` | 按钮变为按下态（scale 0.95）+ 0.2s 按住确认（防误触）。0.2s 后触发 `fade_black`。若 0.2s 内松开，动画回弹，不出发 |
| **S6** | 角色卡片打开中 | 点击已选/可用角色 | Party Select 内容区略微暗化（alpha 降至 60%），仅卡片区域完全可见。详见 §7.3 |
| **S7** | 空槽「+ 添加」脉冲 | 首次使用 Party Select 且未选满 4 人 | 第一个空槽边框脉冲高亮（FF6 风格：亮度 100%→60%→100%，周期 1s），脉冲 3 次后自动消失。任何用户交互（点击/按键）立即停止脉冲 |

## 7. Interaction Map

### 7.1 键盘交互

| 按键 | 上下文 | 行为 |
|------|--------|------|
| `↑` / `↓` | Zone L 角色列表聚焦 | 上下移动列表高亮行；自动滚动列表使聚焦行可见 |
| `↑` / `↓` / `←` / `→` | Zone C 队伍槽位聚焦 | 在 2×2 网格内按方向移动焦点——`←→` 切换同行槽位，`↑↓` 切换同列槽位。从 [0,0] 按 `↓` → [1,0]；从 [0,1] 按 `←` → [0,0] |
| `Enter` / `Space` | Zone L 角色列表聚焦某角色 | 将角色添加到第一个空槽（若角色已入选则无操作；若队伍已满 4 人则闪烁提示） |
| `Enter` | Zone C 队伍槽聚焦（满槽） | 打开该槽角色的只读卡片（§7.3） |
| `Enter` | Zone C 队伍槽聚焦（空槽） | 焦点跳回 Zone L 角色列表（等同于"去选人"） |
| `Delete` / `Backspace` | Zone C 队伍槽聚焦（满槽） | 从队伍中移除该角色，槽位恢复空状态 |
| `Esc` | 无卡片打开 | 关闭覆盖层，返回酒馆（路径②）或放弃任务返回任务板（路径①） |
| `Esc` | 角色卡片打开中 | 关闭卡片，返回 Party Select（保留队伍状态） |
| `Tab` | Party Select 主界面 | 在三列 + 操作栏之间循环切换焦点分区：**Zone L → Zone C → Zone R → Zone A → Zone L**。`Shift+Tab` 反向循环 |
| `1-4` | Party Select 主界面 | 快速定位到第 1-4 个队伍槽。槽位在 2×2 网格中的编号：`1`=左上 [0,0]，`2`=右上 [0,1]，`3`=左下 [1,0]，`4`=右下 [1,1]。若槽满→打开角色卡片；若槽空→焦点跳回 Zone L |
| `E` | `btn_depart` 激活时 | 触发出发（等效点击「出发冒险!」——含 0.2s 防误触） |

### 7.2 手柄交互

| 手柄按键 | 上下文 | 行为 |
|---------|--------|------|
| `D-Pad ↑/↓` | Zone L 角色列表聚焦 | 同键盘 `↑/↓` |
| `D-Pad ←/→/↑/↓` | Zone C 队伍槽位聚焦 | 在 2×2 网格内四向移动焦点（同键盘） |
| `A` | 同 `Enter` | 添加角色 / 打开卡片 / 触发出发 |
| `X` | Zone C 队伍槽聚焦（满槽） | 移除角色（等效 `Delete`） |
| `Y` | Zone L 角色列表聚焦某角色 | 打开该角色只读卡片（不添加到队伍，仅检查） |
| `B` | 同 `Esc` | 返回 / 关闭卡片 |
| `LB` / `RB` | Party Select 主界面 | 在焦点分区之间循环：**Zone L → Zone C → Zone R → Zone A**。`LB` 正向，`RB` 反向 |
| `Start` | 全局 | 打开设置覆盖层（保留 Party Select 状态） |

## 8. Events Fired

通过 `IEventBus` 发布，其他系统可订阅：

| 事件名 | 触发时机 | 载荷 | 订阅方（预期） |
|--------|---------|------|---------------|
| `party_select_opened` | Party Select 覆盖层完全打开（`overlay_pop` 动画结束） | `{ entry_path: "quest_board" \| "tavern" }` | 分析系统（追踪玩家行为路径） |
| `party_member_added` | 角色被添加到已选队伍槽 | `{ character_id, slot_index, party_size }` | 音频系统（播放"加入队伍"音效）；队伍评估刷新 |
| `party_member_removed` | 角色从已选队伍槽移除 | `{ character_id, slot_index, party_size }` | 队伍评估刷新 |
| `party_composition_changed` | 队伍构成变化（增/删后） | `{ party_ids: [4×(char_id\|null)], assessment: { level: "warning"\|"caution"\|"ok", missing_roles: [...] } }` | UI（更新评估图标和文字） |
| `character_card_opened` | 角色只读卡片弹出 | `{ character_id }` | 分析系统 |
| `character_card_closed` | 角色只读卡片关闭 | `{ character_id }` | — |
| `adventure_departed` | 「出发冒险!」按下并完成 0.2s 确认 | `{ party: [4×char_id], quest_id }` | 冒险系统（实例化冒险）；存档系统（触发自动存档）；音频系统（出发 BGM 切换） |
| `quest_selected_in_party` | 路径②中玩家在 Party Select 内选择了任务 | `{ quest_id }` | 冒险预览面板刷新 |

## 9. Transitions & Animations

所有动画遵循 GDD §1D.4 公式。以下为 Party Select 专属动画：

| 动画 | 类型 | 时长 | 缓动 | 说明 |
|------|------|:---:|------|------|
| **覆盖层打开** | `overlay_pop` | 0.15s | EaseOutBack | 从酒馆主屏幕/任务详情弹出 Party Select，从中心缩放 80%→100% + alpha 0→1 |
| **覆盖层关闭** | `overlay_close` | 0.1s | EaseIn | 缩放 100%→90% + alpha 1→0，同时关闭 |
| **角色卡片打开** | `overlay_pop` | 0.15s | EaseOutBack | 从点击的角色槽位置缩放弹出到全尺寸（局部原点，非屏幕中心） |
| **角色卡片关闭** | `overlay_close` | 0.1s | EaseIn | 缩回到触发位置 |
| **角色添加入队** | 槽位动画 | 0.2s | EaseOut | 头像从角色列表行位置"飞入"槽位。同时槽位短暂放大 1.0→1.1→1.0 |
| **角色移除出队** | 槽位动画 | 0.15s | EaseIn | 头像缩小 + alpha→0。槽位恢复空状态 |
| **空槽脉冲引导** | 循环 | 1.0s/周期 | Linear | 边框亮度 100%→60%→100%。循环 3 次，首次交互打断 |
| **出发按钮激活** | 呼吸循环 | 1.5s/周期 | EaseInOut | 暖金色背景亮度 ±5%。队伍=4人+任务已选时启动 |
| **出发按钮按下** | 确认状态 | 0.2s | Linear | Scale 0.95，保持 0.2s 防误触。完成 0.2s 后触发 `fade_black` |
| **出发场景切换** | `fade_black` | 0.3s | Linear | 画面渐黑，进入 ADVENTURE_START。不可逆 |
| **评估图标切换** | 即时 + 颜色过渡 | 0.3s | EaseOut | ⚠️→⚡→✅ 切换时图标带 0.3s 颜色渐变过渡（红→黄→绿），非瞬间跳变 |

## 10. Data Requirements

### 10.1 界面加载所需数据

| 数据 | 来源 | 类型 | 说明 |
|------|------|------|------|
| `quest_id` | 进入时传递（路径①）/ 玩家选择（路径②） | `string` | 当前选中的任务 ID。路径①预填充，路径②初始为 `null` |
| `quest_preview` | `IWorldStateManager` → 任务数据 | `QuestPreview` record | 9 字段完整预览：任务名、层级、推荐等级、金币、经验、节点数、危险构成（`string[]` 标签）、预计时长（分钟）、环境类型 |
| `roster` | `ICharacterManager` → 角色名册 | `List<CharacterSummary>` | 所有角色摘要：ID、名字、职业、Lv、HP 当前/最大、状态枚举（就绪/轻伤/重伤/力竭/任务中）、头像引用 |
| `party_memory` | 本地缓存 / 存档数据 | `string[4]?` | 上次选择的队伍（角色 ID 数组，最多 4）。仅保留在内存中，不跨存档持久化 |
| `character_detail` | `ICharacterManager` → 获取角色 | `CharacterDetail` record | 按需加载（角色卡片打开时）：完整属性、装备列表、技能列表 |

### 10.2 依赖接口

- `IWorldStateManager.GetAvailableQuests()` — 路径②中获取可选任务列表
- `ICharacterManager.GetRoster()` — 获取名册角色摘要
- `ICharacterManager.GetCharacterDetail(character_id)` — 获取角色完整详情（卡片用）
- `IEventBus` — 发布 §8 中定义的所有事件

### 10.3 不依赖的数据

- ❌ 背包/物品数据（此界面不访问）— 角色卡片中的装备信息来自角色数据，非背包
- ❌ LLM Gateway（不在此界面生成文本）
- ❌ 音频资源清单（由音频系统自行管理）

| # | 要求 | 状态 | 实现方式 |
|---|------|:----:|---------|

| **A7** | UI 缩放 | ✅ | 遵循 GDD §1.3 响应式缩放策略（0.8×, 1.0×, 1.2×），所有坐标像素对齐 |

| **A9** | 防误触 | ✅ | 出发按钮 0.2s 按住确认。`Esc` 返回无确认对话框（安全操作）。角色卡片无修改入口（只读） |

### 色盲模式额外调整

在色盲模式下：
- 状态标签用符号完全替代颜色依赖（◉/◔/◑/◕ 替换为纯符号 + 文字标签，去除颜色背景）
- 队伍评估图标 ⚠️/⚡/✅ 保留符号本身（Unicode 字符），不依赖颜色区分
- HP 条增加数字叠加显示（HP: 28/32），不依赖红/绿色条判断

## 12. Localization Considerations

| # | 考虑项 | 处理方式 |
|---|--------|---------|
| **L1** | 角色名字 | 所有角色名字为简体中文（如"索林·铁锤"）。若未来支持其他语言，名字保留原文或提供音译映射表 |
| **L2** | 职业名称 | 使用标准 DND 职业中译（战士/法师/牧师/盗贼/游侠/诗人等）。字符串表中维护 |
| **L3** | 状态标签 | "就绪"/"轻伤"/"重伤"/"力竭"/"任务中"——短文字，长度 ≤ 3 字符。适合用宽度固定的标签 UI 组件 |
| **L4** | 队伍评估文字 | 评估文字为完整句子（如"缺少治疗者！建议至少1名牧师/德鲁伊"）。需支持多语言字符串表。评估文字最大长度限制为 40 字符（中文）以避免溢出 |
| **L5** | 冒险预览字段标签 | "节点数"/"危险构成"/"预计时长"/"环境类型"——标准 UI 标签，字符串表维护 |
| **L6** | 环境类型值 | "地下城"/"森林"/"城市"/"沼泽"/"山脉"/"海岸"——枚举值中译。字符串表维护 |
| **L7** | 字体 | 所有界面文字使用 FontStashSharp + Noto Sans CJK（见 GDD §1.2.1）。Latin 数字/符号使用等宽字体 |
| **L8** | 日期/时间格式 | "预计时长"以分钟为单位显示（如"45 分钟"），不涉及日历日期格式 |
| **L9** | 文本方向 | 默认 LTR。若未来支持 RTL 语言（阿拉伯语/希伯来语），布局需镜像——Zone B 左右分栏交换、已选队伍槽排列方向反转 |

## 13. Acceptance Criteria

QA 测试人员必须能逐一验证以下通过条件：

| # | 验收标准 | 验证方法 |
|---|---------|---------|
| **AC1** | 从任务板（任务详情→「选择队伍」）进入 Party Select 时，任务信息已预填充且 Zone T 顶部显示「放弃此任务」链接 | 任务板→选择任务→任务详情→「选择队伍」。验证 Zone R 显示任务名+9字段，「放弃此任务」链接可见 |

| **AC3** | 点击「放弃此任务」正确关闭 Party Select 和任务详情，返回任务板 | 路径①进入→点击「放弃此任务」。验证返回任务板覆盖层，任务板状态正常，无任务锁定 |
