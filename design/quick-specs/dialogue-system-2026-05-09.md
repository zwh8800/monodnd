# Quick Design Spec: 对话系统 (Dialogue System)

**Type**: New Small System
**Scope**: 管理玩家与 NPC 之间的对话交互——展示叙事文本、呈现选择选项、记录对话历史和后果。对话系统是 LLM 生成的故事与玩家之间的唯一界面窗口。
**Date**: 2026-05-09
**Design Source**: 提取自 `07-tavern-system.md` §8（对话系统架构）、`09-ui-ux-design.md` §10（对话界面）、`02-llm-integration.md`（LLM 对话生成管线）
**Estimated Implementation**: 已部分设计，本 spec 整合并定义独立系统边界

---

## Overview

对话系统是 Pillar 1（角色驱动的故事）的**交付窗口**。LLM 生成了"兽人对你威胁"、"酒馆老板分享情报"、"队友在篝火旁吐露心声"——对话系统把这些叙事变成玩家**看得见、可选择、有后果**的交互。

系统做三件事：
1. **展示**：在像素风窗口中逐字显示 NPC 的对话文本（打字机效果）
2. **选择**：呈现 2-4 个选项让玩家选择如何回应——有些选项需要技能检定
3. **记录**：记住玩家与每个 NPC 说过什么、选过什么、关系如何变化

MVP 阶段对话发生在两个场景中：
- **酒馆**：与酒馆老板、招募的冒险者、来访 NPC 对话
- **冒险中**：与遭遇的 NPC（商人/俘虏/敌人）对话

## Core Rules

### 1. 对话结构

对话以**有向图**形式组织——NPC 说一句话是一个"节点"，玩家选择回应是一条"边"跳到下一个节点。

```
NPC: "欢迎回来！要来一杯吗？还是打听点消息？"
         │
    ┌────┼────┬──────────┐
    ▼    ▼    ▼          ▼
 "冒险  "酒馆  "来一杯"  "只是路过"
  消息"  闲聊"              │
    │                        ▼
    ▼                      "注意安全。"
   "听说北边矿坑           (对话结束)
    出了怪事…"
```

- 每个节点属于一个 **NPC**（或"系统"——如金币不足提示）
- 每个节点可以有 0-4 个选项
- 选项可以：跳转到另一个节点、触发检定（说服/威吓等）、触发后果（金币/关系/任务）、或结束对话

### 2. 对话文本来源

对话文本有两个来源，由节点类型决定：

**固定文本**：设计师预先写好（如酒馆老板马莉亚的基本问候）。程序填充 `{变量}` 占位符——玩家名字、金币数、冒险者名字。

**LLM 生成文本**：标记为"动态"的节点，调用 DM Agent 实时生成。例如酒馆老板的"八卦"内容、冒险中 NPC 的对话——这些内容每次冒险都不同，不可能预写。

MVP 策略：
- 酒馆固定 NPC（老板马莉亚）：基本对话预写 + 情报内容 LLM 动态生成
- 冒险中遭遇 NPC：全部 LLM 动态生成（离线时使用模板文本降级）
- 冒险者队友对话：Phase 2（关系系统驱动）

### 3. 对话选项类型

玩家在对话中看到的选项分为三类：

**普通对话选项**：简单的语言回应。"最近有什么消息？""知道了，谢谢。"——没有检定，没有消耗，直接跳转。

**技能检定选项**：选项旁边显示 DC 和成功概率。玩家看到 `[DC 15 威吓] 你的加值: +3 成功率: 45%`——做出有信息的风险决策。
- 检定由骰子系统执行
- 成功 → 跳转到成功分支（NPC 被吓到/说服/欺骗）
- 失败 → 跳转到失败分支（NPC 更加敌对/拒绝/识破谎言）
- 自然 20 不自动成功（对话检定中 Nat20 无特殊效果——DND 5e RAW）

**代价选项**：选项有资源消耗。"来一杯。（5 GP）"——选择后自动扣除金币，给予对应效果。

### 4. 对话后果

所有对话的**机械后果由程序决定**，LLM 只负责叙事文本——遵循项目核心原则"LLM = 皮肤层"。

| 后果类型 | 示例 | 触发方式 |
|---------|------|---------|
| 关系变化 | 支持 NPC A → A 好感 +1 | 选择标记为 `relationship_change` 的选项 |
| 金币变化 | 买酒 -5 GP | 选择标记为 `gold_change` 的选项 |
| 任务触发 | 接取 NPC 个人请求 | 到达标记为 `quest_trigger` 的节点 |
| 情报解锁 | 得知隐藏冒险地点 | 到达标记为 `knowledge_unlock` 的节点 |
| 状态效果 | 喝酒 → 微醺 buff (+1 CHA) | 选择标记为 `buff_apply` 的选项 |
| 物品获取 | NPC 赠予物品 | 到达标记为 `item_grant` 的节点 |

### 5. 对话记忆

系统记住玩家与每个 NPC 的对话历史。这影响后续互动：

- **已访问节点**：NPC 不会重复已经说过的内容。第二次对话时，"最近有什么冒险消息？"选项可能不可用（因为它指向的八卦节点已经访问过）——或者指向更新的八卦内容
- **对话次数**：NPC 记得你们聊过几次——第 1 次和第 10 次的态度不同
- **NPC 记忆**：重要事件（"玩家曾帮我找回戒指"）被记录，影响 LLM 生成对话时的上下文

MVP 简化：仅追踪已访问节点和对话次数。NPC 记忆系统 Phase 2。

### 6. 对话在游戏流程中的位置

```
酒馆 → 玩家点击 NPC → 进入对话 → 选择选项 → 后果生效 → 退出对话 → 回到酒馆
冒险地图 → 触发对话节点 → 进入对话 → … → 退出对话 → 回到冒险地图
```

- 对话期间游戏暂停（战斗和探索不进行）
- 按 Esc 可退出对话（前提是当前节点允许退出）
- 对话历史可滚动查看（按 H 键）

## Player Experience

**玩家看到的**：
- 背景半透明暗化，前景对话窗口
- NPC 头像（32×32 像素）+ 名字 + 关系值
- NPC 对话文本逐字显示（打字机效果，0.03 秒/字）
- 2-4 个选项排列在下方，数字键 1-4 选择
- 技能检定选项显示 DC 和成功率

**玩家感受到的**：
- 选择有分量——"如果我选威吓而不是说服，NPC 会怎么反应？"
- NPC 不是复读机——多次对话内容会变化
- 对话在推进故事——不是在"跳过文本"，而是在"塑造叙事"

## Acceptance Criteria

- [ ] **AC-DIAL.01**：点击酒馆中的 NPC → 对话窗口打开，显示 NPC 名字和头像
- [ ] **AC-DIAL.02**：NPC 对话文本逐字显示（打字机效果），可按键跳过显示完整文本
- [ ] **AC-DIAL.03**：玩家看到 2-4 个选项，按 1-4 数字键选择，按 Esc 退出
- [ ] **AC-DIAL.04**：技能检定选项显示 DC、玩家加值和成功率（如 `[DC 15 威吓] +3 45%`）
- [ ] **AC-DIAL.05**：选择检定选项 → 骰子结果展示 → 成功/失败分支正确跳转
- [ ] **AC-DIAL.06**：选择后果正确执行：金币扣除/关系变化/任务生成/物品获取
- [ ] **AC-DIAL.07**：第二次与同一 NPC 对话时，已访问过的节点不再出现相同文本
- [ ] **AC-DIAL.08**：对话历史可查看（按 H 键），包含"NPC 说"和"你说"的完整记录
- [ ] **AC-DIAL.09**：LLM 生成的动态文本正常显示；离线时降级为预设模板文本
- [ ] **AC-DIAL.10**：对话结束后返回原场景（酒馆或冒险地图），场景状态不受破坏
