# Quick Design Spec: 场景管理 (Scene Management)

**Type**: New Small System
**Scope**: 定义玩家在游戏中可以进入哪些"空间"，以及如何在它们之间移动。酒馆是"家"，冒险地图是"旅途"，战斗是"危机"——场景切换本身就是"家→旅途→归来"循环的物理表达。
**Date**: 2026-05-09
**Estimated Implementation**: 已实现（Core/ 中的 Scene 框架），本 spec 补齐设计规范

---

## Overview

场景管理定义了《酒馆与命运》的**空间结构**——玩家在游戏世界中可以身处哪些地方，以及如何在它们之间移动。

游戏的三个核心幻想都在场景切换中体现：
- **Pillar 1（角色驱动的故事）**：从酒馆出发、经历冒险、带着故事归来——场景切换承载了叙事节奏
- **Pillar 3（持续演进的世界）**：回到酒馆时看到变化（新设施、新 NPC、冒险日志更新）——场景切换是"世界变了"的关键时刻

MVP 阶段玩家在 4 个场景之间流动：主菜单 → 酒馆 → 冒险地图 → 战斗 → 酒馆 → ……

---

## Scene Graph（玩家可进入的空间）

### MVP 场景（4 个）

```
主菜单 ──→ 酒馆 ──→ 冒险地图 ──→ 战斗
  ↑          ↑                      │
  │          └──────── 归来 ────────┘
  │          （结算后自动返回）
  └── 退出游戏 / 返回标题
```

**主菜单**（游戏入口）
- 玩家看到：标题画面、"开始游戏"按钮、版本信息
- 玩家操作：按 Enter 进入酒馆（新游戏）或选择加载存档
- 离开后不会回到这里（除非主动退出）——游戏一旦开始就是连续的

**酒馆**（"家"——Pillar 1 和 3 的物理容器）
- 玩家看到：招募板、任务板、队伍信息、酒馆环境
- 玩家操作：招募冒险者、接受任务、管理队伍、出发冒险
- 每次从冒险归来，酒馆可能不同：新设施解锁、冒险日志新增条目、角色状态变化

**冒险地图**（"旅途"——Pillar 1 的核心舞台）
- 玩家看到：地图节点、可选路线、当前队伍状态
- 玩家操作：选择下一节点（战斗/事件/休息）、主动撤退
- 进入方式：从酒馆组好队伍出发，携带选定的任务配置

**战斗**（"危机"——Pillar 2 的核心舞台）
- 玩家看到：战斗网格、敌我双方、HP/状态、行动菜单
- 玩家操作：回合制战术决策（移动/攻击/施法/使用物品）
- 进入方式：冒险地图中触发遭遇节点
- 离开后去往：胜利→回冒险地图继续；失败→回酒馆（伤亡结算后）

### Phase 2 场景（3 个附加）

**对话场景** — 冒险地图中触发叙事事件时进入，展示 NPC 对话树和选择
**据点场景** — 特定冒险类型结算后进入，管理建造和资源
**加载场景** — 过渡画面，资源加载时显示提示文字

## Transition Rules（场景切换规则）

### 什么样的玩家行为触发场景切换

| 源场景 | 玩家行为 | 目标场景 | 携带什么信息 |
|--------|---------|---------|-------------|
| 主菜单 | 选择"开始游戏"或加载存档 | 酒馆 | 存档数据（如果是加载）或空（如果是新游戏） |
| 酒馆 | 组队完成，选择任务，点击"出发冒险" | 冒险地图 | 队伍名单（4 个角色）+ 任务配置 |
| 冒险地图 | 选择遭遇节点 | 战斗 | 敌人配置、地图类型、队伍当前状态 |
| 冒险地图 | 主动撤退（撤退检定成功） | 酒馆 | 存活角色、消耗的资源、撤退原因 |
| 战斗 | 胜利（清除所有敌人） | 冒险地图 | 存活角色、消耗的法术位/HP、获得的战利品 |
| 战斗 | 失败（全灭或撤退） | 酒馆 | 伤亡名单、幸存者伤疤、丢失的装备、获得的经验 |

### 切换时发生什么（玩家感知）

1. **出发（酒馆→冒险）**：酒馆画面淡出，冒险地图加载。玩家感觉"踏出安全区"。
2. **遇敌（冒险→战斗）**：冒险地图定格，战斗网格展开。玩家感觉"危险降临"。
3. **胜利归来（冒险/战斗→酒馆）**：结算画面展示战利品和经验，然后酒馆加载。玩家感觉"回家了"——并且看到变化。
4. **失败归来（战斗→酒馆）**：伤亡结算展示，伤疤记录，然后酒馆加载。玩家感觉"代价沉重，但故事继续"。

### 什么不能做

- 不能从战斗直接回主菜单——战斗结果必须先结算
- 不能跳过酒馆直接进入冒险——必须先组队
- MVP 中不能 Save/Load 在战斗中途——存档只在酒馆触发

## Acceptance Criteria

- [ ] **AC-SM.01**：新游戏 → 主菜单按 Enter → 进入酒馆，看到招募板和任务板
- [ ] **AC-SM.02**：酒馆组好 1-4 人队伍 → 选择任务 → 点击出发 → 进入冒险地图
- [ ] **AC-SM.03**：冒险地图选择遭遇节点 → 进入战斗，敌人配置与任务匹配
- [ ] **AC-SM.04**：战斗胜利 → 战利品提示 → 返回冒险地图，队伍状态反映战斗消耗
- [ ] **AC-SM.05**：冒险完成（或战斗失败）→ 结算画面 → 回到酒馆，冒险日志有新条目
- [ ] **AC-SM.06**：回到酒馆后，角色 HP/法术位/状态如实反映上一场冒险的结果
- [ ] **AC-SM.07**：完整往返一次（酒馆→冒险→战斗→酒馆）不崩溃、不丢失数据、不残留旧场景元素
- [ ] **AC-SM.08**：MVP 中没有"战斗中途直接回主菜单"的路径
