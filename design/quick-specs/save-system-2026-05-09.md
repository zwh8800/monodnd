# Quick Design Spec: 存档系统 (Save/Load)

**Type**: New Small System
**Scope**: 管理游戏存档的创建、加载和删除。MVP 阶段支持酒馆场景下的手动存档和中冒险途中的自动存档点。
**Date**: 2026-05-09

---

## Overview

存档系统是 Pillar 3（持续演进的世界）的持久化基础——玩家的角色、关系、酒馆状态、冒险日志必须在关闭游戏后继续存在。存档系统不负责"设计什么值得保存"（那是各子系统自己的事）——它只负责序列化、存储和恢复。

MVP 存档策略：
- 短冒险（~30 分钟）：无中途存档——一次性体验。完成或失败后自动保存结算结果
- 中冒险（~3 小时）：每完成 1/3 节点时自动存档一次
- 酒馆：随时可手动存档

---

## Core Rules

### 存档时机

| 时机 | 触发方式 | 说明 |
|------|---------|------|
| 冒险结算完成 | 自动 | 回到酒馆后自动保存——角色变化、战利品、冒险日志全部写入 |
| 中冒险检查点 | 自动 | 每完成 1/3 进度时弹出"检查点已保存"提示，不打断游戏 |
| 酒馆手动存档 | 玩家操作 | 酒馆菜单选择"存档"，可覆盖已有存档或新建存档位 |

### 存档内容

存档保存以下数据：
- 角色数据：所有冒险者的 HP、属性、法术位、装备、伤疤、关系值
- 酒馆状态：酒馆等级、已解锁设施、招募池
- 世界状态：城镇安全度、势力关系、已发现的冒险
- 冒险日志：已完成冒险的名称/结局/摘要
- 元数据：存档时间、存档版本号

### 存档位

- 3 个存档位 + 1 个自动存档位
- 自动存档位不可手动覆盖——仅在冒险结算时写入
- 删除存档需要确认

### 版本兼容

- 存档文件包含版本号
- 新版本游戏可读取旧版本存档（向前兼容）
- 读取旧版本存档后自动升级为当前版本格式（迁移）
- 无法迁移的旧版本存档标记为"不兼容"，不可加载

### 什么不存档

- 战斗中途状态（本游戏无战斗中存档）
- LLM 生成的叙事文本（缓存在内存中，不需要持久化——重新生成即可）
- 临时 Buff 和状态（微醺 Buff 等——下次冒险重新计算）

## Acceptance Criteria

- [ ] **AC-SAVE.01**：酒馆中手动存档 → 退出游戏 → 重新启动 → 加载存档 → 角色和酒馆状态与存档时一致
- [ ] **AC-SAVE.02**：冒险完成后自动保存 → 退出 → 重新加载 → 冒险日志有新条目
- [ ] **AC-SAVE.03**：中冒险检查点自动保存 → 退出 → 重新加载 → 从检查点继续（而非冒险开头）
- [ ] **AC-SAVE.04**：3 个手动存档位 + 1 个自动存档位均可正常读写
- [ ] **AC-SAVE.05**：删除存档有二次确认，不可撤销删除
- [ ] **AC-SAVE.06**：连续 10 次存档加载循环（存档→加载→再存→再加载）数据不损坏、不丢失
