# 敌人AI系统 — 技术设计文档

> **Status**: In Design
> **Author**: Sisyphus + user
> **Last Updated**: 2026-05-09
> **Implements Pillar**: P2 (战术深度——原汁原味DND)

## Overview

敌人AI系统为所有非玩家战斗单位提供决策能力。它决定了敌人选择攻击谁、使用什么能力、移动到什么位置、何时逃跑——这些决策共同构成了玩家面对的战术挑战。没有AI，敌人就是木桩，战斗退化为纯数值交换——违背 Pillar 2"战术深度"的核心承诺。

系统基于分层决策模型：每回合依次完成态势评估→目标选择→行动选择→位置选择的四步流程，不同敌人类型（近战/远程/施法者/治疗者/刺客）有不同的行为模板，不同难度级别（Easy/Normal/Hard/Tactical）影响决策精度。MVP 阶段提供 Normal 难度的 5 种行为模板，覆盖所有基础敌人类型。

本 GDD 从 `04-combat-system.md` §11 提取并扩展，成为敌人AI行为的单一权威设计文档。

## Player Fantasy

> 敌人AI是玩家面对的"看不见的对手"——聪明的AI让每一场战斗都像在对抗一个活生生的敌人而非训练木桩。哥布林从侧翼包抄、骷髅无视恐惧冲锋、巨魔优先攻击治疗者——这些行为模式让不同敌人类型有可辨识的"个性"，让玩家需要针对不同敌人调整战术。AI不追求打败玩家，而是追求让玩家打败它时感到自己是真的赢了。

## Detailed Design

### Core Rules

#### AI决策流程

```
每回合每个敌人依次执行:
  Step 1: 评估战场态势
    - 统计双方存活人数
    - 识别高威胁目标（高伤害输出/低HP）
    - 评估自身HP、资源和条件状态

  Step 2: 选择目标
    - 以打分制评估每个可用目标
    - 分数 = 距离分(越近越高) + HP分(越低越高) + 威胁分(伤害输出) + 特殊分(专注中/治疗者)
    - 取最高分目标
    - 特定敌人类型可覆盖此逻辑（如治疗者优先目标=受伤最重队友）

  Step 3: 选择行动
    - 基于敌人类型的行为模板
    - 优先使用最有效的可用行动（CD技能→高伤害技能→基础攻击）
    - 考虑条件效果：被目盲→不选择依赖视觉的技能；被恐慌→不靠近恐慌源

  Step 4: 选择位置
    - 近战: 移向目标，首选侧翼位置
    - 远程: 保持距离（至少20尺），利用掩体
    - 施法者: 远离前线，保持法术射程内
    - 考虑地形：避免困难地形，优先高地
```

#### 敌人行为模板 (MVP 5种)

| 类型 | 战斗策略 | 目标偏好 | 位置策略 | 撤退条件 |
|------|----------|----------|----------|:--:|
| 近战战士 | 冲锋→攻击→驻守 | 最近敌人 | 前线，堵路 | HP<20% |
| 远程射手 | 射击→后退→掩护 | 最低HP敌人 | 后方，掩体后 | HP<15% |
| 施法者 | 法术→撤退→保护 | 高威胁敌人 | 远离前线 | HP<25% |
| 治疗者 | 治疗→辅助→自保 | 受伤最重队友 | 中后方 | HP<20% |
| 刺客 | 潜行→偷袭→脱离 | 最高威胁敌人 | 侧翼，绕后 | 无(激进) |

#### 协作逻辑

- **集火**: 多个敌人攻击同一目标时，第二个开始获得命中+2（象征战术配合）
- **保护**: 近战敌人挡在远程/施法者前方，阻碍玩家接近后排
- **响应**: 当治疗者被攻击时，1个近战敌人切换目标保护治疗者

#### 难度级别

| 难度 | 目标选择 | 行动选择 | 位置策略 | 玩家体验 |
|:---:|----------|----------|----------|----------|
| Easy | 随机 | 随机 | 随机 | 教学/轻松模式 |
| Normal | 启发式打分 | 模板驱动 | 模板驱动 | **MVP 标准** |
| Hard | 精确计算 | 最优选择 | 最优位置 | Phase 2 |
| Tactical | 全局最优 | 配合队友 | 利用一切 | Boss 专用 |

### Interactions

| 系统 | 交互 | 说明 |
|------|------|------|
| 角色系统 (01) | 读取敌人/玩家属性用于打分和目标评估 | 硬依赖 |
| 条件效果 (10) | 查询目标/自身条件以影响决策（如恐慌→不靠近、麻痹→优先攻击） | 硬依赖 |
| 战斗系统 (04) | 提供回合控制、移动/行动API；AI决策驱动敌人行动 | 硬依赖 |
| 地图探索 (05) | 读取地形数据用于位置选择（掩体/高地/困难地形） | 软依赖 |

## Formulas

### 目标选择打分

```
target_score = distance_score + hp_score + threat_score + special_score

  distance_score = max(0, 20 - distance_in_feet) × 1.0
  hp_score = (1 - target.current_hp / target.max_hp) × 15
  threat_score = target.estimated_dpr × 2
  special_score =
    +10 if target.is_concentrating
    +8  if target.is_healer
    -5  if target.has_condition("poisoned")
    +5  if target.has_condition("paralyzed") or target.has_condition("unconscious")
```

### 撤退判定

```
should_retreat = (enemy.current_hp / enemy.max_hp) < retreat_threshold
  - 不同类型有不同的 retreat_threshold (近战20%, 远程15%, 施法者25%, 治疗者20%, 刺客无)
  - 被标记 no_retreat 的敌人（如亡灵）忽略此判定
  - 撤退条件: 队伍存活人数 ≥ 2 (至少有人能断后)
```

## Edge Cases

- **若所有目标均不可达（被地形阻断）**: 选择最近可达位置移动，本回合跳过攻击
- **若敌人被恐慌且恐慌源是唯一可选目标**: 攻击劣势+不能移动靠近（恐慌规则优先）
- **若敌人同时满足多个行为模板（如 Boss 阶段切换）**: Boss 阶段定义的模板优先于基础类型模板
- **若治疗者自身 HP<30% 且无其他受伤队友**: 治疗自己而非等待
- **若所有近战敌人死亡，仅剩远程**: 远程敌人改变位置策略→边退边打，不再固守后方
- **若玩家全灭**: 敌人停止攻击，播放胜利动画

## Tuning Knobs

| 参数 | 默认值 | 说明 |
|------|:-----:|------|
| 距离分权重 | 1.0 | 距离对目标选择的影响 |
| HP分权重 | 15 | 低HP目标吸引力 |
| 威胁分权重 | 2 | 伤害输出对标分的影响 |
| 专注打断加分 | 10 | 攻击专注中的目标加分 |
| 撤退生命阈值(近战) | 20% | — |
| 集火命中加成 | +2 | 协作集火的命中奖励 |
| 默认难度 | Normal | MVP 阶段的AI难度 |

## Acceptance Criteria

- **GIVEN** 一个近战哥布林，**WHEN** 3个玩家角色在场，**THEN** 哥布林选择最近的目标攻击
- **GIVEN** 一个远程射手+一个被麻痹的玩家，**THEN** 射手优先攻击被麻痹的角色
- **GIVEN** 一个治疗者+一个受伤队友(HP<50%)，**THEN** 治疗者选择治疗行动而非攻击
- **GIVEN** 一个HP<20%的骷髅（标记 no_retreat），**THEN** 骷髅不撤退，继续攻击
- **GIVEN** 一个专注中的法师被哥布林评估，**THEN** 哥布林的目标打分含 +10 专注打断分
- **GIVEN** 一个被恐慌的敌人且恐慌源在视线内，**THEN** 该敌人不选择靠近恐慌源的位置

## Open Questions

- **Boss 多阶段 AI**: Boss 阶段切换时是否使用全新行为模板？（建议: 是——每个阶段有独立的行为模板，如巨魔 P1=狂暴近战, P2=投石远程）
- **AI 作弊边界**: Hard/Tactical 难度下 AI 是否可以"预知"玩家行动？（建议: 不可——AI 不应访问玩家计划中的行动，Hard/Tactical 的区别仅在于决策精度而非信息不对称）
