## 1. 概述

### 1.1 文档目的

本文档定义《酒馆与命运》中**每一个屏幕、每一种交互、每一处视觉规格**的完整技术设计。它是美术师的像素资产参考、程序员的实现蓝图、设计师的交互规范。

### 1.2 视觉风格指南

#### 1.2.1 核心参考：SFC/FF6 像素艺术 + 中世纪手稿点缀

本游戏的视觉风格遵循 SFC（Super Famicom）时代的像素艺术传统，核心参考为《最终幻想6》（FF6）和《超时空之轮》（Chrono Trigger）。UI 窗口框架采用 FF6 暗色半透明科技窗风格（确保战术信息高可读性），辅以中世纪手稿风格的装饰点缀（窗角花纹、仿手稿标题字体）——形成"冷UI 框架 + 暖装饰细节"的对比层次。

> **2026-05-10 裁决**：视觉风格 = FF6 暗色窗框架 + 中世纪手稿装饰点缀 + 暖色酒馆场景背景（背景透过半透明窗口可见，窗口角色为"信息容器"）。

| 元素 | 规格 | 说明 |
|------|------|------|
| 角色精灵 | 16×24 px | 2头身比例，4方向行走，每方向3帧动画 |
| 地图Tile | 32×32 px | 5尺×5尺游戏内网格 |
| 窗口框架 | 深色镶边 + 半透明背景 | FF6风格暗色边框，内填充深蓝/深灰半透明（rgba(10,10,30,0.85)） |
| 窗角装饰 | 8×8 px 像素花纹 | 中世纪手稿风格的花纹装饰，置于窗口四角 |
| 字体 | TrueType CJK + 等宽 Latin | CJK: FontStashSharp + Noto Sans CJK；Latin: 可选 8×8 位图字体备选方案 |
| 调色板 | 冷色UI框架 + 暖色场景背景 | 见§12完整色板定义 |

> **窗角装饰说明**：每个窗口的四角各放置一个 8×8 px 的手稿花纹 Tile（藤蔓/叶片/照明图案），为 FF6 暗色窗注入温暖手工感。花纹 Tile 不占用窗口内容区域，置于 Chrome 外边框与内边框之间的角位。

#### 1.2.2 窗口Chrome风格

所有UI窗口采用统一的FF6风格窗口框架：

```
窗口Chrome结构 (放大示意):
┌──────────────────────────────┐  ← 外边框: 2px 深色 (#1a1a2e)
│╔════════════════════════════╗│  ← 内边框: 1px 高光 (#4a4a6a)
│║                            ║│
│║    窗口内容区域              ║│  ← 背景: 半透明深色 (rgba(10,10,30,0.85))

```

#### 1.2.3 核心设计原则

| 原则 | 说明 | 实施要求 |
|------|------|----------|
| **像素完美** | 所有UI元素对齐像素网格，无亚像素偏移 | 使用像素对齐，确保UI元素对齐像素网格 |
| **可读性优先** | 文字在任何背景下清晰可读 | 所有文字带1px深色描边或背景板 |
| **响应式缩放** | 支持1280×720到2560×1440 | 采用响应式缩放策略，基准分辨率1280×720 |
| **怀旧但功能** | SFC美学不牺牲现代UX | 保留hover提示、快捷键、工具提示 |
| **信息层级** | 重要信息突出，次要信息弱化 | 使用颜色/大小/动画区分层级 |
| **一致性** | 所有屏幕共享同一套UI组件库 | 统一Theme Resource |

## 1A. 玩家体验幻想

- **"像素DND的质感"** — SFC黄金时代像素风 + 中世纪手稿UI元素，创造怀旧但精致的视觉体验。
- **"信息清晰，战术可见"** — 战斗UI必须清晰呈现所有DND状态和数值。像素美学不能以牺牲可读性为代价。

## 1D. 公式

> 本章节汇总 UI/UX 系统中所有需要数学定义的规格值。像素渲染和动画系统依赖这些公式进行确定性计算。

### 1D.1 缩放计算公式

| 变量 | 符号 | 定义 |
|------|------|------|
| 设计基准宽度 | `BASE_W` | 1280 px |
| 设计基准高度 | `BASE_H` | 720 px |
| 实际窗口宽度 | `window_w` | 运行时获取 |
| 实际窗口高度 | `window_h` | 运行时获取 |
| 用户缩放偏好 | `user_scale` | 0.8, 1.0, 1.2（设置中可选） |
| 计算缩放因子 | `scale_factor` | 见下方公式 |

**缩放因子计算**（整数倍像素完美优先 — 仅用于精灵/Tile/像素美术资产）：

```
raw_scale_x = window_w / BASE_W
raw_scale_y = window_h / BASE_H
base_scale = min(raw_scale_x, raw_scale_y)
final_scale = floor(base_scale × user_scale)
final_scale = clamp(final_scale, 1.0, 4.0)
```

- `floor()` 确保整数缩放因子（用于精灵/Tile），避免亚像素偏移
- `clamp(1.0, 4.0)` 防止过小（不可读）或过大（超出显存）
- **文字缩放独立**：`text_scale_factor = base_scale × user_scale`（不取 floor，允许亚像素文字渲染）。FontStashSharp 支持非整数缩放因子，文字可独立于精灵以非整数倍渲染
- 缩放后视口偏移居中：`offset_x = max(0, (window_w - BASE_W × final_scale) / 2)` — 防御性 `max(0, ...)` 防止窗口小于渲染区域时产生负偏移

**示例计算**：
- 1280×720, user_scale=1.0 → `scale_factor = 1.0`, `text_scale = 1.0`（基准）
- 1920×1080, user_scale=1.0 → raw=1.5, floor→`1.0`（1×精灵缩放，黑边）；`text_scale = 1.5`（文字以 1.5× 渲染，充分利用屏幕像素）
- 1920×1080, user_scale=1.2 → raw=1.5, floor(1.5×1.2)=floor(1.8)→`1.0`（精灵仍 1×）；`text_scale = 1.5 × 1.2 = 1.8`（文字以 1.8× 渲染）
- 2560×1440, user_scale=1.0 → raw=2.0, floor→`2.0`（像素完美 2×精灵）；`text_scale = 2.0`
- 2560×1440, user_scale=1.2 → floor(2.0×1.2)=floor(2.4)→`2.0`（精灵 2×）；`text_scale = 2.0 × 1.2 = 2.4`

### 1D.2 像素对齐公式

所有 UI 元素坐标在渲染前执行像素对齐：

```
aligned_x = round(x)
aligned_y = round(y)
aligned_width = round(width)
aligned_height = round(height)
```

- 对齐时机：每帧渲染前对所有 UI 元素执行
- 精度：`round()` 四舍五入到最近整数像素
- 影响范围：窗口位置、文字坐标、精灵位置、按钮边界

### 1D.3 字体渲染尺寸计算

字体使用 FontStashSharp TrueType 渲染（Noto Sans CJK），通过独立文字缩放因子进行亚像素缩放。
像素美术字体的尺寸计算（基于基准字体大小 16px，设计空间中的参考值）：

```
// 三种基准字号（设计空间 1280×720 下的参考值）
BASE_FONT_TITLE = 16  // px — 屏幕标题、重要标签
BASE_FONT_BODY  = 12  // px — 对话、描述、菜单
BASE_FONT_SMALL = 9   // px — 工具提示、次要信息（≥9px 确保可读性）
BASE_FONT_DAMAGE = 14 // px — 伤害数字、属性值

// 渲染尺寸 = 设计空间基准 × 文字缩放因子（非整数，FontStashSharp 支持）
display_font_size = BASE_FONT_SIZE × text_scale_factor

// 字体度量（用于布局计算）
CJK_CHAR_WIDTH = display_font_size      // 中文字符近似等宽 1:1
LATIN_CHAR_WIDTH = display_font_size × 0.55 // 拉丁字符平均宽度约为中文字符的 55%
LINE_HEIGHT = display_font_size × 1.5    // 行高（1.5倍行距）

// 文字缩放因子（独立于精灵缩放，见 §1D.1）
text_scale_factor = base_scale × user_scale  // 不取 floor，允许非整数
```

> **字体方案说明**：CJK 使用 FontStashSharp + Noto Sans CJK TrueType 渲染（禁用抗锯齿 + 整数缩放近似像素效果）。Latin 可另选 8×8 等宽位图字体作为备选方案（需另行制作）。不再要求 16×16 位图 CJK 字体——中文字符集规模（6,000+ 字符）使全像素字体资产不可行。

### 1D.4 动画时序公式

| 动画类型 | 公式 | 说明 |
|----------|------|------|
| 面板滑入/滑出 | `t = 0.3s / anim_speed` | `anim_speed` 为用户设置（0.5-2.0）；最小 0.08s |
| 按钮悬停 | `brightness += 10%` over `0.1s / anim_speed` | 线性插值 |
| 按钮按下 | `scale 1.0 → 0.95` over `max(0.05, 0.05 / anim_speed)s` | 缓出曲线；最小 0.05s（3 帧）确保动画始终可见 |
| 骰子动画 | `t = 1.0s / combat_speed` | 受战斗速度设置影响（1×/2×/4×）；4×速度下为 0.25s 跳过旋转 |
| 浮动伤害数字上浮 | `y_offset = -(damage_float_distance) × t / T` where `T = 0.8s / anim_speed`, `damage_float_distance = 48` px | 固定总位移 48px，仅时长随速度变化 |
| 浮动伤害数字消失 | `alpha(t) = 1.0 - t` over `max(0.08, 0.1 / anim_speed)s`（最后阶段） | 线性淡出；最小 0.08s（5 帧）确保淡出可见 |

> **安全防护**：所有 `x / anim_speed` 操作前须检查 `anim_speed > 0`。若配置异常导致 `anim_speed ≤ 0`，则回退至 `anim_speed = 1.0`。战斗相关动画受 `combat_speed` 影响（§11.6），其他动画受全局 `anim_speed` 影响。

> **§1D.4 与 §13 的关系**：§1D.4 定义了动画的数学公式，§13 定义了各动画的具体视觉规格（帧数、精灵、颜色）。当两者冲突时，以 §1D.4 公式为准——§13 为视觉参考，§1D.4 为可计算规范。已统一伤害浮动总量为 48px（替换原 §13.5 中不一致的 40px 描述）。

### 1D.5 窗口布局尺寸计算

**酒馆主窗口**（居中，最大 80% 视口）：

```
max_width = window_w × 0.8
max_height = window_h × 0.8
content_width = min(BASE_W × 0.75 × scale_factor, max_width)
content_height = min(BASE_H × 0.7 × scale_factor, max_height)
chrome_border = 4 × scale_factor  // 外边框2px + 内边框2px
total_width = content_width + chrome_border × 2
total_height = content_height + chrome_border × 2
```

**战斗 HUD 布局**（底部横栏）：

```
hud_height = 120 × scale_factor
hud_y = window_h - hud_height
action_bar_height = 48 × scale_factor
character_bar_height = 64 × scale_factor
```

## 1E. 边界情况

> 以下边界情况描述了 UI/UX 系统在极端条件下的预期行为。每个场景均明确程序应如何处理，不得仅标记为"优雅处理"。

### 1E.1 分辨率边界

| # | 场景 | 预期行为 |
|:--:|------|----------|
| E1 | 窗口宽度 < 1280px | 锁定最小尺寸 1280×720，阻止进一步缩小（显示提示："最小分辨率 1280×720"） |
| E2 | 窗口高度 < 720px | 同 E1，锁定最小高度 |
| E3 | 窗口宽度 > 3840px | 缩放因子上限 4.0（4K 以上不再放大）。超出部分显示黑色背景 |
| E4 | 窗口比例非 16:9（如 21:9 超宽屏） | 居中渲染 16:9 内容区域，左右黑边填充。UI 元素不拉伸变形 |
| E5 | 用户缩放 0.8× 在 1280×720 下 | `scale_factor = floor(1.0 × 0.8) = 0`，clamp 到 1.0。提示用户"当前分辨率不支持缩小" |

### 1E.2 输入设备边界

| # | 场景 | 预期行为 |
|:--:|------|----------|

| E12 | 100+ 条目的列表（如物品栏） | 启用虚拟化滚动，仅渲染可见行 + 上下各 2 行缓冲区。不可一次性渲染全部条目 |

```

  ├─ 戏法 (无限)
  │   ├─ Fire Bolt (1d10 fire)  → 目标选择 → 确认 → 执行
  │   └─ Minor Illusion         → 位置选择 → 确认 → 执行
  ├─ 1环 (剩余: 3/4)           ← 法术位显示
  │   ├─ Magic Missile (3×1d4+1) → 目标选择 → 自动命中
  │   ├─ Shield (反应)           → 被动触发
  │   └─ Thunderwave (2d8)       → 确认 → AOE范围显示
  ├─ 2环 (剩余: 2/3)
  │   └─ Scorching Ray (3×2d6)  → 目标选择(可多目标) → 确认
  └─ 3环 (剩余: 1/2)
      └─ Fireball (8d6)         → 位置选择 → AOE范围预览 → 确认

[物品]
  ├─ 治疗药水 (2d4+2 HP)       → 目标选择(自身/队友) → 确认
  ├─ 解毒剂                    → 目标选择 → 确认
  └─ 火把                      → 位置选择 → 确认

[技能]
  ├─ Hide (隐匿检定)            → 确认 → 骰子
  ├─ Help (援助队友)            → 队友选择 → 确认
  ├─ Dodge (躲避)               → 确认
  └─ Dash (冲刺)                → 确认

[待机]
  └─ 结束回合，不执行任何行动
```

## 14. 无障碍设计

### 14.1 色盲模式

> **医学分类修正（2026-05-10 评审）**：原文档将 Deuteranopia 误标为"蓝黄色盲"（实为红绿色盲/绿色弱），将 Tritanopia 误标为"全色盲"（实为蓝黄色盲）。以下为正确的医学分类及逐元素色彩映射。

| 模式 | 正确医学名称 | 感知特点 | 逐元素色彩映射 |
|------|-------------|----------|---------------|
| **红色盲** (Protanopia) | 红色视锥细胞缺失 | 红色感知极弱，红绿难辨 | 伤害红→橙色(#FF8C00)；治疗绿→蓝色(#4488FF)；HP绿→青色(#44FFFF)；敌方红框→橙色虚线框；HP红→橙色 |
| **绿色盲** (Deuteranopia) | 绿色视锥细胞缺失 | 绿色感知极弱，红绿难辨 | 伤害红→品红(#FF44AA)；治疗绿→青色(#44FFFF)；HP绿→青色(#44FFFF)；友好绿线→蓝色虚线；非常高绿→青色 |
| **蓝色盲** (Tritanopia) | 蓝色视锥细胞缺失 | 蓝黄难辨 | 蓝→紫色(#AA44FF)；黄→橙色(#FF8C00)；法术蓝→紫色(#AA44FF)；当前选中蓝→白色边框 |
| **高对比度模式** | 全色盲/低视力辅助 | 所有颜色感知弱 | 全部 UI 元素使用纯黑/纯白/纯黄高对比；所有边框≥2px；半透明背景→纯黑；依赖形状+文字双重编码 |

> **色彩映射实施要求**：每种色盲模式须对以下所有 UI 元素执行色彩替换：(1)伤害/治疗数字颜色、(2)HP条绿/黄/红三区、(3)敌/友边框、(4)关系连线颜色、(5)交互物体标签颜色、(6)稀有度颜色、(7)战斗日志强调色、(8)条件图标颜色。渲染顺序：颜色调整→缩放→渲染（见 §1E E17）。

### 14.2 文字缩放

> 文字使用独立的 `text_scale_factor`（见 §1D.1），与精灵/Tile 的整数 `scale_factor` 解耦。FontStashSharp 支持非整数缩放因子，确保文字在所有分辨率下均可独立放大。

| 选项 | 缩放因子 | 1280×720 下正文 | 1920×1080 下正文 | 说明 |
|------|:---:|:---:|:---:|------|
| 小 | 0.8× | 9.6px | 14.4px (base_scale=1.5) | 更多信息可见 |
| 标准 | 1.0× | 12px | 18px | 默认 |
| 大 | 1.3× | 15.6px | 23.4px | 无障碍增强 |
| 特大 | 1.6× | 19.2px | 28.8px | 无障碍优先（≥WCAG 18px） |

> **注意**：1920×1080 分辨率下 `base_scale=1.5`，因此 `text_scale = base_scale × user_scale = 1.5 × user_scale`。在"特大"模式下，正文可达 28.8px——满足 WCAG 2.1 AA 要求。

**THEN** 金币减为 1248 GP；匕首出现在背包中；商店列表项若有库存限制则数量减1

#### AC-UI-22: 金币不足拒绝购买
**GIVEN** 商店购买标签页已打开，玩家金币=0 GP  
**WHEN** 尝试购买任意物品  
**THEN** [购买]按钮灰色禁用，或点击后显示"金币不足"提示(2秒后自动消失)；背包不变

#### AC-UI-23: 空背包显示
**GIVEN** 角色背包为空（0个物品）  
**WHEN** 打开装备标签页的背包区域  
**THEN** 背包区域显示"背包为空"提示文字，无异常空白或布局错位

### 15.6 响应式缩放（3 条）

#### AC-UI-24: 1280×720 酒馆主屏幕布局正确
**GIVEN** 窗口分辨率=1280×720，user_scale=1.0  
**WHEN** 渲染酒馆主屏幕并截图  
**THEN** 顶部HUD栏完整可见(等级/声望/金币)；底部操作栏6按钮均可见；无UI元素裁剪或重叠

#### AC-UI-25: 1920×1080 文字独立缩放
**GIVEN** 窗口分辨率=1920×1080，文字缩放=1.6×（特大），user_scale=1.0  
**WHEN** 渲染角色面板属性标签页并截图  
**THEN** sprite缩放=1.0×（精灵保持整数像素）；文字以 1.6×1.5=2.4× 实际尺寸渲染；正文清晰可读无模糊；精灵无亚像素偏移

#### AC-UI-26: 2560×1440 像素完美
**GIVEN** 窗口分辨率=2560×1440，user_scale=1.0  
**WHEN** 渲染酒馆主屏幕并截图  
**THEN** sprite以2×整数倍渲染；窗口Chrome的2px外边框在截图中呈现4个整像素宽度；UI文字边缘清晰锐利无亚像素色彩溢出

### 15.7 无障碍（4 条）

#### AC-UI-27: 色盲模式颜色正确替换
**GIVEN** 设置中启用了红色盲(Protanopia)模式，战斗模式中的伤害数字  
**WHEN** 造成一次伤害并截图  
**THEN** 伤害数字颜色为橙色(#FF8C00)而非默认红色(#FF4444)；治疗数字为蓝色(#4488FF)而非默认绿色

#### AC-UI-28: 文字缩放不导致溢出
**GIVEN** 文字缩放=1.6×（特大），酒馆主屏幕  
**WHEN** 渲染顶部HUD（等级/声望/金币）  
**THEN** 所有文字标签在各自容器内完整可见，无截断或相互重叠；若空间不足则省略号截断

#### AC-UI-29: 高对比度模式可用
**GIVEN** 启用高对比度模式  
**WHEN** 渲染商店购买列表、角色面板属性标签、战斗日志  
**THEN** 所有文字使用纯白(#ffffff)或纯黄(#ffff00)；所有背景使用纯黑(#000000)（半透明→纯黑）；所有边框≥2px粗线；交互元素轮廓清晰可辨

#### AC-UI-30: 屏幕震动关闭生效
**GIVEN** 设置中屏幕震动=关闭，暴击闪烁=开启  
**WHEN** 在战斗中触发暴击  
**THEN** 屏幕无震动位移；暴击金色闪烁特效正常播放（两者独立控制）

### 15.8 性能（3 条）

#### AC-UI-31: 战斗 UI 保持 60fps
**GIVEN** 完整战斗UI激活（4角色状态栏+先攻条+战术地图+行动菜单+战斗日志）  
**WHEN** 在战斗场景连续运行30秒，使用内置fps计数器记录帧率  
**THEN** 平均帧率 ≥ 59fps；最低瞬时帧率 ≥ 55fps

#### AC-UI-32: 100+物品列表滚动不卡顿
**GIVEN** 物品栏包含100个物品条目  
**WHEN** 从列表顶部持续滚动至底部（鼠标滚轮），使用fps计数器记录帧率  
**THEN** 滚动期间平均帧率 ≥ 58fps；列表项无渲染重叠/缺失/闪烁

#### AC-UI-33: 覆盖层打开延迟可接受
**GIVEN** 酒馆主屏幕，角色面板处于关闭状态  
**WHEN** 按下C键，测量从按键到面板首帧完成渲染的时间  
**THEN** 延迟 < 100ms；关闭（Esc）延迟 < 100ms

### 15.9 边界情况（3 条）

#### AC-UI-34: 覆盖层上限与降级(引用 §1E E10)
**GIVEN** 酒馆主屏幕中已打开4层覆盖层  
**WHEN** 尝试打开第5层覆盖层  
**THEN** 最底层覆盖层自动关闭(FIFO)；弹出"覆盖层已达上限"提示(2秒消失)；第5层成功打开

#### AC-UI-35: 加载时输入屏蔽(引用 §1E E16)
**GIVEN** 屏幕转换动画正在执行（fade_black过程）  
**WHEN** 测试人员快速按下多个键盘/鼠标按键  
**THEN** 所有输入被忽略；动画完成后0.2秒才恢复输入响应；无队列积压导致的意外行为

#### AC-UI-36: LLM降级状态UI
**GIVEN** LLM API调用超时或返回错误  
**WHEN** 对话覆盖层等待LLM生成文本  
**THEN** 显示"生成中…"加载指示器；若3次重试失败，降级显示本地模板文本；对话选项正常可用（离线可玩性保持）
