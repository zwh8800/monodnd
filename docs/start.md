## 二、任务系统——三档冒险的差异化设计

短/中/长三档的时长控制很重要，要让玩家感受到**结构上的本质区别**，而不只是长度不同：

短冒险（30分钟）适合设计为单点事件：一场劫匪袭击、一个地下室清剿、一次神秘商人交易，剧情几乎线性，重点在战斗和选择。中冒险（3小时）应该有2-3个地点、一个核心谜题和1-2个分支，玩家能感受到叙事弧线。长冒险（6小时）则需要完整的三幕结构，包含真正的道德困境、会记住玩家选择的关键NPC，以及多个结局可能性。

## 三、大模型生成——这是最核心也最需要工程设计的部分

建议把生成拆成**两阶段**：

第一阶段在冒险开始前完成"世界构建"——生成地图结构、剧情大纲、关键NPC数据（性格、秘密、目标），以及关键剧情节点的脚本。这些是结构性数据，可以用JSON schema严格约束输出格式，转成游戏系统可直接消费的数据。

第二阶段是冒险**进行中**的动态响应——NPC对玩家行为的即时反应、检定失败后的叙事描述、环境互动的文字反馈。这部分是轻量实时调用，要控制延迟。

失败惩罚的生成也需要精心设计：要让惩罚感觉"合理且有戏剧性"，比如被火焰怪击败的战士获得"惧焰"特质，而不是随机生成一个莫名其妙的debuff。可以把战斗日志和剧情上下文一并喂给模型，让它生成符合叙事逻辑的惩罚内容。

## 四、战斗系统——DND味道要做足

DND战斗的精髓在于**不确定性和策略叠加**，建议保留：先攻骰决定行动顺序；每次攻击的命中/未命中用可见的骰子动画呈现（像素风下的骰子动画会非常有趣）；每种职业有显著不同的战斗节奏，法师脆皮但AOE强，战士能控场，游荡者依赖先攻偷袭。

2D场景中可以做**地形交互**：推敌人落入悬崖、点燃油桶、翻倒桌子当掩体——这些在2D下完全可以实现，而且比3D更容易让玩家一眼看清局势。

## 五、Roguelike核心循环——永久后果的设计

这是整个游戏的情感核心。几个补充思路：

**角色伤疤系统**——成功冒险中积累的荣耀（比如曾击败过某个BOSS）和失败留下的伤疤（义肢、失忆、烙印）都应该永久留存，并体现在角色的外观和属性上，让每个角色都变得独一无二。

**英雄传记**——角色退役或阵亡时，由大模型根据ta的冒险记录生成一段"传记"，存入酒馆"英雄之壁"。这对玩家的情感投入会有很强的强化作用。

## 三、冒险生成系统：LLM的"剧本工厂"

这是整个项目的技术核心，也是最大的风险点。你需要设计一个**结构化生成管线**，而不是让LLM自由发挥。

### 3.1 三层生成架构

```
第一层：元数据生成（LLM）
├─ 冒险主题（如"被诅咒的精灵王国"）
├─ 核心冲突（如"王位继承+远古邪神苏醒"）
├─ 关键NPC（3-5个有完整动机的人物）
├─ 剧情大纲（起承转合的5-7个关键节点）
└─ 氛围标签（Gothic Horror / High Fantasy / Noir...）

第二层：结构化转换（程序）
├─ 将大纲切分为"场景节点"
├─ 为每个节点分配地图模板（森林/地城/城镇...）
├─ 根据冲突设计战斗遭遇的CR（Challenge Rating）
├─ 生成分支逻辑（选择A→节点X，选择B→节点Y）
└─ 设计奖励池（装备、道具、情报）

第三层：内容填充（LLM + 资产库）
├─ 为每个场景节点生成具体描述文本
├─ 为关键NPC生成对话脚本（多分支）
├─ 根据氛围标签选择像素Tileset和BGM
└─ 生成任务日志和物品描述
```

### 3.2 时长控制机制
你提到的30分钟/3小时/6小时，需要**量化设计**：

| 类型 | 场景节点数 | 战斗场数 | 剧情决策点 | 存档点 |
|------|-----------|---------|-----------|--------|
| 短冒险 | 5-8个 | 3-5场 | 8-12个 | 无（一次性） |
| 中冒险 | 15-25个 | 8-15场 | 20-30个 | 每完成1/3可存档 |
| 长冒险 | 30-50个 | 15-25场 | 40-60个 | 自由存档+自动存档 |

**关键设计**：短冒险可以"rogue-like纯随机"，中长冒险则更像"模块化DND模组"——由LLM生成一个完整的故事弧，但具体的房间布局、敌人配置、掉落物品是程序随机的。

### 3.3 LLM Prompt工程建议
不要指望一个Prompt解决所有问题。建议设计**专用Agent**：

- **编剧Agent**：负责生成主题、冲突、大纲
- **DM Agent**：负责根据玩家行动实时生成剧情分支和NPC反应
- **文案Agent**：负责生成物品描述、环境描写、对话文本
- **平衡Agent**：负责检查生成的战斗遭遇是否合理（CR是否匹配队伍等级）

每个Agent都有固定的System Prompt和输出Schema（用JSON/XML约束输出格式）。

## 四、战斗与场景交互：2D版博德之门3

### 4.1 战斗系统
既然要"原汁原味DND"，建议直接基于**DND 5e SRD（开放游戏许可）**做简化：

**保留的核心机制**：
- 六维属性（STR/DEX/CON/INT/WIS/CHA）
- 熟练加值（Proficiency Bonus）
- AC（护甲等级）命中判定
- 动作/附赠动作/反应（Action/Bonus Action/Reaction）
- 短休/长休资源管理

**简化的部分**：
- 法术位系统可以简化为"充能制"（每个法术有冷却回合）
- 取消复杂的子职业选择，改为"天赋树"形式
- 环境互动（油桶+火焰箭=爆炸）保留，但用2D物理简化计算

### 4.2 场景交互（2D版）
这是"博德之门感"的关键：

```
交互类型：
├─ 环境利用：推落悬崖、点燃草丛、破坏桥梁
├─ 物品互动：翻阅书籍获得线索、使用绳索跨越裂隙
├─ 社交解决：很多战斗可以通过对话避免（基于CHA检定）
├─ 潜行路线：非线性地图，支持绕后、伏击、逃跑
└─ 机关解谜：基于INT/WIS的简易谜题（不要做成纯解谜游戏）
```

**技术实现建议**：给每个场景对象打上**交互标签**（Pushable, Flammable, Climbable, Readable...），LLM生成场景时只需指定"这里有一个可推落的石像"，具体物理效果由游戏系统处理。

## 五、Rogue-like元素设计：失败与成长的循环

你的设计已经隐含了Rogue-like的核心循环，但可以更系统化：

### 5.1 元进度（Meta-progression）
- **酒馆等级**：永久解锁新职业、新种族、更高级的装备锻造
- **传说墙**：记录你完成的所有冒险，某些"传说"会解锁特殊任务线
- **知识传承**：失败冒险中获得的"情报"可以在下一次面对类似主题时提供加成（比如"已知这个地城有陷阱"→获得被动感知加成）

### 5.2 失败惩罚系统（由LLM生成）
这是你的亮点设计，但要避免"挫败感过强"：

| 惩罚等级 | 示例 | 恢复方式 |
|---------|------|---------|
| 轻微 | 丢失部分金币、消耗品 | 自动恢复 |
| 中等 | 装备损坏需修复、角色临时debuff | 酒馆休息/铁匠修复 |
| 严重 | 角色永久伤残（-1 STR）、装备彻底损毁 | 特殊任务/高价治疗 |
| 致命 | 角色死亡（但可招募新角色继承部分记忆） | 无法恢复，但解锁"墓志铭" |

**LLM生成惩罚的Prompt建议**：
```
基于以下冒险失败场景：[场景描述]
生成3个符合DND规则的惩罚选项：
1. 一个与剧情逻辑紧密相关的惩罚（如"被诅咒的宝藏"导致贪婪者失去金币）
2. 一个角色成长向的惩罚（如"目睹队友死亡"导致心理创伤）
3. 一个带有后续任务线的惩罚（如"被邪恶势力标记"导致未来遭遇伏击）

要求：惩罚必须有叙事包装，不能只是"扣50%金币"。
```

### 5.3 继承机制
- **装备继承**：成功冒险中获得的"灵魂绑定"装备可以跨冒险保留
- **角色记忆**：老角色退休时可以将一个"特技"传授给新角色
- **酒馆传承**：即使全员阵亡，酒馆本身保留等级和部分资源（真正的Rogue-lite设计）
